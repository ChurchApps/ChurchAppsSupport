# Zapier

<div class="article-intro">

B1.church पर आधिकारिक Zapier ऐप आपके चर्च में होने वाली घटनाओं (नया व्यक्ति, नया दान, नया समूह सदस्य, ...) पर प्रतिक्रिया करने देता है और B1 में रिकॉर्ड वापस लिखता है। कोई कोडिंग नहीं, कोई बुनियादी ढांचा नहीं — आप इसे Zapier के ड्रैग-और-ड्रॉप संपादक में सेट करते हैं, एक API कुंजी पेस्ट करते हैं, और Zap को चालू करते हैं।

</div>

<div class="prereqs">
<h4>शुरू करने से पहले</h4>

- एक [Zapier](https://zapier.com) खाता (मुफ्त स्तर कुछ Zaps के लिए पर्याप्त है)
- B1Admin में **Edit Settings** अनुमति वाला चर्च व्यवस्थापक (आप एक API कुंजी बनाएंगे)
- एक विचार कि आप क्या करना चाहते हैं — उदाहरण के लिए "जब B1 में कोई व्यक्ति जोड़ा जाए, तो उन्हें मेरी Mailchimp सूची में जोड़ें"

</div>

## ट्रिगर और क्रियाएं

| प्रकार | क्या | B1 घटना / एंडपॉइंट |
|---|---|---|
| **ट्रिगर** | नया व्यक्ति | `person.created` |
| **ट्रिगर** | अपडेट किया गया व्यक्ति | `person.updated` |
| **ट्रिगर** | नया दान | `donation.created` |
| **ट्रिगर** | नया समूह सदस्य | `group.member.added` |
| **ट्रिगर** | नई फॉर्म सबमिशन | `form.submission.created` |
| **क्रिया** | व्यक्ति बनाएं | एक नया व्यक्ति जोड़ता है |
| **क्रिया** | दान जोड़ें | एक दान रिकॉर्ड करता है |
| **क्रिया** | समूह सदस्य जोड़ें | किसी व्यक्ति को एक समूह में जोड़ता है |
| **खोज** | व्यक्ति खोजें | नाम या ईमेल से किसी व्यक्ति को देखता है |

इनमें से कोई भी Zapier के 7,000+ समर्थित ऐप्स के साथ स्वतंत्र रूप से संयोजन करें।

## सेटअप

### 1. एक B1 API कुंजी बनाएं

1. B1Admin में **Settings → Developer → API Keys** पर जाएं।
2. **New API Key** पर क्लिक करें, इसे "Zapier" जैसा नाम दें, और Zap को आवश्यक स्कोप चुनें।
3. **महत्वपूर्ण:** Zapier ट्रिगर आपकी ओर से एक webhook पंजीकृत करता है जब Zap चालू होता है, जिसके लिए **`settings:write`** स्कोप की आवश्यकता होती है। यदि आपके किसी भी Zaps B1 ट्रिगर का उपयोग करता हैं तो हमेशा `settings:write` शामिल करें।
4. साथ ही उन स्कोप को प्रदान करें जो क्रियाओं को आवश्यकता है — उदाहरण के लिए "Add Donation" क्रिया को `donations:write` चाहिए, "Create Person" को `people:write` चाहिए।
5. सहेजें। पूर्ण `cak_…` कुंजी **एक बार** दिखाई जाती है — इसे कॉपी करें।

### 2. Zapier को B1 से जोड़ें

1. Zapier में, एक नया Zap बनाएं।
2. जब आप पहली बार B1 ट्रिगर या क्रिया चुनते हैं, तो Zapier आपको **Sign in to B1.church** के लिए कहता है।
3. चरण 1 से API कुंजी पेस्ट करें और **Yes, Continue** पर क्लिक करें। Zapier इसे आपके चर्च के विरुद्ध सत्यापित करता है।

कनेक्शन Zapier में सहेजा जाता है और आपके खाते पर प्रत्येक Zap द्वारा पुन: उपयोग किया जाता है।

### 3. Zap बनाएं

एक ट्रिगर चुनें, फिर एक या अधिक क्रिया चरण जोड़ें। नीचे उदाहरण दिए गए हैं।

## आम व्यंजन

### नए B1 लोगों को Mailchimp में जोड़ें

- **ट्रिगर** — B1: नया व्यक्ति
- **क्रिया** — Mailchimp: सबस्क्राइबर जोड़ें/अपडेट करें। B1 के `name__first`, `name__last`, `contactInfo__email` को Mailchimp के First Name / Last Name / Email फील्ड में मैप करें।

### दान को एक Slack चैनल में अंतर्निहित कनेक्टर से बेहतर कार्ड के साथ पोस्ट करें

- **ट्रिगर** — B1: नया दान
- **क्रिया** — Slack: चैनल संदेश भेजें। कोई भी लेआउट बनाएं — बटन, संलग्नक, आदि — जो अंतर्निहित [Slack कनेक्टर](./slack-discord) नहीं कर सकता।

### एक Google Group में नए समूह सदस्यों को जोड़ें

- **ट्रिगर** — B1: नया समूह सदस्य (विशिष्ट `groupId` को फ़िल्टर किया गया)
- **क्रिया** — Zapier द्वारा फ़िल्टर करें: केवल जारी रखें यदि B1 समूह वह है जिसकी आप परवाह करते हैं
- **क्रिया** — B1: व्यक्ति खोजें (ईमेल लाने के लिए ट्रिगर के `personId` का उपयोग करें)
- **क्रिया** — Google Groups: सदस्य जोड़ें

### फॉर्म सबमिशन को एक प्रोजेक्ट ट्रैकर में आगे बढ़ाएं

- **ट्रिगर** — B1: नई फॉर्म सबमिशन
- **क्रिया** — Notion / Linear / Asana / Trello: पृष्ठ / समस्या / कार्य बनाएं

## ट्रिगर कैसे काम करते हैं

ट्रिगर **REST हुक** हैं, पोलिंग नहीं — Zapier हर 15 मिनट में B1 को नहीं पिंग करता है। जब आप Zap को चालू करते हैं, तो Zapier B1 को एक webhook पंजीकृत करने के लिए कहता है जो एक निजी Zapier URL की ओर इशारा करता है; जब घटना होती है, तो B1 Zapier को एनवेलॉप को POST करता है और आपका Zap **कुछ सेकंड में** शुरू हो जाता है। Zap को बंद करें और Zapier B1 को webhook को हटाने के लिए कहता है — कोई अनाथ सदस्यता नहीं।

इसका मतलब यह है कि ट्रिगर केवल **Zap चालू होने के बाद** होने वाली घटनाओं के लिए आग लगाता है। कोई बैकफिल नहीं है — Zap को चालू करने से कल के दान को फिर से नहीं दिखाया जाता है।

## सीमाएं और नोट्स

- **समान ट्रिगर वाले कई Zaps** प्रत्येक अपना B1 webhook पंजीकृत करता है — कोई संघर्ष नहीं है, लेकिन यह जानने लायक है यदि आप **Settings → Developer → Webhooks** का निरीक्षण कर रहे हैं और सोच रहे हैं कि तीन समान `Zapier — donation.created` पंक्तियां वहां क्यों हैं।
- **Zap सेटअप में परीक्षण डेटा** — जब आप Zap बनाते हैं, तो Zapier फील्ड को मैप करने के लिए नमूना डेटा मांगता है। यदि कोई हो तो यह B1 से सबसे हाल की मिलान घटना खींचेगा; अन्यथा यह ऐप परिभाषा से एक सिंथेटिक नमूना उपयोग करता है।
- **क्रिया विफलताएं Zap त्रुटियों के रूप में सामने आती हैं** Zapier के कार्य इतिहास में। सामान्य कारण: एक API कुंजी बिना सही स्कोप (उदाहरण के लिए "Add Donation" क्रिया को `donations:write` चाहिए)। सही स्कोप के साथ कुंजी को फिर से मिंट करें और Zapier में फिर से कनेक्ट करें।
- **आउटबाउंड API कॉल कोटा** — एक क्रिया से B1 API का प्रत्येक कॉल आपके Zapier कार्य कोटा की ओर गिना जाता है, B1 के पक्ष में कुछ नहीं।

## समस्या निवारण

- **कनेक्ट करते समय "Authentication failed"** — API कुंजी गलत है, रद्द की गई है, या Zap को आवश्यक स्कोप गायब हैं। इसे B1Admin में कम से कम `settings:write` और जो भी संसाधन स्कोप Zap छूता है के साथ फिर से मिंट करें, फिर कनेक्शन को अपडेट करें।
- **ट्रिगर कभी आग नहीं लगता है** — पुष्टि करें कि webhook वास्तव में पंजीकृत हुआ है: B1Admin में, **Settings → Developer → Webhooks** अब "Zapier — &lt;event&gt;" नाम की एक पंक्ति दिखानी चाहिए। यदि यह वहां नहीं है, तो API कुंजी में संभवतः Zap को चालू करने के समय `settings:write` की कमी थी। कुंजी को ठीक करें, Zap को बंद करें और फिर से चालू करें।
- **ट्रिगर दो बार आग लगता है** — Zapier कभी-कभी फिर से सुस्ताता है यदि इसकी स्वीकृति खो गई थी। Zap की एक अनोखी id (उदाहरण के लिए व्यक्ति का `id`) पर "Filter by Zapier" कदम का उपयोग करें यदि आपको सख्त डीडुप्लिकेशन की आवश्यकता है।
