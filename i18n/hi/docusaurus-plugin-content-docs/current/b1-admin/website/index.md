# वेबसाइट

<div class="article-intro">

B1 Admin में वेबसाइट अनुभाग आपको अपने चर्च के लिए एक पूर्ण-विशेषताओं वाला वेबसाइट बिल्डर प्रदान करता है। आप पृष्ठ बना सकते हैं, अपनी साइट के रूप और अनुभव को अनुकूलित कर सकते हैं, मीडिया फ़ाइलें प्रबंधित कर सकते हैं, और इवेंट कैलेंडर प्रकाशित कर सकते हैं -- बिना कोई कोड लिखे।

</div>

## आप क्या कर सकते हैं

वेबसाइट बिल्डर पांच मुख्य क्षेत्रों में व्यवस्थित है:

### पृष्ठ

अपनी चर्च वेबसाइट पर सभी पृष्ठ बनाएं और प्रबंधित करें। **Pages** तालिका प्रत्येक पृष्ठ को उसकी स्थिति के साथ दिखाती है -- या तो **Generated** (सिस्टम द्वारा स्वचालित रूप से बनाया गया) या **Custom** (आपके द्वारा बनाया गया)। आप नए कस्टम पृष्ठ जोड़ सकते हैं, मौजूदा संपादित कर सकते हैं, या ऑटो-जनरेटेड पृष्ठों को उनकी सामग्री पर पूर्ण नियंत्रण के लिए कस्टम पृष्ठों में बदल सकते हैं।

बायां साइडबार आपके नेविगेशन लिंक को ट्री स्ट्रक्चर में प्रदर्शित करता है। आप ड्रैग-एंड-ड्रॉप का उपयोग करके लिंक को पुनर्व्यवस्थित कर सकते हैं ताकि आगंतुकों को सही पदानुक्रम और क्रम दिखाई दे।

### ब्लॉक

पुन: प्रयोज्य सामग्री घटक बनाएं जिन्हें आप कई पृष्ठों पर रख सकते हैं। ब्लॉक आपका समय बचाते हैं जब आपके पास कॉल-टू-एक्शन, घोषणा बैनर, या संपर्क अनुभाग जैसी सामग्री होती है जो एक से अधिक स्थानों पर दिखाई देती है। एक ब्लॉक को एक बार अपडेट करें और परिवर्तन हर जगह लागू हो जाता है।

### अपीयरेंस

अपनी वेबसाइट की थीम, रंग, फ़ॉन्ट, लोगो और स्पेसिंग अनुकूलित करें। आप त्वरित शुरुआत के लिए पूर्व-निर्मित स्टाइल थीम लागू कर सकते हैं या रंग पैलेट संपादक, टाइपोग्राफी सेटिंग्स और कस्टम CSS के साथ हर विवरण को सूक्ष्मता से ट्यून कर सकते हैं। पूर्ण विवरण के लिए [अपीयरेंस](appearance) गाइड देखें।

### फ़ाइलें

अपनी सभी मीडिया संपत्तियां -- छवियां, दस्तावेज़, और अन्य फ़ाइलें -- एक केंद्रीय स्थान पर अपलोड और व्यवस्थित करें। चीज़ों को व्यवस्थित रखने के लिए फ़ोल्डर का उपयोग करें और अपनी साइट पर कहीं भी उपयोग के लिए फ़ाइल URL कॉपी करें। अधिक जानकारी के लिए [फ़ाइलें](files) देखें।

### कैलेंडर

अपनी वेबसाइट पर इवेंट कैलेंडर प्रकाशित करें ताकि आगंतुक देख सकें कि आपके चर्च में क्या हो रहा है। क्यूरेटेड कैलेंडर बनाएं जो कई समूहों के कार्यक्रमों को एक दृश्य में खींचते हैं।

:::tip
यदि आप पहली बार अपनी वेबसाइट कॉन्फ़िगर कर रहे हैं तो [प्रारंभिक सेटअप](initial-setup) से शुरू करें। यह आपको डोमेन सेटअप, अपना पहला पृष्ठ बनाना, और अपनी साइट की अपीयरेंस कॉन्फ़िगर करने के बारे में बताता है।
:::
