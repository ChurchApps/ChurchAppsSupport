# एपीआई

<div class="article-intro">

ChurchApps एपीआई एक **मॉड्यूलर मोनोलिथ** है -- एक एकल कोडबेस जो छह डेटा मॉड्यूल को सेवा प्रदान करता है, जिनमें से प्रत्येक के अपने डेटाबेस हैं। यह आर्किटेक्चर आपको माइक्रोसर्विसेज के संगठनात्मक लाभ (स्पष्ट सीमाएं, स्वतंत्र डेटा स्टोर) एक एकल तैनाती के संचालनात्मक सरलता के साथ देता है।

</div>

## मॉड्यूल

| मॉड्यूल | उद्देश्य |
|--------|---------|
| **Membership** | लोग, समूह, परिवार, अनुमतियां |
| **Attendance** | सेवाएं, सत्र, चेक-इन रिकॉर्ड |
| **Content** | पृष्ठ, खंड, तत्व, स्ट्रीमिंग |
| **Giving** | दान, निधि, भुगतान प्रसंस्करण |
| **Messaging** | बातचीत, सूचनाएं, ईमेल |
| **Doing** | कार्य, योजनाएं, असाइनमेंट |

## तकनीकी स्टैक

- **Runtime:** Node.js 22.x और TypeScript के साथ (ES मॉड्यूल)
- **Framework:** Express
- **Dependency Injection:** Inversify (डेकोरेटर-आधारित रूटिंग)
- **Database:** MySQL -- प्रत्येक मॉड्यूल के लिए एक डेटाबेस, प्रत्येक का अपना कनेक्शन पूल
- **Auth:** `CustomAuthProvider` के माध्यम से JWT-आधारित प्रमाणीकरण
- **Deployment:** Serverless Framework v3 के माध्यम से AWS Lambda

## पोर्ट

| प्रोटोकॉल | पोर्ट | विवरण |
|----------|------|---------|
| HTTP | `8084` | मुख्य REST API |
| WebSocket | `8087` | रीयल-टाइम सॉकेट कनेक्शन |

## Lambda फंक्शन

जब AWS में तैनात किया जाता है, तो API छह Lambda फंक्शन के रूप में चलता है:

- **`web`** -- सभी HTTP अनुरोधों को संभालता है
- **`socket`** -- WebSocket कनेक्शन को प्रबंधित करता है
- **`timer15Min`** -- ईमेल सूचनाओं के लिए हर 30 मिनट में चलता है (नाम ऐतिहासिक है)
- **`timerMidnight`** -- दैनिक पाचन ईमेल और रखरखाव कार्यों के लिए दैनिक चलता है
- **`timerScheduledTasks`** -- देय स्वचालन और अतिदेय वर्कफ़्लो प्रसंस्करण के लिए दैनिक चलता है
- **`timerWebhooks`** -- कतारबद्ध आउटबाउंड webhooks प्रदान करने के लिए हर मिनट चलता है

## साझा लाइब्रेरीज

API दो साझा ChurchApps पैकेजों पर निर्भर करता है:

- **[`@churchapps/helpers`](../shared-libraries/helpers)** -- आधार उपयोगिताएं (DateHelper, ApiHelper, आदि)
- **[`@churchapps/apihelper`](../shared-libraries/api-helper)** -- Express सर्वर उपयोगिताएं जिनमें auth, डेटाबेस हेल्पर, और AWS एकीकरण शामिल हैं

:::info
API ES मॉड्यूल का उपयोग करता है (`"type": "module"` `package.json` में)। सुनिश्चित करें कि आपके आयात ES मॉड्यूल सिंटैक्स का उपयोग करते हैं।
:::
