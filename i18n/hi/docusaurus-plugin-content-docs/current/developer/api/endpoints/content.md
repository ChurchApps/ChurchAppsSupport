# सामग्री एंडपॉइंट

<div class="article-intro">

सामग्री मॉड्यूल वेबसाइट पृष्ठों, अनुभागों, तत्वों, ब्लॉकों, ब्लॉग पोस्ट, रीडायरेक्ट, उपदेश, प्लेलिस्ट, स्ट्रीमिंग सेवाएं, कार्यक्रम, तैयार कैलेंडर, फ़ाइलें, गैलरी, बाइबल अनुवाद और छंद लुकअप, गीत, व्यवस्था, वैश्विक शैलियां, स्टॉक फोटो, और सेटिंग्स का प्रबंधन करता है। यह API में सबसे बड़ा मॉड्यूल है और सभी ChurchApps अनुप्रयोगों में CMS, मीडिया/स्ट्रीमिंग, उपासना योजना, और बाइबल विशेषताओं को शक्ति देता है।

</div>

**आधार पथ:** /content

## पृष्ठ

आधार पथ: /content/pages

| विधि | पथ | प्रमाणीकरण | अनुमति | विवरण |
|--------|------|------|------------|-------------|
| GET | /:churchId/tree?url=&id= | सार्वजनिक | — | URL या ID द्वारा पूर्ण पृष्ठ पेड़ (अनुभाग, तत्व, ब्लॉक) लोड करें। URL द्वारा प्राप्त करते समय आंतरिक ID को हटा दें। URL-आधारित प्राप्तियां \pages.visibility\ लागू करती हैं — एक गेटेड पृष्ठ \{ restricted: true, visibility }\ वापस करता है जब तक कि (वैकल्पिक) JWT गेट को संतुष्ट नहीं करता |
| GET | /public/:churchId | सार्वजनिक | — | सार्वजनिक पृष्ठ सूचीबद्ध करें (\url\, \	itle\, \metaDescription\); केवल \isibility = everyone\ |
| GET | /:id | JWT | — | ID द्वारा पृष्ठ प्राप्त करें |
| GET | \/\ | JWT | — | चर्च के लिए सभी पृष्ठ सूचीबद्ध करें |
| POST | \/duplicate/:id\ | JWT | Content.Edit | सभी अनुभागों और तत्वों के साथ पृष्ठ की नकल करें |
| POST | \/temp/ai\ | JWT | Content.Edit | AI-जेनरेटेड पृष्ठ सहेजें (एक कॉल में पृष्ठ, अनुभाग, और तत्व) |
| POST | \/\ | JWT | Content.Edit | पृष्ठ बनाएं या अपडेट करें (बैच) |
| DELETE | \/:id\ | JWT | Content.Edit | पृष्ठ को हटाएं |

### उदाहरण: पृष्ठ पेड़ लोड करें

}

## अनुभाग

| विधि | पथ | प्रमाणीकरण | अनुमति | विवरण |
|--------|------|------|------------|-------------|
| GET | /:id | JWT | — | ID द्वारा अनुभाग प्राप्त करें |
