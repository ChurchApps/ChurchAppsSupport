# Solicitações de Adesão a Grupos

<div class="article-intro">

Quando um grupo está configurado com uma política de adesão baseada em aprovação, as pessoas podem enviar solicitações para ingressar. Os líderes do grupo e administradores revisam essas solicitações e as aprovam ou rejeitam. Isso dá à sua igreja controle sobre a associação ao grupo, facilitando que as pessoas expressem interesse em ingressar.

</div>

<div class="prereqs">
<h4>Antes de Começar</h4>

- Você precisa de permissão para gerenciar grupos, ou você precisa ser um líder do grupo específico. Consulte [Papéis e Permissões](../people/roles-permissions.md) para detalhes.
- O grupo deve ter sua política de adesão definida como **Solicitação** (aprovação necessária). Consulte [Criando Grupos](./creating-groups.md) para saber como configurar políticas de adesão.

</div>

## Compreendendo Políticas de Adesão

Os grupos podem ter três políticas de adesão diferentes:

- **Aberto** -- Qualquer um pode ingressar imediatamente sem aprovação
- **Solicitação** -- As pessoas enviam uma solicitação de adesão que requer aprovação
- **Fechado** -- Ninguém pode solicitar adesão (os membros devem ser adicionados manualmente)

Quando um grupo usa a política **Solicitação**, todas as tentativas de adesão passam pelo fluxo de aprovação descrito nesta página.

## Visualizando Solicitações Pendentes

### Para Líderes de Grupo

1. Navegue até **Grupos** na B1 Admin
2. Clique no nome do grupo
3. Clique na aba **Solicitações de Adesão**
4. Você verá todas as solicitações pendentes para este grupo

### Para Administradores

Os administradores com permissões de gerenciamento de grupos podem visualizar solicitações pendentes em todos os grupos:

1. Navegue até **Grupos** na B1 Admin
2. Procure um emblema de notificação ou indicador de solicitações pendentes
3. Clique para visualizar todas as solicitações pendentes da igreja

## Revisando uma Solicitação de Adesão

Cada solicitação de adesão mostra:

- **Nome e foto da pessoa** -- A pessoa solicitando para ingressar
- **Mensagem opcional** -- Uma mensagem pessoal explicando por que quer ingressar (se fornecida)
- **Data da solicitação** -- Quando a solicitação foi enviada

Para revisar uma solicitação:

1. Leia a mensagem da pessoa, se ela forneceu uma
2. Clique no nome da pessoa para visualizar seu perfil, se necessário
3. Decida se aprova ou rejeita

## Aprovando uma Solicitação

1. Clique em **Aprovar** na solicitação de adesão
2. A pessoa é imediatamente adicionada ao grupo como membro
3. O solicitante recebe uma notificação de que sua solicitação foi aprovada
4. A solicitação é marcada como aprovada no sistema

:::tip
Quando você aprova uma solicitação, a pessoa se torna um membro regular do grupo. Você pode promovê-la a líder de grupo mais tarde, se necessário, na página [Membros do Grupo](./group-members.md).
:::

## Rejeitando uma Solicitação

1. Clique em **Rejeitar** na solicitação de adesão
2. Opcionalmente forneça um motivo para rejeitar (até 500 caracteres)
3. Clique em **Confirmar**
4. O solicitante recebe uma notificação com seu motivo de rejeição (se fornecido)
5. A solicitação é marcada como rejeitada

:::info
Fornecer um motivo de rejeição ajuda a pessoa a entender por que sua solicitação não foi aprovada e pode encorajá-la a tentar novamente mais tarde ou explorar outros grupos.
:::

## Notificações

O sistema de solicitação de adesão envia automaticamente notificações:

- **Quando uma solicitação é enviada** -- Todos os líderes do grupo recebem uma notificação
- **Quando uma solicitação é aprovada** -- O solicitante recebe uma confirmação
- **Quando uma solicitação é rejeitada** -- O solicitante recebe uma notificação com qualquer motivo de rejeição

As notificações aparecem no centro de notificações do usuário em B1.church e no aplicativo móvel.

## Gerenciando Solicitações do Lado do Membro

As pessoas podem gerenciar suas próprias solicitações de adesão a partir de B1.church:

- Visualizar o status de suas solicitações pendentes na página de detalhes do grupo
- Cancelar uma solicitação pendente se mudarem de ideia
- Ver se sua solicitação foi aprovada ou rejeitada

## Melhores Práticas

- **Responda prontamente** -- Tente revisar solicitações dentro de 24-48 horas para que as pessoas não fiquem esperando
- **Seja claro nos motivos de rejeição** -- Ajude as pessoas a entender os próximos passos ou opções alternativas
- **Verifique perfis** -- Revise o perfil da pessoa para ver se é uma boa opção para o grupo
- **Comunique expectativas** -- Certifique-se de que sua descrição de grupo declara claramente para quem o grupo é
