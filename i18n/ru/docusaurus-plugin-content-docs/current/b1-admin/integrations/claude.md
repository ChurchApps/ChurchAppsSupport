# Claude

<div class="article-intro">

Подключите Anthropic's Claude к данным вашей церкви B1. С API-ключом и несколькими минутами настройки вы можете задать Claude вопросы, такие как "сколько новичков пришло в воскресенье?" или "напиши благодарственное письмо людям, которые дали в фонд строительства в этом месяце" — и Claude будет читать ответы прямо из записей вашей церкви с учетом ваших разрешений.

</div>

<div class="prereqs">
<h4>Перед началом</h4>

- Администратор церкви с разрешением **Edit Settings** (для создания API-ключа)
- Один из вариантов: **Claude Code** (CLI / IDE extension), **Claude Desktop** (Mac/Windows), или аккаунт **Claude Pro/Max/Team**
- Полный URL вашего B1 API — обычно `https://api.churchapps.org` для размещенных церквей или ваш самостоятельно размещенный хост Api

</div>

## Что может видеть Claude

Claude взаимодействует с B1 через встроенный в B1 API **Model Context Protocol (MCP) сервер**. Каждый запрос Claude выполняется с использованием тех же правил аутентификации, разрешений и области церкви, что и запрос из B1Admin — что означает Claude:

- Видит данные только для **вашей** церкви
- Ограничен **разрешениями и областями**, которые несет переданный вами API-ключ
- Не может получить доступ к вебхукам, конечным точкам OAuth администратора или другим путям только для операторов (они занесены в черный список)

Ключ `donations:read` позволяет Claude суммировать пожертвования, но не может записать подарок. Ключ `people:write` может добавить человека, но не может видеть пожертвования. Выберите области, соответствующие вашей работе.

## Настройка

### 1. Создайте API-ключ

1. В B1Admin перейдите в **Settings → Developer → API Keys**.
2. Нажмите **New API Key**, назовите его `Claude`, и выберите области, которые должен иметь Claude. Обычные начальные наборы:
   - **Только для чтения помощник:** `people:read`, `groups:read`, `attendance:read`, `donations:read`, `content:read`
   - **Чтение + добавление заметок / задач:** добавьте `people:write`
   - **Полный операционный помощник:** добавьте соответствующие `:write` области, которые вам нужны
3. Сохраните. Полный ключ `cak_…` показывается **один раз** — скопируйте его.

См. [API Keys](/docs/developer/api/api-keys) для справки по каждой области.

### 2. Подключите Claude

Выберите используемого вами клиента Claude:

#### Claude Code (CLI)

В терминале:

```bash
claude mcp add --transport http b1 https://api.churchapps.org/mcp \
  --header "Authorization: Bearer cak_<prefix>.<secret>"
```

Вот и все. Внутри любой Claude Code сессии введите `/mcp` для подтверждения подключения сервера `b1`, затем задавайте Claude любые вопросы о вашей церкви.

#### Claude Desktop

Отредактируйте файл конфигурации Claude Desktop:

- **macOS:** `~/Library/Application Support/Claude/claude_desktop_config.json`
- **Windows:** `%APPDATA%\Claude\claude_desktop_config.json`

Добавьте запись сервера `b1`. Новые версии Claude Desktop поддерживают HTTP MCP изначально:

```json
{
  "mcpServers": {
    "b1": {
      "url": "https://api.churchapps.org/mcp",
      "headers": {
        "Authorization": "Bearer cak_<prefix>.<secret>"
      }
    }
  }
}
```

Если ваша версия Claude Desktop поддерживает только stdio серверы, переведите через `mcp-remote`:

```json
{
  "mcpServers": {
    "b1": {
      "command": "npx",
      "args": [
        "-y", "mcp-remote",
        "https://api.churchapps.org/mcp",
        "--header", "Authorization:Bearer cak_<prefix>.<secret>"
      ]
    }
  }
}
```

Перезагрузите Claude Desktop. Значок соединителя в окне чата будет показывать `b1` с тремя инструментами (`list_endpoints`, `describe_endpoint`, `api_call`).

#### Claude.ai (веб) — Пользовательский соединитель

Функция "Add custom connector" Claude.ai требует OAuth, который сервер B1 MCP сегодня не поддерживает. Вместо этого используйте Claude Code или Claude Desktop.

### 3. Задайте Claude что-нибудь

После подключения специальный синтаксис не требуется — Claude обнаруживает доступное на лету. Примеры:

- *"Сколько людей в моей церкви и какие активные группы?"*
- *"Суммируйте пожертвования этого месяца по фондам."*
- *"Перечислите людей, которые посетили службу в 10 утра в прошлое воскресенье, но не были в группе среды в течение последних 60 дней."*
- *"Напишите приветственное письмо четырем людям, добавленным на этой неделе, обращаясь к ним по имени."*

За кулисами Claude вызовет инструменты MCP — сначала для открытия правильной конечной точки, затем для получения данных — и ответит на простом языке.

## Как это работает

B1 API предоставляет одну конечную точку MCP в `/mcp`. Claude подключается к ней, аутентифицируется с вашим ключом `cak_…` и получает доступ к трем инструментам:

| Инструмент | Что он делает |
|---|---|
| `list_endpoints` | Перечисляет REST конечные точки, которые может вызвать Claude, фильтруемые по пути. Используется для обнаружения. |
| `describe_endpoint` | Возвращает краткое резюме и пример запроса/ответа для конкретной конечной точки. |
| `api_call` | Фактически вызывает REST конечную точку как аутентифицированный пользователь. |

Это та же самая поверхность `/membership/people`, `/giving/donations`, `/attendance/visits` и т. д., которая использует ваш B1Admin — каждое правило авторизации применяется идентично.

## Безопасность и ограничения

- **Изоляция по церквям.** API-ключ разрешается в одну церковь. Claude не имеет способа видеть данные других церквей.
- **Ограничено разрешениями.** Если вы удалите разрешение у человека, который создал ключ в B1Admin, Claude потеряет его при следующем вызове — мгновенно.
- **Отзываемо.** Удалите ключ в **Settings → Developer → API Keys** и доступ Claude заканчивается немедленно.
- **Черный список.** Вебхуки провайдера, конечные точки администратора OAuth клиента и маршрут `apiEmails` только для операторов не вызываются через MCP.
- **Потолок размера ответа.** Один ответ инструмента ограничен 64 КБ, чтобы длинные списки не перегружали контекст Claude — Claude сужает запрос с фильтрами, когда это происходит.
- **Трейл аудита.** Мутации проходят через тот же журнал аудита, что и действия B1Admin; вы можете просмотреть их в **Reports → Audit Log**.

## Стоимость

ChurchApps бесплатен и имеет открытый исходный код — сервер MCP является частью API, который ваша церковь уже запускает. Anthropic взимает плату за использование Claude в соответствии с их планами. Нет платы за вызов со стороны ChurchApps.

## Устранение неполадок

**Claude сообщает "Unauthorized" или 401:** токен-носитель отсутствует, неправильно сформирован, или ключ был отозван. Проверьте заново заголовок `Authorization: Bearer cak_…` (обратите внимание на пробел и буквальный `Bearer`).

**Вызов инструмента возвращает 403:** API-ключ не имеет области для этой конечной точки. Добавьте область в **Settings → Developer → API Keys** (вам нужно создать новый ключ — области не могут быть изменены на месте) и обновите конфигурацию Claude.

**Claude не может найти конечную точку:** попросите его вызвать `list_endpoints` с фильтром, например *"используйте list_endpoints с фильтром 'donations' для поиска правого пути"*. Реестр маршрутов создается из живого API, поэтому все, что вы можете получить с помощью `curl`, есть.

**Локальная разработка:** замените `https://api.churchapps.org/mcp` на `http://localhost:8084/mcp` — та же аутентификация, те же инструменты.
