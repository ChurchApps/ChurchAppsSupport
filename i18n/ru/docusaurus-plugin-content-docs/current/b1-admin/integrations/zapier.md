# Zapier

<div class="article-intro">

Официальное приложение B1.church на Zapier позволяет реагировать на события в вашей церкви (новый человек, новое пожертвование, новый член группы, ...) и записывать данные обратно в B1. Никакого кодирования, никакой инфраструктуры — вы подключаете всё в редакторе Zapier с перетаскиванием, вставляете ключ API и включаете Zap.

</div>

<div class="prereqs">
<h4>Перед началом</h4>

- Учётная запись [Zapier](https://zapier.com) (бесплатный уровень подойдёт для нескольких Zap'ов)
- Администратор церкви с разрешением **Редактировать параметры** в B1Admin (вы создадите ключ API)
- Представление о том, что вы хотите сделать — например "когда в B1 добавляется человек, добавить его в список Mailchimp"

</div>

## Триггеры и действия

| Тип | Что | Событие B1 / конечная точка |
|---|---|---|
| **Триггер** | Новый человек | `person.created` |
| **Триггер** | Обновлённый человек | `person.updated` |
| **Триггер** | Новое пожертвование | `donation.created` |
| **Триггер** | Новый член группы | `group.member.added` |
| **Триггер** | Новое отправление формы | `form.submission.created` |
| **Действие** | Создать человека | добавляет нового человека |
| **Действие** | Добавить пожертвование | записывает пожертвование |
| **Действие** | Добавить члена группы | добавляет человека в группу |
| **Поиск** | Найти человека | ищет человека по имени или эл. почте |

Свободно комбинируйте эти параметры с любым из 7000+ поддерживаемых приложений Zapier.

## Установка

### 1. Создайте ключ API B1

1. В B1Admin перейдите в **Параметры → Разработчик → Ключи API**.
2. Нажмите **Новый ключ API**, дайте ему имя, например "Zapier", и выберите необходимые области.
3. **Важно:** Триггеры Zapier регистрируют вебхук от вашего имени при включении Zap'а, для чего требуется область **`settings:write`**. Всегда включайте `settings:write`, если какой-либо из ваших Zap'ов использует триггер B1.
4. Также предоставьте области, необходимые для действий — например действие "Добавить пожертвование" требует `donations:write`, "Создать человека" требует `people:write`.
5. Сохраните. Полный ключ `cak_…` показывается **один раз** — скопируйте его.

### 2. Подключите Zapier к B1

1. В Zapier создайте новый Zap.
2. Когда вы впервые выберете триггер или действие B1, Zapier попросит вас **Войти в B1.church**.
3. Вставьте ключ API из шага 1 и нажмите **Да, продолжить**. Zapier проверит его для вашей церкви.

Соединение сохраняется в Zapier и используется повторно каждым Zap'ом в вашей учётной записи.

### 3. Создайте Zap

Выберите триггер, затем добавьте один или несколько шагов действия. Примеры приведены ниже.

## Типичные рецепты

### Добавляйте новых людей из B1 в Mailchimp

- **Триггер** — B1: Новый человек
- **Действие** — Mailchimp: Добавить/обновить подписчика. Сопоставьте `name__first`, `name__last`, `contactInfo__email` из B1 с полями Имя / Фамилия / Почта в Mailchimp.

### Отправляйте пожертвования в канал Slack с более богатой карточкой, чем встроенный коннектор

- **Триггер** — B1: Новое пожертвование
- **Действие** — Slack: Отправить сообщение в канал. Создайте любой макет — кнопки, вложения и т. д. — который встроенный [коннектор Slack](./slack-discord) не может.

### Добавляйте новых членов группы в Google Group

- **Триггер** — B1: Новый член группы (отфильтрованный по определённому `groupId`)
- **Действие** — Фильтр от Zapier: продолжить только если группа B1 — это та, которая вам нужна
- **Действие** — B1: Найти человека (используйте `personId` из триггера для получения эл. почты)
- **Действие** — Google Groups: Добавить члена

### Пересылайте отправления формы в отслеживатель проектов

- **Триггер** — B1: Новое отправление формы
- **Действие** — Notion / Linear / Asana / Trello: Создать страницу / задачу / проблему

## Как работают триггеры под капотом

Триггеры — это **REST-крючки**, а не опрос — Zapier не проверяет B1 каждые 15 минут. Когда вы включаете Zap, Zapier просит B1 зарегистрировать вебхук, указывающий на приватный URL Zapier; когда событие срабатывает, B1 отправляет конверт в Zapier и ваш Zap запускается **в течение секунд**. Выключите Zap и Zapier просит B1 удалить вебхук — нет потерянных подписок.

Это означает, что триггер срабатывает только для событий, которые происходят **после** включения Zap'а. Нет возврата — включение Zap'а не воспроизводит вчерашние пожертвования.

## Ограничения и примечания

- **Несколько Zap'ов с одним триггером** — каждый регистрирует свой собственный вебхук B1 — конфликтов нет, но стоит знать, если вы проверяете **Параметры → Разработчик → Вебхуки** и думаете, почему там три одинаковых строки "Zapier — donation.created".
- **Тестовые данные при создании Zap'а** — когда вы создаёте Zap, Zapier просит примеры данных для сопоставления полей. Он извлечёт самое последнее совпадающее событие из B1, если оно есть; иначе использует синтетический пример из определения приложения.
- **Ошибки действия отображаются как ошибки Zap'а** в истории задач Zapier. Частая причина: ключ API без правильной области (например действие "Добавить пожертвование" требует `donations:write`). Переиздайте ключ с правильными областями и повторно подключитесь в Zapier.
- **Квоты исходящих вызовов API** — каждый вызов API B1 из действия считается в сторону вашей квоты задач Zapier, а не в сторону B1.

## Устранение неполадок

- **"Ошибка аутентификации"** при подключении — ключ API неправильный, отозван или не имеет необходимых областей для Zap'а. Переиздайте его в B1Admin с хотя бы `settings:write` плюс все области ресурсов, к которым подключается Zap, затем обновите соединение.
- **Триггер никогда не срабатывает** — проверьте, что вебхук действительно был зарегистрирован: в B1Admin **Параметры → Разработчик → Вебхуки** теперь должна отображаться строка с названием "Zapier — &lt;событие&gt;". Если её там нет, ключ API, вероятно, не имел `settings:write` при включении Zap'а. Исправьте ключ, выключите Zap и включите его снова.
- **Триггер срабатывает дважды** — Zapier иногда переделает доставку, если его подтверждение было потеряно. Используйте шаг "Фильтр от Zapier" с уникальным id (например id человека) если вам нужна строгая дедупликация.
