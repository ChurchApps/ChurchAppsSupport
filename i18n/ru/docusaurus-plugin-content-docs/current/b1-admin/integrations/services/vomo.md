# VOMO

<div class="article-intro">

VOMO это платформа волонтерского взаимодействия — люди регистрируются на проекты, заходят в киоск и накапливают часы. Если вы используете VOMO для расписания волонтеров, но B1 для записей людей, Zapier может синхронизировать членство и check-ins между ними, чтобы ни одна сторона не плыла.

</div>

<div class="prereqs">
<h4>Перед началом</h4>

- Аккаунт [VOMO](https://vomo.org) на плане, который предоставляет Zapier (проверьте с поддержкой VOMO, если не уверены)
- Аккаунт [Zapier](https://zapier.com)
- Пользователь B1Admin с разрешением **Edit Settings**

</div>

## Что вы можете подключить

Приложение Zapier VOMO предоставляет четыре мгновенных триггера и четыре действия. Рецепты, которые большинство церквей хотят:

| Направление | Триггер | Действие |
|---|---|---|
| VOMO → B1 | VOMO Membership (создан) | B1: Find Person → Create Person (если новый) |
| VOMO → B1 | VOMO Kiosk check-in | B1: Add Group Member к группе "Currently Serving", или запись как посещаемость |
| B1 → VOMO | B1 `person.created` | VOMO: Find Organizer (по электронной почте); иначе пользовательский шаг |
| Оба | VOMO Participation (регистрации) | B1: Add Group Member к группе специфичной для проекта |

Действия VOMO ограничены **проектированием проектов** и **поиском** существующих организаторов/проектов — нет действия "добавить этого человека к проекту VOMO" сегодня. Интересное подключение это в основном VOMO → B1.

## Настройка

### 1. Создайте B1 API-ключ

**Settings → Developer → API Keys → New API Key**. Области:

- `people:read`, `people:write` — для поиска и создания волонтеров как людей B1
- `groups:write` — для добавления их в группы служащих команд
- (Опционально) `attendance:write`, если вы обрабатываете проверки киоска как посещаемость

### 2. Создайте Zap синхронизации членства

1. **Trigger** — VOMO: Membership (event = `created`).
2. **Action** — B1.church: Find Person, согласованный по электронной почте.
3. **Filter / Path** — fork на найдено vs. не найдено:
   - Not found → B1.church: Create Person, затем Add Group Member к соответствующей группе волонтеров.
   - Found → B1.church: Add Group Member напрямую.
4. Включите. Новые волонтеры VOMO теперь появляются в B1 с правильным членством в группе.

### 3. (Опционально) Создайте Zap check-in киоска

1. **Trigger** — VOMO: Kiosk
2. **Action** — B1.church: Find Person (по электронной почте)
3. **Action** — ваш выбор:
   - *If treating as attendance* — Webhooks by Zapier POST к конечной точке B1's `/attendance/visits` (приложение B1 Zapier не имеет еще действия первого класса *Record Attendance*). B1 [API key](/docs/developer/api/api-keys) идет в заголовок `Authorization: Bearer cak_…`.
   - *If treating as group membership* — B1.church: Add Group Member с группой "Currently Serving (Today)", и второй Zap позже в день удаляет их через запланированную очистку.

## Обычные рецепты

### Синхронизация групп per-project

- **Trigger** — VOMO: Participation (создан)
- **Action** — Filter by Zapier на project id, затем
- **Action** — B1.church: Add Group Member к группе B1, чье имя зеркалирует проект VOMO.

Когда проект VOMO заканчивается, вручную очистите группу B1 (или объедините это с триггером `group.member.removed`, который их удаляет).

### Отправьте "спасибо за регистрацию" текст через SMS

Объедините VOMO Participation → Clearstream Send Text или Text In Church Send Message в том же Zap. Оба имеют действия первого класса Zapier — см. [Clearstream](./clearstream) и [Text In Church](./text-in-church).

### Обнаружьте отсев

Запустите ежедневный триггер Zapier *Schedule*, который вызывает Find Organizer в VOMO для списка людей B1, которые присоединились к служащей команде этот месяц — если VOMO возвращает "not found", они не активировали VOMO и требуют толчка.

## Ограничения и примечания

- **Email это ключ присоединения.** Полезные нагрузки VOMO предоставляют пользовательскую электронную почту, но не id человека B1. Дающие, которые используют разные электронные почты в каждой системе, создадут дубликаты.
- **Никакого действия "Add to Project" в приложении Zapier VOMO сегодня.** Если вам нужно B1 → регистрация проекта VOMO, вы бы отправили POST на REST API VOMO из *Webhooks by Zapier* шага, что это пользовательская интеграция.
- **Бесплатные / более низкие уровни VOMO могут не включать Zapier.** Подтвердите с поддержкой VOMO перед обещанием даты подключения.

## Устранение неполадок

- **Триггер никогда не срабатывает** — мгновенные триггеры VOMO требуют, чтобы API токен оставался действительным. Повторно протестируйте Zap; подключите VOMO заново, если токен был повернут.
- **B1 *Add Group Member* не удается с 422** — id группы в действии не существует. Откройте **B1Admin → Groups**, нажмите на группу, и скопируйте id из URL сегмента в шаг Zap.
- **Дублированные люди B1 из одного волонтера VOMO** — они, вероятно, зарегистрировались под другой электронной почтой, чем уже имеют в B1. Либо стандартизируйте электронные почты, либо добавьте Zapier *Path*, который также ищет по телефону перед созданием.
