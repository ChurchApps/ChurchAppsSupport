# Подключенные услуги

<div class="article-intro">

Самый быстрый способ подключить B1 к другому инструменту церковной технологии — это обычно рецепт Zapier или Make — вам не нужна новая инфраструктура B1, и третья сторона размещает рабочий процесс. Эта страница является куратированным списком услуг, которые мы подтвердили, что они подключаются сегодня, с кратким пошаговым руководством по настройке для каждой.

</div>

## С первого взгляда

| Услуга | Категория | Мост | Что вы можете подключить |
|---|---|---|---|
| [Mailchimp](./mailchimp) | Email | Zapier или Make | Синхронизируйте новых людей B1 / дающих в аудиторию Mailchimp |
| [Donorbox](./donorbox) | Пожертвования | Zapier | Отправьте пожертвования Donorbox и дающих обратно в B1 |
| [Subsplash](./subsplash) | Приложение / Пожертвования | Zapier | Извлеките пожертвования Subsplash и людей в B1 |
| [VOMO](./vomo) | Волонтерство | Zapier | Синхронизируйте регистрации волонтеров между группами B1 и проектами VOMO |
| [Clearstream](./clearstream) | SMS | Zapier | Запустите текстовое сообщение Clearstream из событий B1; поглощайте ответы как записи B1 |
| [Text In Church](./text-in-church) | SMS / Рабочие процессы | Zapier | Запустите рабочие процессы Text In Church из B1; получите данные Connect Card |
| [Mobile Message](./mobile-message) | SMS (AU) | Webhooks by Zapier или Make | Отправьте SMS из любого события B1 |
| [Checkr](./checkr) | Проверки биографии | Make | Запустите проверку биографии, когда кто-то присоединяется к служащей команде |

## Что у всех этих общего

1. **B1 сторона подключения идентична** — создайте API-ключ с правильными областями в **B1Admin → Settings → Developer → API Keys**, затем либо позвольте мосту регистрировать вебхук для триггера (Zapier / Make делают это автоматически, требуется `settings:write`), либо вызывайте REST конечные точки B1 из нижестоящего действия.
2. **Мост вам счета, не нам.** ChurchApps бесплатен; Zapier и Make оба имеют бесплатные уровни, которые охватывают несколько рецептов.
3. **Вы можете протестировать подключение без передачи в эфир.** Оба моста имеют кнопку "Test step", которая срабатывает триггер один раз против B1 и показывает вам форму данных перед включением рецепта.

## Выбор моста

- **Zapier** более дружественный и имеет большой каталог приложений — используйте его по умолчанию, если стоимость не является проблемой.
- **Make** дешевле в масштабе и имеет более гибкую логику маршрутизации/фильтрации — используйте его, когда один рабочий процесс имеет fan-out, условные условия или много шагов.
- **Webhooks by Zapier** (используется для документа Mobile Message) это общий адаптер, который позволяет вам отправлять POST на любую конечную точку HTTP из Zap, когда целевой не является приложением Zapier первого класса.

Если вы хотите что-то не из этого списка, [REST API](/docs/developer/api) и [вебхуки](/docs/developer/api/webhooks) B1 открыты — вы можете создать один мост с [`@churchapps/integration-sdk`](https://www.npmjs.com/package/@churchapps/integration-sdk) за несколько часов.

## Чего нет здесь (и почему)

Несколько популярных инструментов церковной технологии — Tithe.ly, Pushpay, Vanco, PastorsLine, Gloo, Notebird, KidCheck, MinistrySafe — не имеют опубликованного приложения Zapier или модуля Make сегодня. У них есть свои API, но подключение их к B1 это пользовательская работа кода, а не рецепт, поэтому они не подходят в этот список. Если поставщик добавит приложение Zapier или модуль Make, мы добавим документ.

Мы также намеренно пропустили инструменты, специфичные для Planning Center-Services (музыка, представление), конкурирующие продукты ChMS, консультантов по миграции и инструменты, которые только очищают данные, специфичные PCO — ни один из них не имеет полезного пути подключения в B1.
