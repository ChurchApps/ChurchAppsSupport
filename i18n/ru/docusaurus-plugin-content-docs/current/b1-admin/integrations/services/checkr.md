# Checkr

<div class="article-intro">

[Checkr](https://checkr.com) проводит проверку фона для персонала и волонтеров -- практически универсальная необходимость для любой церкви, ведущей программу для детей или молодежи. B1 **не имеет встроенной функции проверки истории** -- заказ проверок, отслеживание результатов и контроль соответствия все находятся в Checkr; рецепт ниже только подключает события B1 к нему. Checkr не имеет приложения Zapier, но [интеграция Checkr на Make.com](https://www.make.com/en/integrations/checkr) проверена и предоставляет действия, которые вам нужны, чтобы запустить проверку из события B1.

</div>

## Что вы можете подключить

Приложение Checkr на Make предоставляет 1 триггер и 6 действий:

| Направление | Триггер B1 / Make | Действие |
|---|---|---|
| B1 → Checkr | B1 `group.member.added` (отфильтровано по группе волонтеров) | Checkr: Create Candidate → Create Background Check Invitation |
| Checkr → B1 | Checkr webhook (приглашение / событие отчета) | B1: Update the person's record (например, тег "Checkr cleared") |

Действия Checkr на Make: Create Candidate, Create Background Check Invitation, Get Candidate, Get Report, Get Report's ETA, Get an Invitation. Плюс 4 модуля поиска.

## Установка

### 1. Создайте ключ API B1

**Settings → Developer → API Keys → New API Key**:

- `settings:write` -- для триггера webhook
- `people:read` -- для поиска имени/электронной почты человека при запуске проверки
- (Необязательно) `people:write` если вы хотите записать статус отчета как пользовательское поле или тег

### 2. Создайте сценарий "запустить проверку при регистрации волонтера" на Make

1. **Триггер** -- B1.church: Watch Events (`group.member.added`).
2. **Фильтр** -- продолжайте только если `data.groupId` совпадает с вашей группой "Волонтеры для детей" (или эквивалент).
3. **Действие** -- B1.church: Find Person (по `data.personId`) для получения электронной почты + имени/фамилии.
4. **Действие** -- Checkr: Create Candidate. Сопоставьте имя/фамилию/электронную почту из шага 3.
5. **Действие** -- Checkr: Create Background Check Invitation. Сопоставьте новый идентификатор кандидата из шага 4 с полем *candidate_id*. Выберите пакет скрининга (например, `tasker_standard` или то, что предоставляет ваш аккаунт).
6. (Необязательно) **Действие** -- Slack: уведомьте вашего координатора по безопасности служения о том, что проверка инициирована.

Включите сценарий. Новые волонтеры в целевой группе получают автоматическое приглашение Checkr по электронной почте; они завершают это на своем телефоне или ноутбуке; Checkr проводит экран.

### 3. (Необязательно) Получите отчет обратно

1. **Триггер** -- Checkr: Watch Events (webhook). Make регистрирует webhook Checkr при активации.
2. **Фильтр** -- продолжайте только если `event_type = report.completed`.
3. **Действие** -- Checkr: Get Report (используйте идентификатор отчета из webhook).
4. **Действие** -- B1.church: Find Person (по электронной почте кандидата).
5. **Действие** -- Условный Slack / Email: уведомьте координатора со статусом `clear` / `consider` / `suspended`.

Примечание: B1 не имеет встроенного поля "статус проверки фона" сегодня. Практические опции: (a) разместите результат в приватном канале Slack для проверки, (b) запишите его в Google Sheet для аудита, или (c) добавьте человека в группу B1 "Одобренные волонтеры" на `clear`.

## Общие рецепты

### Переполучение волонтеров каждые 2 года

Свяжите вышеупомянутое с триггером расписания Make:

- **Триггер** -- Make: Schedule (ежемесячно)
- **Действие** -- B1.church: List Group Members для "Одобренные волонтеры"
- **Действие** -- Фильтр по Make: дата очистки старше 22 месяцев
- **Действие** -- Checkr: Create Background Check Invitation (то же, что в начальном потоке)

### Блокировка доступа на этап 1 до завершения проверки

Если ваша церковь использует членство в группе B1 для управления доступом (например, только члены группы "Одобренные" появляются в расписаниях служения), держите новых волонтеров в группе ожидания, пока событие `report.completed` Checkr их не перекинет.

## Ограничения и примечания

- **Checkr только для США** для большинства пакетов скрининга. Церквям в Австралии, Великобритании и Канаде потребуется альтернатива.
- **Цены** за проверку -- каждая Create Invitation на Make стоит реальную проверку. Сначала протестируйте в песочнице Checkr / staging аккаунт (приложение Checkr на Make соблюдает учетные данные, которые вы передаете в подключении, поэтому обмен учетных данных переключает песочницу/live).
- **Доступ к API Checkr управляется планом.** Меньшие аккаунты Checkr могут находиться на уровне только UI; свяжитесь с Checkr для включения API.

## Устранение неполадок

- **Create Candidate выходит с ошибкой `403`** -- токен API Checkr только для чтения или не имеет правильных разрешений аккаунта. Переиздайте его из панели управления Checkr с областью записи.
- **Приглашение никогда не поступает** -- проверьте электронную почту кандидата на шаге 3; B1 может иметь пустое поле электронной почты для этого человека. Добавьте фильтр, требующий электронную почту перед шагом Checkr.
- **Триггер webhook не срабатывает** -- регистрация webhook Checkr иногда молча выходит из строя, если ваш аккаунт Make не находится на платном уровне, который поддерживает исходящие webhook. Проверьте на странице *Webhooks* панели управления Checkr, что URL Make указан.
