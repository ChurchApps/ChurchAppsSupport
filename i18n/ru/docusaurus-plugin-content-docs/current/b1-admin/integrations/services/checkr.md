# Checkr

<div class="article-intro">

[Checkr](https://checkr.com) проводит проверки биографии для персонала и волонтеров — почти универсальная необходимость для любой церкви, управляющей детской или молодежной программой. У Checkr нет приложения Zapier, но [интеграция Checkr на Make.com](https://www.make.com/en/integrations/checkr) проверена и предоставляет действия, необходимые для запуска проверки из события B1.

</div>

<div class="prereqs">
<h4>Перед началом</h4>

- Аккаунт [Checkr](https://checkr.com) с доступом API и по крайней мере одним настроенным пакетом скрининга
- Аккаунт [Make](https://www.make.com)
- Пользователь B1Admin с разрешением **Edit Settings**

</div>

## Что вы можете подключить

Приложение Checkr Make предоставляет 1 триггер и 6 действий:

| Направление | Триггер B1 / Make | Действие |
|---|---|---|
| B1 → Checkr | B1 `group.member.added` (отфильтровано в группу волонтеров) | Checkr: Create Candidate → Create Background Check Invitation |
| Checkr → B1 | Вебхук Checkr (приглашение / событие отчета) | B1: Update the person's record (например tag "Checkr cleared") |

Действия Checkr Make: Create Candidate, Create Background Check Invitation, Get Candidate, Get Report, Get Report's ETA, Get an Invitation. Плюс 4 модуля поиска.

## Настройка

### 1. Создайте B1 API-ключ

**Settings → Developer → API Keys → New API Key**:

- `settings:write` — для триггера вебхука
- `people:read` — для поиска имени/электронной почты человека при запуске проверки
- (Опционально) `people:write`, если вы хотите написать статус отчета обратно как пользовательское поле или tag

### 2. Создайте сценарий "запустить проверку на регистрацию волонтера" в Make

1. **Trigger** — B1.church: Watch Events (`group.member.added`).
2. **Filter** — продолжайте только если `data.groupId` совпадает с вашей группой "Children's Volunteers" (или эквивалент).
3. **Action** — B1.church: Find Person (по `data.personId`) для получения электронной почты + имени/фамилии.
4. **Action** — Checkr: Create Candidate. Отобразите имя/фамилию/электронную почту из шага 3.
5. **Action** — Checkr: Create Background Check Invitation. Отобразите новый id кандидата из шага 4 в поле *candidate_id*. Выберите пакет скрининга (например `tasker_standard` или то, что предоставляет ваш аккаунт).
6. (Опционально) **Action** — Slack: уведомите вашего координатора безопасного министерства, что проверка была инициирована.

Включите сценарий. Новые волонтеры в целевую группу получают автоматическое приглашение Checkr по электронной почте; они завершают это на телефоне или ноутбуке; Checkr запускает экран.

### 3. (Опционально) Получите отчет обратно

1. **Trigger** — Checkr: Watch Events (вебхук). Make регистрирует вебхук Checkr при активации.
2. **Filter** — продолжайте только если `event_type = report.completed`.
3. **Action** — Checkr: Get Report (используйте id отчета из вебхука).
4. **Action** — B1.church: Find Person (по адресу электронной почты кандидата).
5. **Action** — Условный Slack / Email: уведомите координатора с `clear` / `consider` / `suspended` статусом.

Примечание: B1 не имеет встроенного поля "background-check status" сегодня. Практичные варианты это (a) отправить результат в приватный канал Slack для обзора, (b) написать его в Google Sheet для аудита, или (c) добавить человека в группу B1 "Cleared volunteers" на `clear`.

## Обычные рецепты

### Повторно проверяйте волонтеров каждые 2 года

Объедините вышеуказанное с триггером расписания Make:

- **Trigger** — Make: Schedule (ежемесячно)
- **Action** — B1.church: List Group Members для "Cleared volunteers"
- **Action** — Filter by Make: дата очищена старше 22 месяцев
- **Action** — Checkr: Create Background Check Invitation (как начальный поток)

### Заблокируйте доступ на этап 1 до завершения проверки

Если ваша церковь использует членство в группе B1 для доступа к воротам (например, только члены группы "Cleared" появляются в расписании служения), держите новых волонтеров в группе ожидания до тех пор, пока событие Checkr `report.completed` их не поменяет.

## Ограничения и примечания

- **Checkr только для США** для большинства пакетов скрининга. Австралийские, британские и канадские церкви нуждаются в альтернативе.
- **Стоимость** за проверку — каждое Create Invitation в Make сжигает реальную проверку. Сначала протестируйте в песочнице Checkr / staging аккаунте (приложение Checkr Make уважает учетные данные, которые вы передаете в соединение, поэтому замена учетных данных переключает sandbox/live).
- **Доступ Checkr API зависит от плана.** Меньшие аккаунты Checkr могут быть на уровне только UI; свяжитесь с Checkr, чтобы включить API.

## Устранение неполадок

- **Create Candidate не удается с `403`** — токен API Checkr только для чтения или не имеет правильных разрешений аккаунта. Пересоздайте его из панели управления Checkr с областью записи.
- **Приглашение никогда не прибывает** — проверьте электронную почту кандидата на шаге 3; B1 может иметь пустое поле электронной почты для этого человека. Добавьте фильтр, необходимый для электронной почты перед шагом Checkr.
- **Триггер вебхука не срабатывает** — регистрация вебхука Checkr иногда молчаливо не удается, если ваш аккаунт Make не находится на платном уровне, поддерживающем исходящие вебхуки. Проверьте на странице *Webhooks* панели управления Checkr, что URL Make указан.
