# Donorbox

<div class="article-intro">

Если ваша церковь принимает пожертвования через Donorbox, но отслеживает людей и выписки в B1, вы можете заставить мгновенные триггеры Zapier Donorbox создавать совпадающие записи пожертвований внутри B1 — и создавать дающего как человека B1, если они еще не существуют. Никакого ручного согласования, никакого ежемесячного экспорта.

</div>

<div class="prereqs">
<h4>Перед началом</h4>

- Аккаунт [Donorbox](https://donorbox.org) с по крайней мере одной кампанией
- Аккаунт [Zapier](https://zapier.com)
- Пользователь B1Admin с разрешением **Edit Settings**

</div>

## Что вы можете подключить

| Направление | Триггер Donorbox | Действие B1 |
|---|---|---|
| Donorbox → B1 | New or Updated Donation (мгновенный) | Find Person → Add Donation |
| Donorbox → B1 | New or Updated Donor | Create Person |
| Donorbox → B1 | New or Updated Plan (повторяющийся) | Find Person → Add Donation (используйте Plan id как заметку) |

Donorbox публикует свои триггеры как **мгновенный** — они срабатывают в течение секунд реального пожертвования. Нет задержки опроса.

## Настройка

### 1. Создайте B1 API-ключ

В B1Admin: **Settings → Developer → API Keys → New API Key**. Области:

- `people:read` — для поиска дающего по электронной почте
- `people:write` — для создания его, если он новый
- `donations:write` — для записи подарка

Триггеры в этом направлении это триггеры Donorbox, не B1's, поэтому вам не нужен `settings:write` здесь.

### 2. Создайте "запись пожертвования" Zap

1. **Trigger** — Donorbox: New Donation. Подключитесь с API-ключом Donorbox (в Donorbox: *Account → Profile → API Settings*).
2. **Action** — B1.church: Find Person. Отобразите электронную почту дающего из триггера в поле *Email* поиска.
3. **Action** — Filter by Zapier (опционально): продолжайте только если дающий не был найден, затем…
4. **Action** — B1.church: Create Person. Отобразите имя/фамилию/электронную почту, чтобы дающий приземлился как член, а не просто запись подарка.
5. **Action** — B1.church: Add Donation. Отобразите:
   - Amount → `data.amount`
   - Donation Date → дата пожертвования триггера
   - Fund → выберите фонд B1, который зеркалирует кампанию Donorbox (Zapier позволяет вам переключать фонды на основе шага фильтра или форматера)
   - Method → "Online"
   - Notes → id транзакции Donorbox (удобно при согласовании)

Включите Zap. Следующее живое пожертвование через Donorbox попадает в **B1Admin → Donations** автоматически.

## Обычные рецепты

### Один Zap на фонд

Если вы запускаете несколько кампаний Donorbox, которые отображаются на отдельные фонды B1, чистый макет это один Zap на кампанию с фильтром Donorbox *campaign* вверху — таким образом отображение фонда жестко закодировано, и вы пропускаете шаг поиска.

### Обрабатывайте обновленные пожертвования как исправления

Donorbox's *New or Updated Donation* срабатывает на правки тоже. Используйте Zapier *Path* шаг на `event_type` для fork: "new" → Add Donation, "updated" → Find Donation + Update (примечание: приложение B1 Zapier сейчас не имеет действия Update Donation — пока логируйте "updated" события в канал Slack для ручного обзора).

### Синхронизируйте изменения повторяющегося плана в канал Slack

- **Trigger** — Donorbox: New or Updated Plan
- **Action** — Slack: Send Message в `#donations` (например "Plan changed — Sarah's monthly gift is now $200")

## Ограничения и примечания

- **Отобразите дающих по электронной почте.** Donorbox не делит id человека B1; единственный долговечный ключ присоединения это электронная почта. Дающие, которые дают под другой электронной почтой, создадут нового человека B1 — ваша ежемесячная согласование должна смотреть на эти.
- **Возвраты не отдельно открыты** — Donorbox испускает обновление статуса на том же пожертвовании. Приложение B1 Zapier сейчас не имеет действия *Update Donation*; безопасный паттерн сегодня это логировать события возврата вне пределов диапазона и регулировать пожертвование вручную.
- **Сначала протестируйте в песочнице Donorbox**, чтобы избежать создания поддельных подарков в производстве B1. Donorbox предоставляет учетные данные тестового режима, отдельные от живых.

## Устранение неполадок

- **"Person not found" предупреждение каждый запуск** — это в порядке, если вы заказали шаги, чтобы *Create Person* был выполнен в ветви not-found. Если шаг Create Person никогда не работает либо, дважды проверьте API-ключ имеет `people:write`.
- **Сумма пожертвования выглядит в 100 раз слишком большой или маленькой** — Donorbox отправляет центы в некоторых вариантах полезной нагрузки и доллары в других. Используйте *Formatter by Zapier — Numbers* шаг для деления на 100, если нужно.
- **Дублированные пожертвования из одного подарка** — Donorbox срабатывает и *New Donation*, и *Updated Donation*. Либо фильтруйте на `event_type = "donation.succeeded"`, либо создайте два Zaps с неперекрывающимися фильтрами.
