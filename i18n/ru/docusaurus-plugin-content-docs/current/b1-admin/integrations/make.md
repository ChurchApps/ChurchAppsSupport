# Make

<div class="article-intro">

[Make](https://www.make.com) (ранее Integromat) — платформа визуальной автоматизации рабочих процессов — похожа по духу на Zapier, с более гибкой логикой и более дешевым счетом в масштабе. Официальное приложение B1.church Make позволяет вам создавать "сценарии", которые мгновенно реагируют на события B1 и записывают данные обратно в B1.

</div>

<div class="prereqs">
<h4>Перед началом</h4>

- Аккаунт [Make](https://www.make.com) (бесплатный уровень охватывает небольшие рабочие процессы)
- Администратор церкви с разрешением **Edit Settings** в B1Admin
- Приблизительное представление о сценарии, который вы хотите создать

</div>

## Модули

| Тип | Что | Событие B1 / конечная точка |
|---|---|---|
| **Мгновенный триггер** | Watch Events | любое подписанное событие B1 (`person.created`, `donation.created`, …) |
| **Action** | Create Person | добавляет нового человека |
| **Action** | Add Donation | записывает пожертвование |
| **Action** | Add Group Member | добавляет человека в группу |
| **Search** | Search People | находит людей по имени или электронной почте |

Мгновенный триггер позволяет вам выбрать, какое событие слушать — один модуль триггера на сценарий, настроенный для каждого события.

## Настройка

### 1. Создайте B1 API-ключ

1. В B1Admin перейдите в **Settings → Developer → API Keys**.
2. Нажмите **New API Key**, назовите его "Make", и предоставьте нужные области.
3. **Включите `settings:write`**, если какой-либо из ваших сценариев использует мгновенный триггер — Make регистрирует вебхук от вашего имени, когда сценарий включен.
4. Также предоставьте области, необходимые модулям действий (например `donations:write` для модуля Add Donation).
5. Сохраните и скопируйте ключ `cak_…`.

### 2. Установите соединение

1. В Make создайте новый сценарий и поместите модуль триггера **B1.church** на холст.
2. При появлении запроса **Create a connection**. Вставьте API-ключ в поле *API Key* и оставьте *API Base URL* как `https://api.churchapps.org` (если вы не тестируете против staging).
3. Нажмите **Save** — Make проверяет ключ, прочитав профиль вашей церкви.

Соединение сохраняется в вашем аккаунте Make и повторно используется в сценариях.

### 3. Настройте триггер

1. Откройте параметры модуля **Watch Events**.
2. Выберите событие, которое вы хотите — например `donation.created`.
3. Сохраните. Make генерирует уникальный URL вебхука и сохраняет его внутри.

### 4. Добавьте нижестоящие модули

Поместите любой из сотен модулей приложений Make на холст — Mailchimp, Google Sheets, Slack, HubSpot, вашу собственную конечную точку HTTP и т. д. Отобразите выходные данные триггера (`event`, `churchId`, `data.id`, `data.amount`, …) в их входные поля. Модули Make flatten / iterator / router / aggregator позволяют вам создавать ветвящиеся и параллельные потоки, которые было бы сложно в Zapier.

### 5. Включите сценарий

Переключите **Active** в заголовке сценария. Make вызывает B1's `POST /membership/webhooks` для регистрации URL. С этого момента каждое совпадающее событие B1 течет через сценарий в реальном времени.

Отключение сценария вызывает `DELETE /membership/webhooks/{id}`, поэтому нет сиротских подписок.

## Обычные рецепты

### Синхронизируйте пожертвования в Google Sheet для проверки финансов

- **Trigger** — B1: Watch Events (`donation.created`)
- **Action** — Google Sheets: Add a Row. Отобразите `data.donationDate`, `data.amount`, `data.personId`, `data.method`, `data.batchId` в столбцы таблицы.

### Условное уведомление Slack по сумме пожертвования

- **Trigger** — B1: Watch Events (`donation.created`)
- **Router**:
  - Ветвь A — Filter: `data.amount >= 1000` → Slack: отправьте в `#major-gifts`
  - Ветвь B — fallthrough → Slack: отправьте в `#donations`

### Новый человек → CRM + приветственное письмо + Slack

- **Trigger** — B1: Watch Events (`person.created`)
- **Action** — HubSpot: Create Contact
- **Action** — Mailgun: Send Welcome Email
- **Action** — Slack: Notify `#new-people` (параллельно — используйте маршрутизатор Make)

## Как работает мгновенный триггер

Мгновенный триггер основан на вебхуках, не на опросе — при активации Make вызывает `POST /membership/webhooks` со своим сгенерированным URL и выбранным событием. Когда событие происходит в B1, B1 отправляет конверт на URL Make по POST и ваш сценарий работает в течение секунд. Отключение сценария удаляет вебхук.

Триггер срабатывает только для событий, которые происходят **пока сценарий активен**. Нет переполнения.

## Ограничения и примечания

- **Одно событие на модуль Watch Events.** Чтобы слушать несколько событий в одном сценарии, поместите несколько модулей триггера в отдельные сценарии (или используйте один модуль с объединенным списком событий — см. ниже).
- **Проверка подписи не открывается** — Make не проходит `X-B1-Signature` в сценарий; граница доверия — это неугадываемый URL вебхука per-scenario Make. Это нормальная практика Make. Если вам нужны явные проверки подписи, создайте пользовательскую интеграцию с [SDK](/docs/developer/api/webhooks#sdk-support) вместо этого.
- **Счет операций** — каждый вызов API из модуля действия учитывается в квоте операций Make, а не в чем-либо на стороне B1.

## Устранение неполадок

- **Тест соединения не удается** — чаще всего опечатка в API-ключе. Пересоздайте его из B1Admin (полный ключ показывается только один раз; если вы его потеряли, создайте новый ключ).
- **Модуль триггера не активируется** — проверьте **Settings → Developer → Webhooks** в B1Admin. Если вы не видите "Make — &lt;event&gt;" после активации сценария, ключу не хватает `settings:write`. Обновите ключ и переактивируйте.
- **Action возвращает `403 Forbidden`** — API-ключ не имеет области для этой конечной точки. Например, Add Donation требует `donations:write`. Обновите ключ в B1Admin и повторно протестируйте.

## Настройка приложения

Приложение B1.church Make имеет открытый исходный код — определения JSON находятся в репозитории `B1Integrations/Make/`. Если вам нужен модуль, который не существует (например, новое действие для конечной точки, которую мы не охватили), откройте проблему или PR там.
