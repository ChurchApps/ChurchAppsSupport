# Проверка плана и уведомления волонтёров

<div class="article-intro">

B1 Admin автоматически проверяет ваши планы на наличие проблем до воскресенья — незаполненные позиции, конфликты расписания и волонтёры, которые заблокировали эту дату. Когда всё в порядке, вы можете уведомить всю команду одним нажатием.

</div>

<div class="prereqs">
<h4>Перед началом работы</h4>

- Создайте [план служения](./plans.md) и назначьте волонтёров на позиции
- Добавьте [время богослужений](./plans.md) в план, чтобы система обнаружения конфликтов могла проверить пересечения
- Убедитесь, что у волонтёров установлено приложение B1 Mobile для получения push-уведомлений

</div>

## Панель проверки

Каждый план имеет панель **Проверка**, которая запускается автоматически при его формировании. Она проверяет три вещи:

### Незаполненные позиции
Если для позиции требуется больше людей, чем назначено в данный момент, панель проверки точно показывает, что ещё необходимо — например, *"Звукотехник: нужен ещё 1 человек."* Вы можете с первого взгляда увидеть, полностью ли укомплектован ваш план до наступления недели.

### Конфликты расписания
Если волонтёр назначен на две позиции, которые пересекаются по времени в рамках одного плана, панель проверки отмечает конфликт — например, *"Иванова Мария: конфликт времени между Лидером прославления и Регистрацией детей во время Воскресного богослужения."* Это выявляет двойные назначения до того, как они станут проблемой в воскресное утро.

### Заблокированные даты
Волонтёры могут устанавливать даты своей недоступности в B1 Mobile. Если кто-то назначен на план, который попадает на одну из его заблокированных дат, панель проверки автоматически отображает конфликт, чтобы вы могли найти замену.

### Конфликты между планами
Проверка также выполняется по всем вашим планам одновременно. Если один и тот же волонтёр назначен в двух разных планах, которые пересекаются по времени — например, богослужение в 9:00 и богослужение в 10:00, оба из которых продолжаются до 10:30 — B1 Admin отметит этого человека как назначенного дважды в разных планах.

:::tip
Вам не нужно ничего делать для запуска проверки — она обновляется автоматически каждый раз, когда вы добавляете или изменяете назначение. Просто следите за панелью при формировании плана.
:::

## Уведомление волонтёров

Когда ваш план готов, вы можете уведомить всех назначенных волонтёров одновременно прямо из панели проверки.

1. Откройте план и прокрутите до панели **Проверка**
2. Если есть неуведомлённые волонтёры, вы увидите ссылку, показывающую, сколько нужно уведомить (например, *"Уведомить 8 волонтёров"*)
3. Нажмите на ссылку, чтобы отправить push-уведомления всем, кто ещё не был уведомлён
4. Волонтёры получают уведомление на телефон, информирующее их о назначении и предлагающее подтвердить его

:::info
Будут включены только те волонтёры, которые ещё не были уведомлены. Если вы позже добавите кого-то в план, ссылка появится снова, чтобы вы могли уведомить нового участника, не отправляя повторные уведомления остальной команде.
:::

:::warning
Для получения push-уведомлений у волонтёров должно быть установлено приложение B1 Mobile с включёнными уведомлениями. См. [руководство по Уведомлениям](/docs/b1-mobile/community/notifications), чтобы узнать, как волонтёры могут включить их на своём устройстве.
:::
