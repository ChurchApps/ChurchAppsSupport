# Валидация плана и уведомления добровольцев

<div class="article-intro">

B1 Admin автоматически проверяет ваши планы на предмет проблем — незаполненные позиции, конфликты расписания и добровольцы с заблокированными датами.

</div>

<div class="prereqs">
<h4>Перед началом работы</h4>

- Создайте [план служения](./plans.md) и назначьте добровольцев на позиции
- Добавьте [время служения](./plans.md) к плану
- Убедитесь, что добровольцы имеют установленное приложение B1 Mobile

</div>

## Панель валидации

Каждый план имеет **панель валидации**, которая работает автоматически. Она проверяет:

### Незаполненные позиции
Если позиция требует больше людей, чем назначено, панель валидации показывает, что еще нужно.

### Конфликты расписания
Если добровольец назначен на две перекрывающиеся позиции, панель валидации указывает на конфликт.

### Заблокированные даты
Если кто-то назначен на дату, когда они недоступны, панель валидации это обнаруживает.

### Конфликты между планами
Валидация также проверяет все ваши планы одновременно для выявления двойных бронирований.

:::tip
Вам не нужно ничего делать для запуска валидации — она обновляется автоматически.
:::

## Уведомление добровольцев

1. Откройте план и прокрутите к **панели валидации**
2. Если есть неуведомленные добровольцы, вы увидите ссылку
3. Нажмите ссылку для отправки уведомлений

:::info
Только добровольцы, которые не были уведомлены, будут включены.
:::
