# Задачи

<div class="article-intro">

Задачи позволяют вам назначать пункты действий людям или группам в вашей церкви. Является ли это разовым делом или повторяющейся ответственностью, задачи помогают вам отслеживать, что нужно сделать и кто ответственен, чтобы ничего не было упущено.

</div>

<div class="prereqs">
<h4>Перед началом</h4>

- Убедитесь, что люди или [группы](../groups/creating-groups.md), которым вы хотите назначить задачи, существуют в B1 Admin
- Перейдите в раздел **Служение**, чтобы получить доступ к странице Задачи

</div>

## Просмотр задач

Перейдите в **Служение** и откройте страницу **Задачи**. Вы увидите список открытых задач. Каждая задача отображает её название, назначенного лица и статус.

:::tip
По умолчанию отображаются только открытые задачи. Переключитесь на **Показать закрытые**, чтобы также просмотреть завершённые задачи.
:::

## Создание задачи

1. На странице Задачи нажмите **Добавить задачу**.
2. Введите **название** для задачи.
3. Щелкните по окну **Назначить** и введите имя человека или переключитесь на **Группа** и введите название группы.
4. Нажмите **Поиск** и выберите человека или группу из результатов.
5. Добавьте **примечание** с любыми деталями или инструкциями для назначенного лица.
6. Нажмите **Сохранить**.

Задача теперь появится в списке задач и будет видна назначенному лицу на его панели управления и мобильном приложении.

## Управление задачей

Щелкните на любую задачу, чтобы открыть её страницу с подробностями. Отсюда вы можете:

- **Изменить статус** — переключайтесь между **Открыто** и **Закрыто**, используя раскрывающийся список статуса.
- **Переназначить задачу** — обновите назначенного лица другому человеку или группе.
- **Добавить примечания** — включите дополнительную информацию или обновления для назначенного лица.
- **Редактировать название** — обновите название задачи по мере необходимости.

## Жизненный цикл задачи

1. Новая задача начинается со статусом **Открыто**.
2. Назначенное лицо получает уведомление и может просмотреть задачу со своей панели управления, страницы Задачи или мобильного приложения.
3. После завершения работы назначенное лицо (или администратор) меняет статус на **Закрыто**.
4. Закрытые задачи удаляются из представления по умолчанию, но по-прежнему могут быть доступны при включении **Показать закрытые**.

:::info
Вы можете автоматизировать повторяющиеся задачи, чтобы они создавались по расписанию. Подробнее см. [Автоматизация](./automations.md).
:::

:::tip
Когда вам нужно переместить много людей через одинаковый набор этапов — такой как последующее наблюдение за новыми посетителями или процесс членства — используйте [Рабочие процессы](./workflows.md). Рабочие процессы превращают задачи в карточки на доске с возможностью перетаскивания с шагами, сроками выполнения и маршрутизацией.
:::

## Следующие шаги

- Переместите людей через многошаговый процесс с помощью [Рабочих процессов](./workflows.md)
- Настройте [Автоматизацию](./automations.md) для автоматического создания повторяющихся задач
- Используйте задачи наряду с [Планами служений](./plans.md) для отслеживания пунктов действий подготовки служения
