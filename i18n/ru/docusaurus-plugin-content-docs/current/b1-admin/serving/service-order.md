# Порядок служения

<div class="article-intro">

Порядок служения определяет последовательность элементов в вашем богослужении. Используйте его, чтобы выложить всё — от открывающей песни до закрывающей молитвы, чтобы вся ваша команда знала ход служения заранее.

</div>

<div class="prereqs">
<h4>Перед началом</h4>

- [Создайте план служения](./plans.md) на дату, для которой вы хотите построить порядок служения
- При необходимости добавьте [песни](./songs.md) в свою библиотеку, чтобы вы могли связать их непосредственно с порядком служения

</div>

## Доступ к порядку служения

1. Перейдите на **Serving** (Служения) и выберите своё служение.
2. Нажмите на **plan type** (тип плана), затем откройте конкретный **plan** (план).
3. Нажмите на вкладку **Service Order** (Порядок служения) на странице с деталями плана.

## Построение вашего порядка служения

Порядок служения — это список элементов, расположенных в последовательности, в которой они будут происходить во время служения. Общие элементы включают:

- Хвалебные песни
- Молитвы
- Чтение Библии
- Объявления
- Проповедь или послание
- Сбор пожертвований
- Завершение

## Добавление элементов в порядок служения

1. На вкладке **Service Order** (Порядок служения) нажмите **Add Item** (Добавить элемент).
2. Выберите тип элемента, который вы хотите добавить (например, песню из вашей библиотеки, пользовательский элемент или заголовок).
3. Введите детали элемента, такие как название и любые примечания.
4. Нажмите **Save** (Сохранить).

:::tip
Вы можете добавлять песни непосредственно из своей библиотеки [Songs](./songs.md). Это связывает детали песни, включая текст и аранжировки, с вашим планом служения, чтобы ваша команда хвалебного пения имела всё необходимое.
:::

## Переупорядочение элементов

Перетащите элементы, чтобы переставить порядок. Последовательность, показанная на экране, — это порядок, которому будет следовать ваша команда во время служения.

## Редактирование и удаление элементов

- Нажмите на любой элемент в порядке служения, чтобы **edit** (отредактировать) его детали.
- Используйте опцию **delete** (удалить), чтобы удалить элемент из порядка.

:::info
Порядок служения характерен для каждого плана. Изменения, которые вы здесь вносите, влияют только на выбранный план и не изменят другие планы или вашу библиотеку песен.
:::

## Включение элементов в определённые время служения

Если ваш план имеет более одного времени служения (например, богослужение в 8 часов и 10 часов), вы можете выбрать, в каких служениях должен появиться каждый элемент. Это полезно, когда объявление имеет отношение только к одному служению или когда песня поётся в одном служении, но не в другом.

1. Откройте план, который имеет два или более времени служения, определённых на вкладке **Times** (Время).
2. На вкладке **Service Order** (Порядок служения) нажмите на элемент, чтобы отредактировать его.
3. В разделе **Include in Services** (Включить в служения) вы увидите флажок для каждого времени служения с меткой времени.
4. Снимите галочку с любого служения, где элемент должен быть пропущен.
5. Нажмите **Save** (Сохранить).

По умолчанию каждый элемент включается в каждое служение. Исключённые времена служения скрыты, когда вы печатаете план, отфильтрованный по этому служению, поэтому каждое служение получает чистый лист хода с только нужными элементами.

:::tip
Раздел **Include in Services** (Включить в служения) появляется только в том случае, если план имеет более одного времени служения. Если вы видите только одно служение, сначала отредактируйте вкладку **Times** (Время) плана, чтобы добавить дополнительные служения.
:::

## Обмен порядком служения

После завершения порядка служения вы можете распечатать полный план (включая порядок служения) со страницы с деталями плана. Это дает вашей команде полное описание служения.

## Следующие шаги

- Управляйте вашим репертуаром хвалебных песен на странице [Songs](./songs.md)
- Назначьте волонтёров служению на вкладку [Plans](./plans.md) Assignments (Назначения)
- Создавайте [Tasks](./tasks.md) для любых последующих элементов, связанных со служением
