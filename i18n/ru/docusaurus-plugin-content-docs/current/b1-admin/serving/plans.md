# Планы служб

<div class="article-intro">

Планы служб организуют, кто служит и когда. Каждый план связан с конкретной датой и служением, что облегчает координацию ваших команд волонтёров неделю за неделей и гарантирует, что каждое служение полностью укомплектовано персоналом.

</div>

<div class="prereqs">
<h4>Перед началом работы</h4>

- Установите ваши служения и команды в области служения
- Убедитесь, что волонтёры добавлены в ваш [справочник контактов](../people/adding-people.md) и назначены командам

</div>

## Доступ к планам

1. Перейдите на **Serving** (Служение) из главного меню.
2. Выберите **ministry tab** (вкладку служения) в верхней части страницы.
3. Нажмите на **plan type** (тип плана), чтобы увидеть список планов для этого типа.
4. Нажмите на конкретный план, чтобы открыть его.

:::info
Полный доступ администратора не требуется для управления планами. Любой, кто является членом служения, может перейти на страницу служения и создавать, редактировать и планировать планы для своего собственного служения без необходимости разрешения на редактирование планов. Редакторы с ролью Plans Edit (Редактирование планов) могут управлять планами во всех служениях.
:::

## Создание плана

1. Из представления типа плана нажмите **New Plan** (Новый план).
2. Дайте плану имя или используйте дату в качестве имени. Выберите **date** (дату) служения.
3. Если вы хотите скопировать из предыдущего плана, выберите только позиции или позиции и назначения. Если вы не хотите копировать, просто не выбирайте ничего. Вы также можете скопировать порядок служения из моего предыдущего плана.
4. Сохраните план. Теперь вы можете начать назначать членов команды и создавать [порядок служения](./service-order.md).

## Страница сведений плана

Когда вы открываете план, вы видите две вкладки:

- **Assignments** (Назначения) — управляйте, какие члены команды назначены на этот план. Вы можете добавлять людей из ваших существующих команд и видеть, кто подтвердил или всё ещё ожидает.
- **[Service Order](./service-order.md)** (Порядок служения) — создавайте порядок служения с элементами, такими как гимны поклонения, молитвы, объявления и проповедь.

## Назначение членов команды

1. Откройте план и перейдите на вкладку **Assignments** (Назначения).
2. Нажмите на **add Position** (добавить позицию) для развёртывания. Заполните информацию в форме добавления позиции. Для имени категории добавьте любую категорию, которую вам нравится.
3. Нажмите на **People Needed** (Нужны люди) и выберите волонтёров для заполнения этой позиции.
4. Добавьте членов из списка вашей команды, нажав **Add** (Добавить).
5. Назначенные члены появятся в своей команде с их статусом назначения.
6. Нажмите notify volunteers (уведомить волонтёров), чтобы уведомить их в приложении B1 или по электронной почте.

:::tip
Установите ваши команды в параметрах служения перед созданием планов. Таким образом, у вас будет готовый пул волонтёров для назначения.
:::

## Напоминания волонтёрам

B1 может автоматически напоминать волонтёрам перед служениями, на которые они назначены, поэтому вам не нужно ходить за вашей командой каждую неделю. Напоминания поступают ко **всем, кто в расписании** — как тем, кто подтвердил, так и тем, кто ещё не ответил — по электронной почте и как уведомление в приложении/push-уведомление. Каждое напоминание включает позиции волонтёра, дату служения, заметки плана и ваше пользовательское сообщение.

Время и содержание напоминаний устанавливаются для каждого **типа плана**, поэтому каждый вид служения может иметь своё собственное расписание.

1. Из области **Serving** (Служение) выберите служение, содержащее тип плана.
2. Нажмите **edit (pencil) icon** (значок редактирования (карандаш)) рядом с типом плана.
3. В разделе **Reminders** (Напоминания) установите:
   - **Reminder days before service** (Напомнить за дни до служения) — разделённый запятыми список того, на сколько дней раньше отправить, например `7,1,0`. Используйте `0`, чтобы отправить напоминание в день служения. Оставьте это поле пустым, чтобы отключить напоминания для этого типа плана.
   - **Custom reminder message** (Пользовательское сообщение напоминания) *(опционально)* — дополнительный текст, добавленный к напоминанию, например «Приходите за 30 минут раньше для репетиции».
4. Сохраните тип плана.

Новые типы планов напоминают волонтёрам **за 2 дня до** каждого служения по умолчанию, пока вы это не измените.

:::tip
Волонтёры, которые ещё не подтвердили, получают кнопки **Accept** (Принять) и **Decline** (Отклонить) прямо в письме с напоминанием, поэтому они могут ответить без входа.
:::

:::info
Каждое напоминание отправляется один раз. Планы, которые всё ещё имеют предварительный набросок (ещё не отправлены команде), не вызывают напоминания.
:::

## Связывание групп с типом плана

Ниже списка планов на странице типа плана раздел **Groups** (Группы) позволяет вам решить, какие группы могут видеть планы для этого типа плана из своего портала для членов. Это быстрый способ представить предстоящие служения нужным командам без предоставления им доступа администратора.

1. На странице типа плана прокрутите вниз до раздела **Groups** (Группы).
2. Нажмите **Add Group** (Добавить группу) и выберите группу из раскрывающегося списка.
3. В столбце **Shows** (Показывает) выберите, должны ли члены этой группы видеть планы **Past** (Прошлые), **Future** (Будущие) или **Both** (Оба) для этого типа плана.
4. Повторите, чтобы связать дополнительные группы, или нажмите значок мусорной корзины, чтобы удалить группу.

:::info
В средстве выбора появляются только группы, помеченные как **Standard** (Стандартные). Члены связанной группы автоматически видят планы этого типа плана на вкладке [Plans](/docs/b1-church/plans/) (Планы) на портале для членов B1 — ограниченной окном прошлое/будущее/оба, которое вы выбрали.
:::

## Печать планов

Вы можете распечатать план для распространения вашей команде. Откройте план, откройте вкладку порядка служения и используйте опцию **Print** (Печать), чтобы создать версию, пригодную для печати, которая включает назначения и порядок служения. Это полезно для раздачи на репетициях или размещения в общем месте.

:::info
Планы организованы по служениям. Убедитесь, что вы находитесь на правильной вкладке служения перед созданием или просмотром планов.
:::

## Следующие шаги

- Сохраните структуру плана как [Plan Template](./plan-templates.md) (Шаблон плана), чтобы вы могли применить его к будущим планам одним кликом

- Используйте [Tasks](./tasks.md) (Задачи) для назначения элементов действий членам вашей команды
