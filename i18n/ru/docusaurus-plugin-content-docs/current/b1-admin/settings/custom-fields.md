# Пользовательские поля

<div class="article-intro">

**Custom Fields** позволяют вам отслеживать вашу собственную информацию на каждой записи человека — вещи, для которых B1 не имеет встроенного поля, как дата истечения проверки биографии, размер футболки или статус класса крещения. Вы определяете поле один раз в Settings, затем заполняете значение на профиле каждого человека и ищете или создаёте списки по этому поля. Это заменяет старый workaround создания формы People просто для сохранения единственного фрагмента пользовательских данных.

</div>

<div class="prereqs">
<h4>Перед началом работы</h4>

- Вам нужно разрешение на редактирование **People** для определения полей и для заполнения значений, и доступ к области **Settings**. Любой с разрешением на просмотр People может видеть значения. См. [Roles & Permissions](./roles-permissions.md).
- Решите, что вы хотите отслеживать и какой тип лучше всего подходит (текст, число, дата, ответ да/нет или список выбора) перед началом.

</div>

## Открытие пользовательских полей

В B1 Admin перейдите в **Settings** в левой боковой панели и выберите карту **Custom Fields**. Вы также можете перейти прямо туда на **/settings/custom-fields**. Вы увидите список каждого поля, которое вы определили, показывающий его **Name** и **Field Type**. Если вы ещё не создали ни одно, панель читает *"No custom fields have been added yet."*

## Добавление поля

1. Нажмите **Add Field**.
2. В редакторе, который открывается справа, введите **Name** — это метка, которую сотрудники будут видеть на профилях человека и при поиске (например, *Background check expires*).
3. Выберите **Field Type**:
   - **Textbox** — свободный текст.
   - **Whole Number** — числа без десятичных.
   - **Decimal** — числа, которые могут включать десятичные.
   - **Date** — календарная дата.
   - **Yes/No** — простой ответ да или нет.
   - **Multiple Choice** — список выбора. Когда вы выбираете этот тип, **редактор выборов** появляется так что вы можете добавить каждую опцию люди могут выбрать из.
4. Нажмите **Save**.

Поле теперь доступно на профиле каждого человека.

:::info
Типы полей — это тот же набор, используемый для [вопросов формы](../forms/creating-forms.md), так что значения ведут себя последовательно по B1.
:::

## Редактирование поля

Нажмите любую строку поля в списке для переоткрытия его в редакторе. Измените имя, тип или выборы и нажмите **Save**.

:::warning
Изменение **Field Type** поля, которое уже имеет значения (например, из Textbox в Date) может оставить ранее введённые значения в формате, который больше не совпадает с новым типом. Измените типы с осторожностью один раз сотрудники начали заполнять поле.
:::

## Удаление поля

Откройте поле для редактирования и нажмите **Delete**. Вас попросят подтвердить: *"Are you sure you wish to delete this custom field? Its stored values will also be removed."* Удаление поля постоянно удаляет его **и каждое значение, хранящееся для него** на всех людях — это не может быть отменено.

## Заполнение значений для человека

Один раз как минимум одно пользовательское поле существует, его значения находятся прямо рядом со встроенными деталями на каждой записи человека — вы просматриваете их в **Personal Details** и редактируете их на той же форме, которую вы используете для остальной информации человека. Ничего дополнительного не появляется до тех пор пока вы не определили ваше первое поле.

1. Откройте запись человека в **People**.
2. В разделе **Personal Details** нажмите кнопку **Edit** (карандаш).
3. Прокрутитесь к области **Custom Fields** в нижней части формы редактирования и заполните значение для каждого поля. Каждое поле показывает вход, который соответствует его типу — выбирателю даты для Date полей, выпадающего списка да/нет для Yes/No полей, списка выбора для Multiple Choice и т.д.
4. Нажмите **Save**. Ваши пользовательские значения полей сохраняются вместе с остальной информацией человека.

Вернёмся на профиль, любое поле, которое имеет значение теперь показывается в разделе **Personal Details** (Yes/No ответы читают как *Yes* или *No*, и Multiple Choice показывает метку опции). Поля оставленные пусто просто скрыты. Для удаления значения, отредактируйте человека, очистите поле и сохраните — пустое значение удаляется из записи, а не хранится как пусто.

:::tip
Классический use case — безопасность волонтёров: создайте поле **Date** под названием *Background check expires*, записите дату каждого волонтёра, затем создайте [Saved List](../people/lists.md), которая отмечает любого, чья дата прошла.
:::

## Поиск и создание списков на пользовательских полях

Пользовательские поля полностью доступны для поиска:

1. На странице **People** откройте [Advanced Search](../people/searching-people.md).
2. Расширьте категорию **Custom Fields**.
3. Проверьте поле, на котором вы хотите фильтровать, выберите оператор и введите значение. Предлагаемые операторы совпадают с типом поля:
   - **Textbox** — contains, equals, starts with, ends with.
   - **Whole Number / Decimal** — equals, greater than, greater than or equal, less than, less than or equal.
   - **Date** — equals, after (greater than), before (less than).
   - **Yes/No** — equals Yes or No.
   - **Multiple Choice** — equals or contains one of the choices.

Сохраните любой пользовательский поиск поля как [List](../people/lists.md). Списки — это живые запросы, так что список, построенный на *Background check expires is before today* переоценивает каждого человека каждый раз когда вы его открываете — нет ручного обслуживания.

## Что происходит при слиянии

Когда вы [объединяете две записи человека](../people/adding-people.md), пользовательские значения полей переносятся автоматически. Человек, которого вы сохраняете, держится за свои собственные значения; для любого поля где только удалённый человек имел значение, это значение копируется так что ничего не теряется.
