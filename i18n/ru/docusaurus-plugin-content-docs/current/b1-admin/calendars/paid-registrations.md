# Платные регистрации

<div class="article-intro">

Регистрация события может выходить за рамки простого подсчёта. Вы можете определить типы участников с ценами (как Взрослый и Ребёнок), предложить опциональные дополнения с их собственными ценами и количествами, создать коды скидок и собрать платёж при регистрации через существующего провайдера платежей вашей церкви. Когда событие заполняется, опциональный лист ожидания держит заинтересованных членов в очереди и продвигает их автоматически как открываются места.

</div>

<div class="prereqs">
<h4>Перед началом работы</h4>

- Сначала включите регистрацию на событии — см. [Creating Calendars](creating-calendars#enabling-event-registration)
- Для сбора платежей, ваша церковь нуждается [в сконфигурированных онлайн платежах](../donations/online-giving-setup.md) (Stripe, PayPal или Kingdom Funding). Бесплатные события не нуждаются в настройке платежей.

</div>

## Открытие параметров регистрации

1. В B1 Admin перейдите на страницу **Registrations** и откройте ваше событие (или откройте событие из его календаря).
2. Карта **Registration Settings** показывает основы — **Enable Registration**, **Capacity**, **Registration Opens/Closes**, **Tags** и **Registration Questions**.
3. Ниже основ находятся три аккордеона: **Attendee Types**, **Selections** и **Discount Codes**.

## Типы участников

Типы участников позволяют вам взимать различные цены для различных типов участников — и ограничить каждого отдельно.

1. Расширьте аккордеон **Attendee Types** и нажмите **Add Type**.
2. Введите **Name** (например, "Adult", "Child", "Student").
3. Установите **Price**. Используйте 0 для бесплатного типа.
4. Опционально установите **Capacity** только для этого типа (например, только 20 мест Child). Оставьте пусто для отсутствия ограничения для каждого типа.
5. Нажмите **Save**.

Во время регистрации каждый участник выбирает тип; распроданные типы показываются как **Sold out** и не могут быть выбраны. Реестр показывает тип каждого участника и подсчётов для каждого типа.

## Выборы

Выборы — это опциональные дополнения с ценами — футболки, планы питания, улучшения активности.

1. Расширьте аккордеон **Selections** и нажмите **Add Selection**.
2. Введите **Name**, опциональное **Description** и **Price** (0 показывается как "Free").
3. Опционально установите **Capacity** (всего доступных по всем регистрациям) и **Max Qty** (наибольшее количество, которое одна регистрация может заказать).
4. Нажмите **Save**.

Регистранты выбирают количества во время регистрации и итоги считаются против ёмкости, так что вы никогда не переполняете.

## Коды скидок

1. Расширьте аккордеон **Discount Codes** и нажмите **Add Discount Code**.
2. Введите **Code**, который регистранты будут вводить.
3. Выберите **Type** — **Percent** или **Amount** — и его **Value**.
4. Опционально ограничьте код **Start Date** / **End Date**, **Min Members** (минимальное количество участников на регистрации) и **Max Uses**.
5. Нажмите **Save**.

Каждый код показывает подсчёт **Uses**, так что вы можете видеть, как часто он был погашен. Регистранты получают мгновенную обратную связь когда они применяют код — включая чёткие сообщения когда код истёк, не начался или нуждается в большем количестве участников.

## Лист ожидания

Включите **Enable Waitlist** в карте Registration Settings. Когда событие достигает ёмкости:

- Новые регистранты предлагаются место на листе ожидания вместо того чтобы быть отвергнутыми. Они завершают ту же регистрацию (платёж пропускается во время нахождения на листе ожидания).
- Когда кто-то отменяет, самая старая регистрация на листе ожидания **продвигается автоматически** и получает электронное письмо что место открыто. Если они должны остаток, электронное письмо связывает их с завершением платежа.
- Вы можете продвинуть кого-то вручную в любое время с действием **Promote** на строке на листе ожидания — полезно после повышения ёмкости события.

:::info
Продвинутые регистрации остаются *pending* до тех пор пока любой остаток не будет оплачен; платёж (или отсутствие необходимости платить) подтверждает их.
:::

## Реестр регистраций

Откройте событие со страницы Registrations для просмотра каждой регистрации. Таблица показывает **Name**, **Members**, **Type** (тип каждого участника), **Paid / Total** (с предупреждением о остатке когда деньги всё ещё должны), **Status** и **Date**, плюс чипы подсчётов для каждого типа над таблицей.

- Нажмите значок деталей строки для открытия диалога **Registration Details** — члены, выборы, оплачено/остаток и таблица **Payments**, перечисляющая каждый платёж (сумма, метод, дата).
- **Export CSV** загружает полный реестр со столбцами для членов, типов участников, выборов, оплачено/итого/остаток, статус и один столбец на вопрос регистрации.
- **Add Attendee** всё ещё позволяет вам записывать оффлайн регистрации вручную.

:::info
Возвраты не обрабатываются внутри B1. Если вам нужно вернуть отменённую оплаченную регистрацию, выпустите возврат из панели провайдера платежей (например, Stripe).
:::

## Как работает платёж

Платежи проходят через того же провайдера платежей, которого ваша церковь уже использует для пожертвований — детали карты идут прямо к провайдеру и никогда не касаются серверов B1. Цены всегда вычисляются на сервере из ваших сконфигурированных типов, выборов и кодов скидок, так что регистранты не может изменить итого. Залогинные члены могут платить сохранённой картой; гости вводят карту при завершении.
