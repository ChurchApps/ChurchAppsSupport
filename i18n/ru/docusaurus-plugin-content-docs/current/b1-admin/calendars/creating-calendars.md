# Создание календарей

<div class="article-intro">

Создание календаря в B1 Admin позволяет создать кураторский обзор событий, подключив одну или несколько групп. События управляются руководителями групп в своих группах, и ваш календарь отображает эти события в одном месте. Даже администратор домена не может добавлять или редактировать события непосредственно в разделе календаря, если он не является руководителем группы, к которой относятся события.

</div>

<div class="prereqs">
<h4>Прежде чем начать</h4>

- Настройте [группы](../groups/creating-groups.md), события которых вы хотите включить в свой календарь
- Вам нужен административный доступ к разделу Календари в B1 Admin

</div>

## Создание нового календаря

1. В B1 Admin перейдите в раздел **Веб-сайт**, затем в раздел **Календари**.
2. Нажмите **Добавить календарь**.
3. Введите **название** для вашего календаря (например, "События молодёжного служения" или "Основной календарь церкви").
4. Добавьте необязательное **описание**, чтобы помочь вашей команде понять, для чего предназначен этот календарь.
5. Нажмите **Создать**, чтобы сохранить новый календарь.

## Страница сведений о календаре

После создания календаря нажмите на него, чтобы открыть страницу сведений. Эта страница имеет две основные области:

- **Левый столбец** -- Просмотр календаря с событиями из подключённых групп.
- **Правый столбец** -- Список связанных групп. Здесь вы управляете тем, какие группы включены в этот календарь.

## Подключение групп

Группы, в которых есть события в календаре, автоматически появляются в списке групп в правой части страницы сведений.

1. Нажмите **Добавить** в разделе групп, чтобы связать группу с вашим календарём.
2. Выберите группу из выпадающего списка.
3. Выберите, включать ли **все события** из этой группы или только **конкретные события**.
4. Нажмите **Сохранить**.

:::tip
Подключение групп к вашему календарю -- это мощный способ автоматического объединения событий. Когда руководитель группы добавляет событие в свою [группу](../groups/creating-groups.md), оно может автоматически попасть в ваш общецерковный календарь без дополнительной работы с вашей стороны.
:::

:::info
Если вы хотите создать единый календарь, который собирает события из многих групп вашей церкви, см. [Кураторский календарь](curated-calendar) для упрощённого подхода.
:::

## Включение регистрации на события

Вы можете включить регистрацию для любого события календаря, чтобы участники могли зарегистрироваться через веб-сайт B1 или мобильное приложение.

1. Нажмите на существующее событие или создайте новое.
2. В редакторе событий переключите **Регистрация**, чтобы включить её.
3. Настройте параметры регистрации:
   - **Вместимость** (необязательно) -- Установите максимальное количество регистраций. Оставьте пустым для неограниченного количества.
   - **Открытие регистрации** -- Дата и время, когда регистрация становится доступной.
   - **Закрытие регистрации** -- Дата и время, когда регистрация закрывается.
   - **Теги** -- Метки через запятую (например, "молодёжь, retreat, vbs") для помощи в категоризации событий с регистрацией.
4. Сохраните событие.

После включения регистрации участники увидят кнопку **Зарегистрироваться на это событие** при просмотре события на [веб-сайте B1](../../b1-church/events/registering) или в [мобильном приложении B1](../../b1-mobile/events/registering).

### Управление регистрациями

Для просмотра и управления регистрациями на ваши события:

1. Перейдите на страницу **Регистрации** в B1 Admin.
2. Вы увидите таблицу всех событий с включённой регистрацией, показывающую название события, дату, текущее количество регистраций по сравнению с вместимостью и теги.
3. Нажмите на событие, чтобы увидеть полный список регистраций, включая имена, количество участников, статус и дату регистрации.
4. На странице сведений вы можете:
   - **Отменить** отдельные регистрации
   - **Удалить** регистрации навсегда
   - **Экспортировать** все регистрации в CSV

:::tip
Используйте индикатор прогресса вместимости для мониторинга того, насколько быстро заполняются события. Индикатор становится красным, когда событие достигает вместимости или превышает её.
:::
