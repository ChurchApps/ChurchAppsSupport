# Создание календарей

<div class="article-intro">

Создание календаря в B1 Admin позволяет вам создать организованное представление событий, подключив одну или несколько групп. События управляются лидерами групп в рамках их групп, а ваш календарь отображает эти события в одном месте. Даже администратор домена не может добавлять или редактировать события непосредственно в разделе календаря, если он не является лидером группы, к которой принадлежат события.

</div>

<div class="prereqs">
<h4>Перед тем как начать</h4>

- Установите [группы](../groups/creating-groups.md), события которых вы хотите включить в календарь
- Вам нужен административный доступ к разделу "Календари" в B1 Admin

</div>

## Создание нового календаря

1. В B1 Admin перейдите на **Веб-сайт**, затем в раздел **Календари**.
2. Нажмите **Добавить календарь**.
3. Введите **имя** для вашего календаря (например, "События молодежного служения" или "Основной календарь церкви").
4. Дополнительно добавьте **описание**, чтобы помочь вашей команде понять, для чего предназначен этот календарь.
5. Нажмите **Создать**, чтобы сохранить новый календарь.

## Страница деталей календаря

После создания календаря нажмите на него, чтобы открыть страницу с деталями. На этой странице есть две основные области:

- **Левый столбец** -- представление календаря с событиями из подключенных групп.
- **Правый столбец** -- список связанных групп. Здесь вы управляете тем, какие группы включены в этот календарь.

## Подключение групп

Группы, которые имеют события в календаре, автоматически отображаются в списке групп в правой части страницы с деталями.

1. Нажмите **Добавить** в разделе групп, чтобы связать группу с вашим календарем.
2. Выберите группу из раскрывающегося списка.
3. Выберите, включить **все события** из этой группы или только **конкретные события**.
4. Нажмите **Сохранить**.

:::tip
Подключение групп к вашему календарю - это мощный способ автоматически объединить события. Когда лидер группы добавляет событие в свою [группу](../groups/creating-groups.md), оно может появиться в церковном календаре без дополнительной работы с вашей стороны.
:::

:::info
Если вы хотите создать один календарь, который извлекает события из множества групп в вашей церкви, см. [Организованный календарь](curated-calendar) для упрощенного подхода.
:::

## Включение регистрации на события

Вы можете включить регистрацию для любого события календаря, чтобы участники могли зарегистрироваться через веб-сайт B1 или мобильное приложение.

1. Нажмите на существующее событие или создайте новое.
2. В редакторе событий переключите **Регистрация**, чтобы включить её.
3. Настройте параметры регистрации:
   - **Вместимость** (необязательно) -- установите максимальное количество регистраций. Оставьте пустым для неограниченного количества.
   - **Регистрация открывается** -- дата и время, когда регистрация становится доступной.
   - **Регистрация закрывается** -- дата и время, когда регистрация закрывается.
   - **Теги** -- метки, разделенные запятыми (например, "молодежь, отступление, летняя библейская школа"), чтобы помочь категоризировать события с регистрацией.
   - **Вопросы регистрации** -- дополнительно прикрепите [форму](../forms/creating-forms.md), чтобы регистранты ответили на дополнительные вопросы (диетические ограничения, размер футболки, контактное лицо на случай чрезвычайной ситуации и т.д.) как часть регистрации. Выберите **Нет**, чтобы пропустить вопросы.
   - **Включить список ожидания** -- когда событие заполнится, позвольте дополнительным регистрантам присоединиться к списку ожидания вместо отказа. См. [Платные регистрации](paid-registrations#waitlist).
4. Сохраните событие.

Для платных событий та же страница параметров позволяет вам определить **Типы участников** с ценами, опциональные **Выборы** (дополнения) и **Коды скидок**, с платежом, собранным через поставщика пожертвований вашей церкви. Полное руководство см. в разделе [Платные регистрации](paid-registrations).

После включения регистрации участники увидят кнопку **Зарегистрироваться на это событие**, когда они посмотрят событие на [веб-сайте B1](../../b1-church/events/registering) или [мобильном приложении B1](../../b1-mobile/events/registering). Если вы прикрепили форму, регистранты увидят шаг **Вопросы** во время регистрации и их ответы сохраняются с их регистрацией.

:::info
Вопросы регистрации работают только с формами, которые **не** помечены как Ограниченные. Ограниченная форма автоматически пропускается во время регистрации вместо отображения, поэтому используйте неограниченную форму при прикреплении вопросов к событию.
:::

### Управление регистрациями

Для просмотра и управления регистрациями ваших событий:

1. Перейдите на страницу **Регистрации** в B1 Admin.
2. Вы увидите таблицу всех событий с включенной регистрацией, показывающую название события, дату, текущее количество регистраций в сравнении с вместимостью и теги.
3. Нажмите на событие, чтобы увидеть полный список регистраций, включая имена, количество участников, типы участников, статус платежа и дату регистрации.
4. На странице деталей вы можете:
   - **Добавить участника** -- вручную зарегистрировать кого-то, кто зарегистрировался в автономном режиме или по телефону.
   - **Отменить** отдельные регистрации
   - **Удалить** регистрации постоянно
   - **Повысить** регистрации из списка ожидания, когда открывается место
   - **Экспортировать CSV** -- загрузить все регистрации, включая типы участников, выборы, суммы платежей и ответы на вопросы

Если событие имеет прикрепленные вопросы регистрации, страница деталей также показывает фильтр **Только неотвеченные вопросы**, чтобы быстро найти регистрантов, которые еще не отправили ответы, и кнопку **Просмотр ответов** для каждой отвеченной регистрации, чтобы увидеть их ответы. Платные события добавляют столбец **Тип**, столбец **Оплачено / Итого**, подсчеты по типам и диалог деталей платежей -- см. [Платные регистрации](paid-registrations#the-registration-roster).

:::tip
Используйте полосу хода вместимости для мониторинга того, как быстро заполняются события. Полоса становится красной, когда событие находится на вместимости или превышает её.
:::
