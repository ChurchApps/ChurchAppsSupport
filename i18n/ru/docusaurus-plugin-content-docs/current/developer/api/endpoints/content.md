# Конечные точки контента

<div class="article-intro">

Модуль контента управляет страницами веб-сайта, разделами, элементами, блоками, сообщениями блога, перенаправлениями, проповедями, плейлистами, потоковыми сервисами, событиями, курируемыми календарями, файлами, галереями, переводами Библии и поиском стихов, песнями, аранжировками, глобальными стилями, стоковыми фотографиями и параметрами. Это самый большой модуль в API и питает CMS, медиа/потоковую трансляцию, планирование поклонения и функции Библии во всех приложениях ChurchApps.

</div>

**Базовый путь:** `/content`

## Страницы

Базовый путь: `/content/pages`

| Метод | Путь | Аут. | Разрешение | Описание |
|--------|------|------|------------|-------------|
| GET | `/:churchId/tree?url=&id=` | Public | — | Загрузить полное дерево страницы (разделы, элементы, блоки) по URL или ID. Убирает внутренние ID при выборке по URL. Выборки на основе URL применяют `pages.visibility` — закрытая страница возвращает `{ restricted: true, visibility }`, если только JWT не удовлетворяет условию |
| GET | `/public/:churchId` | Public | — | Список открытых страниц (`url`, `title`, `metaDescription`); только `visibility = everyone` |
| GET | `/:id` | JWT | — | Получить страницу по ID |
| GET | `/` | JWT | — | Список всех страниц для церкви |
| POST | `/duplicate/:id` | JWT | Content.Edit | Дублировать страницу со всеми разделами и элементами |
| POST | `/temp/ai` | JWT | Content.Edit | Сохранить генерируемую AI страницу (страница, разделы и элементы в одном вызове) |
| POST | `/` | JWT | Content.Edit | Создать или обновить страницы (пакет) |
| DELETE | `/:id` | JWT | Content.Edit | Удалить страницу |

### Пример: загрузить дерево страниц

```
GET /content/pages/abc-church-id/tree?url=/about
```

```json
{
  "name": "About",
  "url": "/about",
  "sections": [
    {
      "background": "#FFFFFF",
      "textColor": "dark",
      "elements": [
        { "elementType": "textWithPhoto", "answers": { "text": "Welcome" } }
      ]
    }
  ]
}
```

## Сообщения (блог)

Базовый путь: `/content/posts`

Сообщения блога — это метаданные над обычными страницами: `pageId` каждого сообщения ссылается на страницу, содержащую тело, и строка сообщения добавляет `title`, `slug` (уникально на церковь), `excerpt`, `authorId`, `photoUrl`, `publishDate`, `category` и `tags`. Сообщение опубликовано, когда `publishDate` установлена и находится в прошлом. См. [Архитектура конструктора веб-сайта](../../architecture/website-builder#blog-posts-over-pages).

| Метод | Путь | Аут. | Разрешение | Описание |
|--------|------|------|------------|-------------|
| GET | `/public/:churchId?category=&tag=&page=&pageSize=` | Public | — | Список опубликованных сообщений, разбит на страницы (макс. 50 на странице) |
| GET | `/public/:churchId/slug/:slug` | Public | — | Получить метаданные опубликованного сообщения по slug |
| GET | `/rss/:churchId?siteUrl=` | Public | — | RSS 2.0 лента опубликованных сообщений (ссылки построены как `{siteUrl}/blog/{slug}`) |
| GET | `/:id` | JWT | — | Получить сообщение по ID |
| GET | `/` | JWT | — | Список всех сообщений для церкви |
| POST | `/` | JWT | Content.Edit | Создать или обновить сообщения (пакет) |
| DELETE | `/:id` | JWT | Content.Edit | Удалить сообщение |

## Перенаправления

Базовый путь: `/content/redirects`

Перенаправления URL на церковь (`fromPath` → `toPath`), ограничено 200 на церковь. Пути нормализованы (малые буквы, ведущий слэш, без завершающего слэша) и `fromPath` уникален на церковь. B1App разрешает их на было бы 404 и выдаёт HTTP 308.

| Метод | Путь | Аут. | Разрешение | Описание |
|--------|------|------|------------|-------------|
| GET | `/public/:churchId?path=` | Public | — | Разрешить путь (или список всех перенаправлений, когда `path` опущен) |
| GET | `/:id` | JWT | — | Получить перенаправление по ID |
| GET | `/` | JWT | — | Список всех перенаправлений для церкви |
| POST | `/` | JWT | Content.Edit | Создать или обновить перенаправления. Отклоняет `fromPath = toPath` и применяет ограничение в 200 строк |
| DELETE | `/:id` | JWT | Content.Edit | Удалить перенаправление |
