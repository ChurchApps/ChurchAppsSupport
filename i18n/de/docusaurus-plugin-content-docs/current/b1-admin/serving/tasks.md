# Aufgaben

<div class="article-intro">

Aufgaben ermöglichen es Ihnen, Aktionspunkte an Personen oder Gruppen in Ihrer Gemeinde zuzuweisen. Ob es sich um eine einmalige Aufgabe oder eine wiederkehrende Verantwortung handelt -- Aufgaben helfen Ihnen zu verfolgen, was erledigt werden muss und wer verantwortlich ist, damit nichts übersehen wird.

</div>

<div class="prereqs">
<h4>Bevor Sie beginnen</h4>

- Stellen Sie sicher, dass die Personen oder [Gruppen](../groups/creating-groups.md), denen Sie Aufgaben zuweisen möchten, in B1 Admin vorhanden sind
- Navigieren Sie zum Bereich **Dienst**, um auf die Aufgabenseite zuzugreifen

</div>

## Aufgaben anzeigen

Navigieren Sie zu **Dienst** und öffnen Sie die Seite **Aufgaben**. Sie sehen eine Liste Ihrer offenen Aufgaben. Jede Aufgabe zeigt Titel, Zugewiesenen und Status.

:::tip
Standardmäßig werden nur offene Aufgaben angezeigt. Schalten Sie **Erledigte anzeigen** um, um auch abgeschlossene Aufgaben zu sehen.
:::

## Eine Aufgabe erstellen

1. Klicken Sie auf der Aufgabenseite auf **Aufgabe hinzufügen**.
2. Geben Sie einen **Titel** für die Aufgabe ein.
3. Klicken Sie auf das Feld **Zuweisen an** und geben Sie den Namen einer Person ein, oder wechseln Sie zu **Gruppe** und geben Sie einen Gruppennamen ein.
4. Klicken Sie auf **Suchen** und wählen Sie die Person oder Gruppe aus den Ergebnissen.
5. Fügen Sie eine **Notiz** mit Details oder Anweisungen für den Zugewiesenen hinzu.
6. Klicken Sie auf **Speichern**.

Die Aufgabe erscheint nun in der Aufgabenliste und ist für den Zugewiesenen auf seinem Dashboard und in der mobilen App sichtbar.

## Eine Aufgabe verwalten

Klicken Sie auf eine beliebige Aufgabe, um ihre Detailseite zu öffnen. Von hier aus können Sie:

- **Den Status ändern** -- Mit dem Status-Dropdown zwischen **Offen** und **Erledigt** wechseln.
- **Die Aufgabe umzuweisen** -- Den Zugewiesenen zu einer anderen Person oder Gruppe ändern.
- **Notizen hinzufügen** -- Zusätzliche Informationen oder Aktualisierungen für den Zugewiesenen eintragen.
- **Den Titel bearbeiten** -- Den Aufgabentitel nach Bedarf aktualisieren.

## Aufgaben-Lebenszyklus

1. Eine neue Aufgabe startet mit dem Status **Offen**.
2. Der Zugewiesene erhält eine Benachrichtigung und kann die Aufgabe von seinem Dashboard, der Aufgabenseite oder der mobilen App aus einsehen.
3. Sobald die Arbeit erledigt ist, ändert der Zugewiesene (oder ein Administrator) den Status auf **Erledigt**.
4. Erledigte Aufgaben werden aus der Standardansicht entfernt, sind aber weiterhin über **Erledigte anzeigen** zugänglich.

:::info
Sie können wiederkehrende Aufgaben automatisieren, sodass sie nach einem Zeitplan erstellt werden. Siehe [Automatisierungen](./automations.md) für Details zur Einrichtung.
:::

## Nächste Schritte

- Richten Sie [Automatisierungen](./automations.md) ein, um wiederkehrende Aufgaben automatisch zu erstellen
- Verwenden Sie Aufgaben zusammen mit [Dienstplänen](./plans.md), um Aktionspunkte zur Gottesdienstvorbereitung zu verfolgen
