# Aufgaben

<div class="article-intro">

Aufgaben ermöglichen es Ihnen, Aktionselemente Personen oder Gruppen in Ihrer Kirche zuzuweisen. Ob es sich um eine einmalige Aufgabe oder eine wiederkehrende Verantwortung handelt, Aufgaben helfen Ihnen, zu verfolgen, was getan werden muss und wer verantwortlich ist, damit nichts übersehen wird.

</div>

<div class="prereqs">
<h4>Bevor Sie beginnen</h4>

- Stellen Sie sicher, dass die Personen oder [Gruppen](../groups/creating-groups.md), denen Sie Aufgaben zuweisen möchten, in B1 Admin vorhanden sind
- Navigieren Sie zum Bereich **Dienst**, um auf die Seite Aufgaben zuzugreifen

</div>

## Aufgaben anzeigen

Navigieren Sie zu **Dienst** und öffnen Sie die Seite **Aufgaben**. Sie sehen eine Liste Ihrer offenen Aufgaben. Jede Aufgabe zeigt ihren Titel, den Zugewiesenen und den Status.

:::tip
Standardmäßig werden nur offene Aufgaben angezeigt. Schalten Sie **Abgeschlossene anzeigen** um, um auch abgeschlossene Aufgaben zu sehen.
:::

## Aufgabe erstellen

1. Klicken Sie auf der Seite Aufgaben auf **Aufgabe hinzufügen**.
2. Geben Sie einen **Titel** für die Aufgabe ein.
3. Klicken Sie auf das Feld **Zuweisen an** und geben Sie den Namen einer Person ein, oder wechseln Sie zu **Gruppe** und geben Sie einen Gruppennamen ein.
4. Klicken Sie auf **Suchen** und wählen Sie die Person oder Gruppe aus den Ergebnissen aus.
5. Fügen Sie eine **Notiz** mit Details oder Anweisungen für den Zugewiesenen hinzu.
6. Klicken Sie auf **Speichern**.

Die Aufgabe erscheint nun in der Aufgabenliste und ist für den Zugewiesenen auf seinem Dashboard und in der mobilen App sichtbar.

## Aufgabe verwalten

Klicken Sie auf eine beliebige Aufgabe, um ihre Detailseite zu öffnen. Von hier aus können Sie:

- **Ändern Sie den Status** – Wechseln Sie zwischen **Offen** und **Abgeschlossen** mit dem Status-Dropdown.
- **Weisen Sie die Aufgabe erneut zu** – Aktualisieren Sie den Zugewiesenen zu einer anderen Person oder Gruppe.
- **Notizen hinzufügen** – Fügen Sie zusätzliche Informationen oder Updates für den Zugewiesenen ein.
- **Bearbeiten Sie den Titel** – Aktualisieren Sie den Aufgabentitel nach Bedarf.

## Aufgaben-Lebenszyklus

1. Eine neue Aufgabe startet mit einem Status von **Offen**.
2. Der Zugewiesene erhält eine Benachrichtigung und kann die Aufgabe von seinem Dashboard, der Seite Aufgaben oder der mobilen App anzeigen.
3. Sobald die Arbeit abgeschlossen ist, ändert der Zugewiesene (oder ein Admin) den Status zu **Abgeschlossen**.
4. Abgeschlossene Aufgaben werden aus der Standardansicht entfernt, können aber weiterhin durch Aktivieren von **Abgeschlossene anzeigen** abgerufen werden.

:::info
Sie können wiederkehrende Aufgaben automatisieren, damit sie planmäßig erstellt werden. Weitere Informationen finden Sie unter [Automatisierungen](./automations.md).
:::

:::tip
Wenn Sie viele Personen durch die gleiche Reihe von Phasen verschieben müssen – z. B. Erstkontakt-Nachverfolgung oder einen Mitgliedschaftsprozess – verwenden Sie [Arbeitsabläufe](./workflows.md). Arbeitsabläufe verwandeln Aufgaben in Karten auf einem Drag-and-Drop-Board mit Schritten, Fälligkeitsdaten und Weiterleitungen.
:::

## Nächste Schritte

- Führen Sie Personen mit [Arbeitsabläufen](./workflows.md) durch einen mehrstufigen Prozess
- Richten Sie [Automatisierungen](./automations.md) ein, um wiederkehrende Aufgaben automatisch zu erstellen
- Verwenden Sie Aufgaben zusammen mit [Dienst-Plänen](./plans.md), um Aktionselemente zur Dienstvorbereitung zu verfolgen
