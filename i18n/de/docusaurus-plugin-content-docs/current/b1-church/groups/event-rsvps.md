# Ereignis-RSVPs

<div class="article-intro">

Gruppen-Ereignisse können RSVPs sammeln, damit Leiter wissen, wer kommt. Auf Ihrem Gruppen-Kalender fragt jedes anstehende Ereignis „Wirst du teilnehmen?" — antworte mit einem Tap, ändere deine Meinung jederzeit, und sieh auf einen Blick, wie viele andere gehen.

</div>

<div class="prereqs">
<h4>Bevor Sie beginnen</h4>

- Sie müssen [eingeloggt](../getting-started/logging-in.md) sein und ein Mitglied der Gruppe
- Das Ereignis muss RSVPs erlaubt haben (sie sind standardmäßig an; Leiter können sie pro Ereignis ausschalten)

</div>

## Auf ein Ereignis reagieren

1. Öffnen Sie Ihre Gruppe im Mitglieder-Portal und gehen Sie zur **Ereignisse** / Kalender-Ansicht.
2. Jede anstehende Ereignis-Karte zeigt **Wirst du teilnehmen?** mit drei Schaltflächen: **Teilnehmend**, **Vielleicht** und **Kann nicht gehen**.
3. Tippen Sie auf Ihre Antwort. Ihre Auswahl hebt sich hervor, und jede Schaltfläche zeigt einen Live-Zähler, wie viele Personen diese Antwort gegeben haben.
4. Tippen Sie auf Ihre hervorgehobene Antwort erneut, um sie zu löschen, oder tippen Sie auf eine andere Schaltfläche, um Ihre Antwort zu ändern.

Ihre RSVP gilt für dieses spezifische Vorkommen — für ein wöchentliches Treffen beantworten Sie jedes Woche des Ereignisses separat.

## Für Gruppenleiter

- **Reagierungen anzeigen** -- Leiter sehen eine **Reagierungen anzeigen**-Schaltfläche auf jeder Ereignis-Karte. Sie öffnet das **Reagierungen**-Blatt mit einer Liste aller Personen nach Antwort — Teilnehmend, Vielleicht und Kann nicht gehen. Wenn sich noch niemand gemeldet hat, zeigt es „Noch keine Reagierungen".
- **RSVPs ein- oder ausschalten** -- Beim Erstellen oder Bearbeiten eines Ereignisses verwenden Sie den **RSVPs zulassen**-Schalter. Ereignisse mit RSVPs ausgeschaltet zeigen keine Antwort-Schaltflächen. Mitarbeiter, die Ereignisse in Masse von B1 Admin erstellen, haben dieselbe Option — siehe [Gruppenkalender](../../b1-admin/groups/group-calendar#event-rsvps).

:::tip
RSVPs sind eine leichte Kopfzahl. Wenn Ihr Ereignis Anmeldungsdetails, Kapazitätsverwaltung oder Zahlung benötigt, verwenden Sie stattdessen [Ereignis-Registrierung](../events/registering).
:::
