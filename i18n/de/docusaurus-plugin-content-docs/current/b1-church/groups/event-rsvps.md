# Event-RSVPs

<div class="article-intro">

Gruppen-Events können RSVPs sammeln, damit Leiter wissen, wer kommt. Im Kalender Ihrer Gruppe fragt jedes bevorstehende Event "Wirst du kommen?" – antworten Sie mit einem Tippen, ändern Sie Ihre Meinung jederzeit, und sehen Sie auf einen Blick, wie viele andere gehen.

</div>

<div class="prereqs">
<h4>Bevor Sie beginnen</h4>

- Sie müssen [angemeldet sein](../getting-started/logging-in.md) und ein Mitglied der Gruppe
- Das Event muss RSVPs erlaubt haben (sie sind standardmäßig aktiviert; Leiter können sie pro Event ausschalten)

</div>

## Auf ein Event antworten

1. Öffnen Sie Ihre Gruppe im Mitgliederportal und gehen Sie zur Ansicht **Events** / Kalender.
2. Jede bevorstehende Event-Karte zeigt **Wirst du kommen?** mit drei Schaltflächen: **Gehe hin**, **Vielleicht** und **Kann nicht gehen**.
3. Tippen Sie auf Ihre Antwort. Ihre Wahl wird hervorgehoben, und jede Schaltfläche zeigt eine Live-Zählung, wie viele Menschen diese Antwort gegeben haben.
4. Tippen Sie erneut auf Ihre hervorgehobene Antwort, um sie zu löschen, oder tippen Sie auf eine andere Schaltfläche, um Ihre Antwort zu ändern.

Ihre RSVP gilt für diesen spezifischen Vorkommen – für ein wöchentliches Treffen beantworten Sie das Event jeder Woche separat.

## Für Gruppenleiter

- **Antworten anzeigen** -- Leiter sehen auf jeder Event-Karte eine Schaltfläche **Antworten anzeigen**. Sie öffnet das Blatt **Antworten**, das jeden nach Antwort auflistet – Gehe hin, Vielleicht und Kann nicht gehen. Falls noch niemand geantwortet hat, zeigt es "Noch keine Antworten".
- **RSVPs ein- oder ausschalten** -- Verwenden Sie beim Erstellen oder Bearbeiten eines Events den Schalter **RSVPs zulassen**. Events mit ausgeschalteten RSVPs zeigen keine Antwortschaltflächen. Mitarbeiter, die Events massenweise von B1 Admin erstellen, haben die gleiche Option – siehe [Gruppenkalender](../../b1-admin/groups/group-calendar#event-rsvps).

:::tip
RSVPs sind eine leichte Kopfzahl. Falls Ihr Event Anmeldungsdetails, Kapazitätsverwaltung oder Zahlung benötigt, verwenden Sie stattdessen [Event-Registrierung](../events/registering).
:::
