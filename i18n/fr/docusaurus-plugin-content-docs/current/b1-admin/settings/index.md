# Paramètres

<div class="article-intro">

La section Paramètres est l'endroit où vous configurez le compte de votre église, l'image de marque et les options d'administration dans B1 Admin. Vous pouvez y gérer tout, du nom de votre église et de votre sous-domaine aux autorisations des utilisateurs et à la configuration de l'application mobile.

</div>

## Ce que vous trouverez ici

La zone Paramètres est organisée en quatre sous-sections accessibles depuis la barre latérale :

1. **Paramètres** -- Configurez les informations de base de votre église, l'image de marque et le sous-domaine. Consultez [Paramètres de l'église](./church-settings.md) pour plus de détails.
2. **Applications mobiles** -- Configurez et personnalisez les onglets de navigation qui apparaissent dans l'application B1 Mobile pour vos membres. Consultez [Paramètres de l'application mobile](./mobile-app.md) pour plus de détails.
3. **Administration du serveur** -- Accédez aux outils d'administration avancés pour gérer les paramètres au niveau du serveur de votre église.
4. **Formulaires** -- Créez et gérez des formulaires personnalisés pour recueillir des informations auprès de votre assemblée.

## La page principale des paramètres

Lorsque vous ouvrez la page Paramètres pour la première fois, vous verrez le **nom de votre église** et votre **sous-domaine** affichés en haut. L'en-tête fournit des boutons d'accès rapide pour les tâches courantes :

- **Modifier les paramètres** -- Mettez à jour le nom, l'adresse, les coordonnées et l'image de marque de votre église.
- **Applications mobiles** -- Accédez directement à la configuration de l'application mobile.
- **Rôles** -- Gérez les rôles des utilisateurs et les autorisations pour votre équipe.
- **Import/Export** -- Transférez des données entre les systèmes à l'aide de l'outil d'import/export.

:::tip
Commencez par configurer le nom et l'image de marque de votre église dans **Modifier les paramètres**, puis configurez les [Rôles](./roles-permissions.md) pour inviter les membres de votre équipe avec le niveau d'accès approprié.
:::

:::info
Pour des informations sur la protection de vos données, consultez [Sécurité des données](./data-security.md).
:::
