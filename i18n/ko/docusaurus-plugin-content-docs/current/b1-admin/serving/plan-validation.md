# 계획 검증 및 자원봉사자 알림

<div class="article-intro">

B1 Admin은 일요일 전에 자동으로 계획의 문제를 확인합니다 — 채워지지 않은 위치, 일정 충돌, 해당 날짜를 차단한 자원봉사자. 모든 것이 괜찮으면 단 한 번의 클릭으로 전체 팀에 알릴 수 있습니다.

</div>

<div class="prereqs">
<h4>시작하기 전에</h4>

- [예배 계획](./plans.md) 만들기 및 자원봉사자를 위치에 할당
- 충돌 감지가 중복을 확인할 수 있도록 [예배 시간](./plans.md) 추가
- 자원봉사자가 푸시 알림을 받을 수 있도록 B1 모바일 앱이 설치되어 있는지 확인

</div>

## 검증 패널

모든 계획에는 구축할 때 자동으로 실행되는 **검증** 패널이 있습니다. 세 가지를 확인합니다:

### 채워지지 않은 위치

위치에 현재 할당된 것보다 더 많은 사람이 필요하면 검증 패널에 정확히 무엇이 필요한지 나열합니다 — 예: *"음향 기술: 1명 더 필요"*. 주가 도착하기 전에 계획이 완전히 배치되었는지 한눈에 알 수 있습니다.

### 일정 충돌

자원봉사자가 같은 계획 내에서 시간이 겹치는 두 개의 위치에 할당된 경우 검증 패널은 충돌을 표시합니다 — 예: *"Jane Smith: 일요일 예배 중 경배 지도자와 아동 체크인 간 시간 충돌."* 이것은 이중 예약이 일요일 아침의 문제가 되기 전에 포착합니다.

### 차단 날짜

자원봉사자는 B1 모바일에서 사용할 수 없는 날짜를 설정할 수 있습니다. 누군가 차단 날짜 내에 해당하는 계획에 할당되면 검증 패널이 자동으로 충돌 표면을 드러내어 교체자를 찾을 수 있습니다.

### 교차 계획 충돌

검증은 한 번에 모든 계획을 확인합니다. 같은 자원봉사자가 겹치는 시간의 두 개의 다른 계획에 할당된 경우 — 예를 들어 오전 9시 예배와 오전 10시 30분까지 실행되는 오전 10시 예배 — B1 Admin은 해당 사람을 계획 전체에 이중 예약으로 표시합니다.

:::tip
검증을 실행하기 위해 아무것도 할 필요가 없습니다 — 할당을 추가하거나 변경할 때마다 자동으로 업데이트됩니다. 계획을 구축할 때 패널을 주시하세요.
:::

## 자원봉사자에게 알림

계획이 설정되면 검증 패널에서 직접 할당된 모든 자원봉사자에게 한 번에 알릴 수 있습니다.

1. 계획을 열고 **검증** 패널로 스크롤합니다
2. 알리지 않은 자원봉사자가 있으면 알려야 할 사람 수를 표시하는 링크가 표시됩니다(예: *"8명 자원봉사자에게 알림"*)
3. 링크를 클릭하여 아직 알리지 않은 모든 사람에게 푸시 알림을 보냅니다
4. 자원봉사자는 할당되었으며 확인하도록 요구하는 알림을 받습니다

:::info
아직 알리지 않은 자원봉사자만 포함됩니다. 나중에 계획에 누군가를 추가하면 링크가 다시 나타나므로 팀의 나머지 부분을 다시 알리지 않고 새 추가에 알릴 수 있습니다.
:::

:::warning
자원봉사자는 푸시 알림을 받으려면 B1 모바일 앱이 설치되어 있고 알림이 활성화되어 있어야 합니다. [알림 가이드](/docs/b1-mobile/community/notifications)를 참조하여 자원봉사자가 기기에서 이를 활성화하는 방법을 알아보세요.
:::
