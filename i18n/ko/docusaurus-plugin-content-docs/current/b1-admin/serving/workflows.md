# 워크플로우

<div class="article-intro">

워크플로우는 사람을 시각적 보드의 일련의 단계로 이동시킵니다. 각 사람은 한 단계에서 다음 단계로 이동하는 카드가 됩니다. 첫 방문객 팔로우업에서 교인 절차, 첫 기부자 감사까지, 많은 사람을 같은 일련의 단계로 추적해야 할 때 사용합니다. 각 단계는 자원봉사자에게 무언가를 요구할 수 있으며(**전화 통화, 대화**) **그리고** 자동으로 실행되는 자동화된 작업(이메일 발송, 며칠 대기, 그룹에 추가)을 함께 실행할 수 있습니다. 워크플로우는 [작업](./tasks.md)을 드래그 앤드 드롭 칸반 보드로 확장하므로 아무도 떨어지지 않을 수 있습니다.

</div>

<div class="prereqs">
<h4>시작하기 전에</h4>

- 추적하려는 사람이 B1 Admin에 존재해야 합니다.
- [작업](./tasks.md)의 작동 방식을 숙지하세요. 보드의 각 카드는 작업이기 때문입니다.
- **이메일 발송** 작업을 사용하려면 먼저 보내려는 이메일 템플릿을 만듭니다(**메시징 → 템플릿 관리**에서 관리).
- 적절한 작업 권한이 필요합니다. 카드 보기, 편집, 워크플로우 관리는 별도의 권한 수준입니다([역할 및 권한](../settings/roles-permissions.md) 참고).

</div>

## 워크플로우 보기

**봉사**로 이동하고 **작업** 영역을 열어 메뉴에서 **워크플로우**를 선택합니다. 워크플로우가 나열되고 카테고리별로 그룹화되며, 활성 워크플로우가 강조 표시됩니다. 워크플로우를 클릭하여 보드를 엽니다.

## 워크플로우 만들기

1. 워크플로우 페이지에서 **워크플로우 추가**를 클릭합니다.
2. 시작 방법을 선택합니다:
   - **빈 워크플로우** -- 처음부터 시작하여 자신의 단계를 만듭니다.
   - **템플릿에서** -- 편집할 수 있는 기본 단계 세트로 시작합니다. 기본 제공 템플릿:
     - **새 방문자 팔로우업** -- 환영 이메일 발송 → 개인 전화 통화 → 다음 단계로 초대 → 연결됨
     - **교인 반입식** -- 관심 표현 → 반입식 등록 → 반입식 참석 → 교인권 완료
     - **첫 기부자 감사** -- 감사 편지 발송 → 기부금 영향 공유 → 청지기
3. 워크플로우에 **이름**을 지정합니다.
4. 선택적으로 관련 워크플로우를 그룹화할 **카테고리**를 할당합니다. 드롭다운에서 새 카테고리를 만들 수 있습니다.
5. 워크플로우를 **활성**으로 유지하여 사람을 추가할 수 있도록 하거나, 워크플로우 추가 목록에서 숨기려면 **비활성**으로 설정합니다.
6. **저장**을 클릭합니다.

:::tip
워크플로우 목록의 **복제** 버튼을 사용하여 기존 워크플로우(단계, 자동화된 작업, 라우팅 포함)를 복사하여 새 워크플로우의 시작점으로 삼을 수 있습니다.
:::

## 단계로 보드 구축

각 워크플로우 보드는 왼쪽에서 오른쪽으로 표시되는 **단계** 열로 구성됩니다. 워크플로우를 열고 **단계 추가**를 사용하여 프로세스의 각 단계를 만듭니다.

단계를 추가하거나 편집할 때 다음을 구성할 수 있습니다.

- **단계 이름** -- 열 제목(예: "환영 전화" 또는 "등록 대기중").
- **마감일(일)** -- 카드가 이 단계에 들어올 때 자동으로 마감일을 설정합니다. 마감일이 지난 카드는 **기한 초과**로 표시됩니다.
- **기본 담당자** -- 이 단계의 새 카드가 자동으로 배정되는 사람 또는 그룹.
- **자동화된 작업** -- 카드가 도착할 때 시스템이 자동으로 실행하는 작업(아래 참고).
- **라우팅** -- 카드가 단계를 떠날 때 이동할 위치([라우팅](#outcome-buttons-및-조건으로-카드-라우팅) 참고).

단계 열을 프로세스와 일치하는 순서로 드래그합니다. 이 순서는 다른 라우팅이 적용되지 않을 때 카드가 취하는 기본 경로도 정의합니다.

:::info
새 단계를 먼저 저장합니다. 자동화된 작업과 라우팅은 단계에 연결되므로, 단계가 존재한 후 편집기가 이러한 섹션을 잠금 해제합니다.
:::

## 자동화된 작업

모든 단계는 카드가 **단계에 들어올 때** 자동으로 실행되는 **자동화된 작업** 목록을 수행할 수 있습니다. 누군가가 건드리기 전입니다. 이것이 단계가 자원봉사자에게 **그리고** 동시에 팔로우업 주변의 루틴 작업을 처리하도록 하는 방법입니다.

단계 편집기에서 **자동화된 작업**을 열고, **작업 추가**를 클릭하고, 유형을 선택하고, 설정을 입력하고, 해당 작업의 저장 아이콘을 클릭합니다. 필요한 만큼 추가합니다. 작업은 **위에서 아래로 순서대로** 실행됩니다.

| 작업 | 기능 |
|---|---|
| **이메일 발송** | 선택한 이메일 템플릿을 사람에게 이메일로 보냅니다. 제목 줄을 재정의할 수 있습니다. |
| **대기** | 계속하기 전에 카드를 일정 기간 일시 중지합니다(아래 참고). |
| **그룹에 추가** | 선택한 [그룹](../groups/index.md)에 사람을 추가합니다. |
| **워크플로우에 추가** | 사람을 다른 워크플로우에 시작합니다(프로세스 간 핸드오프에 유용). |
| **메모 추가** | 카드의 기록에 메모를 기록합니다. |
| **필드 설정** | 사람 기록의 필드를 업데이트합니다: 교인 상태, 혼인 상태, 성별, 도시, 주, 우편번호. |
| **웹훅** | 카드의 세부 정보를 제공한 외부 웹 주소(URL)로 보냅니다(다른 시스템 연결용). |

단계의 모든 작업이 완료되면 카드은 **해당 단계에 안주합니다** 그러면 사람이 작업할 수 있습니다. 단계에 카드를 앞으로 이동시키는 자동 라우팅이 없는 한([완전히 자동화된 단계](#완전히-자동화된-단계) 참고).

:::info
자동화된 작업은 카드가 정상 흐름을 통해 도착할 때만 실행됩니다. 추가될 때, 결과 또는 자동 라우팅을 통해 들어올 때, 또는 대기 후에 실행됩니다. 직원이 카드를 수동으로 단계로 드래그하거나 보낼 때는 **다시 실행되지 않으므로** 사람이 같은 이메일을 두 번 받지 않습니다.
:::

### 이메일 발송

**이메일 발송**을 선택하고, 이메일 템플릿 중 하나를 선택하고, 선택적으로 사용자 정의 제목을 입력합니다. 카드가 단계에 들어오면 사람이 그 이메일을 자동으로 받습니다(사람 파일에 이메일 주소가 없으면 단계는 이 작업을 건너뜁니다).

### 며칠 대기(드립 시퀀스)

**대기** 작업은 설정한 일수 동안 카드를 잡아 놓습니다. 대기하는 동안 카드는 **연기됨**으로 표시됩니다. 대기 기간이 끝나면:

1. **같은 단계의 남은 작업** 실행 -- 이메일 발송 → 3일 대기 → 미리 알림 이메일 발송 같은 드립을 만들 수 있습니다.
2. 그 후 단계에 자동 라우팅이 있으면 카드가 이동합니다. 없으면 사람이 선택할 때까지 단계에 안주합니다.

:::tip
단계의 맨 처음에 있는 **대기**는 카드를 자원봉사자에게 표시하기 전에 "잡아 놓는" 간단한 방법입니다. 예: *7일 대기, 그 후 코치가 연락합니다*.
:::

## 카드로 사람 추가

보드에 사람을 추가하는 방법은 여러 가지입니다.

- **보드에서** -- 단계 열 하단의 **카드 추가**를 클릭하고 사람을 선택합니다. 또한 그룹을 선택할 수 있으며, 그 그룹의 모든 구성원이 카드로 추가됩니다.
- **사람의 기록에서** -- 사람의 페이지에서 **워크플로우에 추가**를 사용하여 워크플로우에 드롭합니다.
- **교인 검색에서** -- 여러 사람을 선택하고 대량 **워크플로우에 추가** 작업을 사용하여 모두 한 번에 추가합니다.
- **트리거를 사용하여 자동으로** -- 양식 제출 또는 첫 선물 같은 일이 일어날 때 사람을 추가합니다([트리거](#triggers) 아래 참고).

## 보드 작업

워크플로우를 열어 보드를 봅니다. 각 카드는 사람의 이름, 배정된 담당자, 마감일 또는 상태 칩(**기한 초과** 또는 **연기됨**)을 보여줍니다. 단계 열은 실행하는 자동화된 작업을 위한 작은 배지와 라우팅을 위한 주석도 표시하며, 카드가 어떻게 흐르는지 한눈에 알 수 있게 합니다.

- **카드 이동** -- 카드를 한 열에서 다음 열로 드래그하여 사람이 진행되도록 합니다.
- **카드 열기** -- 카드를 더블 클릭하거나 클릭하여 세부 서랍을 열어 단계를 변경하고, 재배정하고, 메모를 추가하고, 이미 일어난 일을 검토할 수 있습니다.

카드 서랍에서 다음을 할 수 있습니다:

- **배정** 카드를 다른 사람 또는 그룹에.
- **연기** 카드를 1일, 3일 또는 1주일 동안 마감일을 임시로 숨길 수 있습니다.
- **뒤로** 보내 이전 단계로 또는 **건너뛰기** 다음 단계로.
- **배정 고정** -- 단계 간 이동해도 같은 소유자를 카드에 유지합니다. 기본적으로 카드를 새 단계로 이동하면 그 단계의 기본 담당자로 재배정됩니다. 고정하면 현재 사람을 전체에서 책임지게 합니다.
- **완료** 카드를 완료하거나 단계에 구성된 **결과** 버튼을 선택([라우팅](#outcome-buttons-및-조건으로-카드-라우팅) 참고).
- **메모 추가** 및 카드의 **기록** 검토 -- 실행된 자동화된 작업(발송된 이메일, 대기 등)의 로그 포함.

### 대량 작업

여러 카드의 체크박스를 선택하여 함께 작업합니다. 도구 모음이 나타나 선택한 모든 카드를 **완료**, **연기**, **재배정**, 또는 다른 단계로 **이동**할 수 있습니다.

## Outcome Buttons 및 조건으로 카드 라우팅

라우팅은 카드가 단계를 떠날 때 이동할 위치를 제어합니다. 단계의 편집기를 열어 두 가지 라우팅 종류를 구성합니다.

### Outcome Buttons

결과는 해당 단계의 카드를 완료할 때 카드 서랍에 표시되는 버튼입니다. 단일 **완료** 버튼 대신 "그룹에 가입함" 또는 "관심 없음" 같은 선택지를 제공할 수 있습니다. 각 결과는:

- 카드를 **이 워크플로우의 다른 단계**로 보낼 수 있습니다.
- **카드를 다른 워크플로우로** 핸드오프하거나,
- **카드를 완료**합니다.

이것은 한 가지 결정이 사람을 다른 경로로 분기하게 합니다.

### 자동 라우팅(조건부)

자동 라우팅은 카드가 **단계에 들어올 때** (그리고 자동화된 작업 완료 후) 카드를 앞으로 이동시킵니다. 아무도 클릭하지 않아도, 사람이 조건 집합과 일치하면. 라우팅을 추가하고, 대상 단계를 선택하고, 하나 이상의 **조건**(예: 사람의 캠퍼스, 연령 또는 교인 상태)을 정의합니다. 조건이 없는 라우팅은 모두와 일치합니다.

:::info
보드에서 각 단계 열은 라우팅을 설명하는 작은 주석을 표시합니다. 예: 결과 라벨 또는 "다음과 일치하면" 다음에 대상 단계 또는 워크플로우로의 화살표.
:::

## 완전히 자동화된 단계

단계를 완전히 자동으로 실행하도록 만들 수 있습니다. 아무도 작업하지 않습니다. 단계에 **자동화된 작업**과 다음 단계를 가리키는 **자동 라우팅**(조건 없음)을 제공합니다. 카드가 들어오면 작업이 실행되고 라우팅이 즉시 앞으로 이동시킵니다. 카드는 바로 통과합니다.

:::tip
이를 **대기**와 결합: *환영 이메일 발송 → 3일 대기 → 자동으로 "개인 통화" 단계로 진행합니다.* 이메일과 타이밍이 처리되고, 자원봉사자는 인간적인 터치가 필요한 시점에만 카드를 봅니다.
:::

## 트리거

트리거는 무언가가 일어날 때 사람을 워크플로우에 자동으로 추가하므로 카드를 수동으로 추가할 필요가 없습니다. 워크플로우 보드에서 **트리거** 탭을 클릭한 후 **트리거 추가**를 클릭합니다. 두 가지 종류가 있습니다:

### 이벤트 트리거

B1의 레코드가 변경되는 즉시 실행합니다. 이벤트를 선택한 후 선택적으로 일치하는 사람만 추가되도록 **조건**을 추가합니다:

- **개인 · 생성/업데이트됨** -- 예: 상태가 *방문자*가 되는 누구든지 추가.
- **기부금 · 생성됨** -- 예: 첫 번째 또는 큰 선물을 감사 워크플로우에 추가(금액, 기금 또는 방법과 일치).
- **그룹 · 구성원 가입함** / **그룹 · 생성됨**.
- **양식 · 제출됨** -- 선택한 양식을 제출한 누구든지 추가(좋은 카드나 "신규" 카드에 효과적).

### 일정 트리거

반복 기준에 대해 반복적으로 실행합니다 -- 매일, 매주, 매월 또는 매년. 시간 기반 아웃리치에 사용합니다. 예: *교인권 기념일이 오늘인 모두* 또는 *월간 체크인*.

모든 트리거에 대해 다음을 설정할 수도 있습니다:

- 새 카드가 시작하는 **입장 단계**(첫 단계로 기본값).
- **사람당 한 번** -- 같은 사람이 트리거로 워크플로우에 두 번 추가되지 않도록.
- **활성** -- 삭제하지 않고 트리거를 켜거나 끕니다.

:::tip
**양식 · 제출됨** 트리거를 **새 방문자 팔로우업** 템플릿과 짝을 이루어 "연결 카드" 또는 "신규" 양식을 자동 팔로우업 파이프라인으로 바꿉니다.
:::

## 내 카드

자원봉사자와 직원은 모든 보드를 파고다닐 필요가 없습니다. **내 카드** 페이지(워크플로우 페이지에서 링크됨)는 현재 사용자에게 배정된 모든 워크플로우의 모든 카드를 나열합니다. 카드를 클릭하면 속한 보드가 열립니다.

## 보고서

워크플로우를 열고 **보고서**를 클릭하여 그 워크플로우에 대한 분석을 봅니다:

- **기한 초과** -- 마감일이 지난 카드의 수.
- **단계별 카드** -- 각 단계에 현재 있는 카드 수, 열 차트로 표시.
- **완료(30일)** -- 지난 30일 동안의 처리량, 선 차트로 표시.

이를 사용하여 병목 지점을 찾습니다. 예: 카드가 쌓여 진행되지 않는 단계.
