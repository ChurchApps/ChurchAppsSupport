# 서비스 계획

<div class="article-intro">

서비스 계획은 누가 사역하고 언제 사역하는지를 조직화합니다. 각 계획은 특정 날짜와 사역에 연결되어 있어 자원봉사자 팀을 주별로 조율하고 모든 예배가 완전히 인력을 갖추도록 하기가 쉬워집니다.

</div>

<div class="prereqs">
<h4>시작하기 전에</h4>

- 사역 영역에서 사역 및 팀을 설정합니다
- 자원봉사자가 [사람 디렉토리](../people/adding-people.md)에 추가되고 팀에 지정되었는지 확인합니다

</div>

## 계획에 접근하기

1. 주 메뉴에서 **Serving**으로 이동합니다.
2. 페이지 상단의 **사역 탭**을 선택합니다.
3. 해당 유형의 계획 목록을 보려면 **계획 유형**을 클릭합니다.
4. 구체적인 계획을 열려면 클릭합니다.

:::info
계획을 관리하기 위해 전체 관리자 접근이 필요하지 않습니다. 사역의 멤버인 누구든지 사역으로 이동하여 전체 관리자 권한이 필요하지 않고 자신의 사역에 대한 계획을 만들고, 편집하고, 예약할 수 있습니다. Plans Edit 역할을 가진 편집자는 모든 사역에 걸쳐 계획을 관리할 수 있습니다.
:::

## 계획 만들기

1. 계획 유형 보기에서 **New Plan**을 클릭합니다.
2. 계획에 이름을 지정하거나 날짜를 이름으로 사용합니다. 서비스의 **날짜**를 선택합니다.
3. 이전 계획에서 복사하려면 위치만 선택하거나 위치 및 지정을 선택합니다. 복사하지 않으려면 아무것도 선택하지 않습니다. 이전 계획에서 예배 순서를 복사할 수도 있습니다.
4. 계획을 저장합니다. 이제 팀 멤버를 지정하기 시작하고 [예배 순서](./service-order.md)를 구축할 수 있습니다.

## 계획 상세 페이지

계획을 열면 두 개의 탭이 표시됩니다:

- **Assignments** -- 이 계획에 지정된 팀 멤버를 관리합니다. 기존 팀의 사람을 추가하고 누가 확인했거나 아직 보류 중인지 볼 수 있습니다.
- **[Service Order](./service-order.md)** -- 예배 노래, 기도, 공지 및 설교와 같은 요소로 예배 순서를 구축합니다.

## 팀 멤버 지정

1. 계획을 열고 **Assignments** 탭으로 이동합니다.
2. **add Position**을 클릭하여 확장합니다. add a position 양식에 정보를 입력합니다. 카테고리 이름으로 원하는 카테고리를 추가합니다.
3. **People Needed**를 클릭하고 해당 직책을 채울 자원봉사자를 선택합니다.
4. **Add**를 클릭하여 팀 명단에서 멤버를 추가합니다.
5. 지정된 멤버는 할당 상태와 함께 팀 아래에 표시됩니다.
6. 자원봉사자에게 알리기를 클릭하여 B1 앱이나 이메일을 통해 알립니다.

:::tip
계획을 만들기 전에 사역 설정에서 팀을 설정합니다. 이렇게 하면 지정할 준비된 자원봉사자 풀을 가지게 됩니다.
:::

## 자원봉사자 알림

B1은 예정된 서비스 전에 자원봉사자에게 자동으로 알림을 보낼 수 있어 매주 팀을 쫓아다닐 필요가 없습니다. 알림은 확인한 사람과 아직 응답하지 않은 사람 모두 **모두 예정된 사람**에게 이메일 및 인앱/푸시 알림으로 전송됩니다. 각 알림에는 자원봉사자의 직책, 서비스 날짜, 계획 메모 및 사용자 지정 메시지가 포함됩니다.

알림 타이밍과 내용은 **계획 유형**별로 설정되므로 각 서비스 종류는 자신의 일정을 유지할 수 있습니다.

1. **Serving** 영역에서 계획 유형을 포함하는 사역을 선택합니다.
2. 계획 유형 옆의 **edit (pencil) icon**를 클릭합니다.
3. **Reminders** 섹션에서 다음을 설정합니다:
   - **Reminder days before service** -- 보낼 시간 전 일수의 쉼표로 구분된 목록, 예: `7,1,0`. 예배당일 알림을 보내려면 `0`을 사용합니다. 이 계획 유형의 알림을 해제하려면 이 필드를 비워두세요.
   - **Custom reminder message** *(선택사항)* -- 알림에 추가되는 추가 텍스트, 예: "리허설을 위해 30분 일찍 도착합니다."
4. 계획 유형을 저장합니다.

새 계획 유형은 변경할 때까지 기본적으로 각 서비스 **2일 전**에 자원봉사자에게 알림을 보냅니다.

:::tip
아직 확인하지 않은 자원봉사자는 알림 이메일 내에서 바로 **Accept** 및 **Decline** 버튼을 가져야 로그인하지 않고 응답할 수 있습니다.
:::

:::info
각 알림은 한 번 전송됩니다. 아직 연필로 표기된 계획(아직 팀에 보내지지 않음)은 알림을 트리거하지 않습니다.
:::

## 계획 유형과 그룹 연결

계획 유형 페이지의 계획 목록 아래, **Groups** 섹션을 사용하면 이 계획 유형의 계획을 멤버 포털에서 볼 수 있는 그룹을 결정할 수 있습니다. 이는 관리자 접근을 주지 않고도 적절한 팀에 예정된 서비스를 표시하는 빠른 방법입니다.

1. 계획 유형 페이지에서 **Groups** 섹션까지 아래로 스크롤합니다.
2. **Add Group**을 클릭하고 드롭다운에서 그룹을 선택합니다.
3. **Shows** 열에서 해당 그룹의 멤버가 이 계획 유형에 대해 **Past**, **Future** 또는 **Both** 계획을 봐야 하는지 선택합니다.
4. 반복하여 추가 그룹을 연결하거나 휴지통 아이콘을 클릭하여 그룹을 제거합니다.

:::info
**Standard**로 태그된 그룹만 선택기에 나타납니다. 연결된 그룹의 멤버는 자동으로 B1 멤버 포털의 [Plans](/docs/b1-church/plans/) 탭에서 이 계획 유형의 계획을 봅니다 -- 선택한 과거/미래/모두 창으로 제한됩니다.
:::

## 계획 인쇄

팀에 배포하기 위해 계획을 인쇄할 수 있습니다. 계획을 열고 예배 순서 탭을 열고 **Print** 옵션을 사용하여 할당 및 예배 순서를 포함하는 인쇄 가능한 버전을 생성합니다. 이는 리허설에서 나눠주거나 공용 공간에 게시하기에 유용합니다.

:::info
계획은 사역별로 조직됩니다. 계획을 만들거나 보기 전에 올바른 사역 탭에 있는지 확인하세요.
:::
