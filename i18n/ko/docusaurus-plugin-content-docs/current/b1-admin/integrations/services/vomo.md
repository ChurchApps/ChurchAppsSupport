# VOMO

<div class="article-intro">

VOMO는 자원봉사 참여 플랫폼입니다 — 사람들은 프로젝트에 가입하고 키오스크에서 체크인하고 시간을 축적합니다. VOMO를 자원봉사 스케줄에 사용하지만 B1을 사람 기록에 사용하는 경우 Zapier는 둘 간의 멤버십과 체크인을 동기화할 수 있습니다. 한쪽도 떨어지지 않습니다.

</div>

<div class="prereqs">
<h4>시작하기 전에</h4>

- Zapier를 노출하는 계획의 [VOMO](https://vomo.org) 계정(확실하지 않으면 VOMO 지원에 확인)
- [Zapier](https://zapier.com) 계정
- **설정 편집** 권한이 있는 B1Admin 사용자

</div>

## 연결할 수 있는 것

VOMO의 Zapier 앱은 4개 즉시 트리거와 4개 작업을 노출합니다. 가장 많은 교회가 원하는 레시피:

| 방향 | 트리거 | 작업 |
|---|---|---|
| VOMO → B1 | VOMO 멤버십(생성) | B1: 사람 찾기 → 사람 생성(새로운 경우) |
| VOMO → B1 | VOMO 키오스크 체크인 | B1: 그룹 멤버를 "현재 예배" 그룹에 추가 또는 출석으로 기록 |
| B1 → VOMO | B1 `person.created` | VOMO: 조직자 찾기(이메일로); 그 외 맞춤 단계 |
| 어느 쪽이든 | VOMO 참여(가입) | B1: 프로젝트 특정 그룹에 그룹 멤버 추가 |

VOMO 작업은 **프로젝트 초안** 및 기존 조직자/프로젝트 **찾기**로 제한됩니다 — "이 사람을 VOMO 프로젝트에 추가" 작업은 없습니다. 흥미로운 연결은 대부분 VOMO → B1입니다.

## 설정

### 1. B1 API 키 발급

**설정 → 개발자 → API 키 → 새 API 키**. 범위:

- `people:read`, `people:write` — 자원봉사자를 B1 사람으로 찾아보고 생성하기 위해
- `groups:write` — 자원봉사 팀 그룹에 추가하기 위해
- (선택 사항) `attendance:write` — 키오스크 체크인을 출석으로 취급할 경우

### 2. 멤버십 동기화 Zap 빌드

1. **트리거** — VOMO: 멤버십(event = `created`).
2. **작업** — B1.church: 사람 찾기, 이메일로 매칭.
3. **필터 / 경로** — 발견/발견되지 않음에 분기:
   - 발견되지 않음 → B1.church: 사람 생성, 그런 다음 적절한 자원봉사 그룹에 그룹 멤버 추가.
   - 발견됨 → B1.church: 직접 그룹 멤버 추가.
4. 켜세요. 새로운 VOMO 자원봉사자는 이제 올바른 그룹 멤버십으로 B1에 나타납니다.

### 3. (선택 사항) 키오스크 체크인 Zap 빌드

1. **트리거** — VOMO: 키오스크
2. **작업** — B1.church: 사람 찾기(이메일로)
3. **작업** — 선택 사항:
   - *출석으로 취급할 경우* — Zapier 웹훅 POST를 B1의 `/attendance/visits` 엔드포인트로(B1의 Zapier 앱은 아직 1등급 *기록 출석* 작업이 없음). B1 [API 키](/docs/developer/api/api-keys)가 `Authorization: Bearer cak_…` 헤더로 가집니다.
   - *그룹 멤버십으로 취급할 경우* — B1.church: 그룹 멤버를 "현재 예배(오늘)" 그룹에 추가, 그리고 나중에 하루 중 예약된 정리를 통해 제거하는 두 번째 Zap.

## 일반적인 레시피

### 프로젝트별 그룹 동기화

- **트리거** — VOMO: 참여(생성)
- **작업** — Zapier로 필터 — 프로젝트 id를 수행한 다음
- **작업** — B1.church: 그룹 멤버를 B1 그룹(이름이 VOMO 프로젝트를 미러하는)에 추가

VOMO 프로젝트가 끝나면 B1 그룹을 수동으로 지웁니다(*참여 삭제* 트리거와 쌍을 이루면 제거).

### "가입해주셔서 감사합니다" 텍스트를 SMS를 통해 전송

같은 Zap에서 VOMO 참여 → Clearstream 텍스트 전송 또는 Text In Church 메시지 전송으로 연결합니다. 둘 다 1등급 Zapier 작업을 가지고 있습니다 — [Clearstream](./clearstream) 및 [Text In Church](./text-in-church)를 참조하세요.

### 낙오 감지

매일 Zapier *스케줄* 트리거를 실행하여 이번 달에 자원봉사 팀에 가입한 B1 사람 목록의 VOMO에서 조직자를 찾습니다 — VOMO가 "발견되지 않음"을 반환하면 VOMO를 활성화하지 않았으며 밀어붙여야 합니다.

## 제한 및 주의사항

- **이메일이 조인 키입니다.** VOMO의 페이로드는 사용자 이메일을 노출하지만 B1 사람 id는 없습니다. 각 시스템에서 다른 이메일을 사용하는 기부자는 중복을 생성합니다.
- **VOMO의 Zapier 앱에서 "프로젝트에 추가" 작업이 없습니다.** B1 → VOMO 프로젝트 등록이 필요한 경우 *Zapier 웹훅* 단계에서 VOMO의 REST API로 POST하면 맞춤 통합입니다.
- **VOMO의 무료 / 낮은 계층은 Zapier를 포함하지 않을 수 있습니다.** 약속을 하기 전에 VOMO 지원으로 확인합니다.

## 문제 해결

- **트리거가 실행되지 않음** — VOMO의 즉시 트리거는 API 토큰을 유효하게 유지해야 합니다. Zap을 다시 테스트하고 토큰이 회전된 경우 VOMO를 다시 연결합니다.
- **B1 *그룹 멤버 추가*가 422로 실패함** — 작업의 그룹 id가 존재하지 않습니다. **B1Admin → 그룹**, 그룹을 클릭하고 URL의 id 세그먼트를 Zap 단계로 복사합니다.
- **단일 VOMO 자원봉사자에서 중복 B1 사람** — 그들은 아마도 B1에서 이미 가진 것보다 다른 이메일로 가입했을 것입니다. 이메일을 표준화하거나 Zapier *경로*를 추가하여 생성 전에 전화로도 검색합니다.
