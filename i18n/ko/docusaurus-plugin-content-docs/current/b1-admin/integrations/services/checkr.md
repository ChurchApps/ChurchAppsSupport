# Checkr

<div class="article-intro">

[Checkr](https://checkr.com)는 직원 및 자원봉사자를 위한 배경 검사를 실행합니다 — 어린이 또는 청소년 프로그램을 운영하는 거의 모든 교회에 필요합니다. Checkr에는 Zapier 앱이 없지만 [Make.com의 Checkr 통합](https://www.make.com/en/integrations/checkr)은 검증되며 B1 이벤트에서 검사를 시작하는 데 필요한 작업을 노출합니다.

</div>

<div class="prereqs">
<h4>시작하기 전에</h4>

- API 액세스 및 최소 하나의 검사 패키지가 구성된 [Checkr](https://checkr.com) 계정
- [Make](https://www.make.com) 계정
- **설정 편집** 권한이 있는 B1Admin 사용자

</div>

## 연결할 수 있는 것

Make의 Checkr 앱은 1개 트리거와 6개 작업을 노출합니다:

| 방향 | B1 / Make 트리거 | 작업 |
|---|---|---|
| B1 → Checkr | B1 `group.member.added`(자원봉사자 그룹으로 필터링) | Checkr: 지원자 생성 → 배경 검사 초대 생성 |
| Checkr → B1 | Checkr 웹훅(초대 / 보고서 이벤트) | B1: 사람 레코드 업데이트(예: "Checkr 통과" 태그 지정) |

Make의 Checkr 작업: 지원자 생성, 배경 검사 초대 생성, 지원자 가져오기, 보고서 가져오기, 보고서의 ETA 가져오기, 초대 가져오기. 또한 4개의 검색 모듈.

## 설정

### 1. B1 API 키 발급

**설정 → 개발자 → API 키 → 새 API 키**:

- `settings:write` — 트리거 웹훅에 필요
- `people:read` — 검사를 시작할 때 사람의 이름/이메일을 찾아볼 때 필요
- (선택 사항) `people:write` — 보고서 상태를 맞춤 필드나 태그로 다시 쓰려면

### 2. Make에서 "자원봉사자 가입 시 검사 시작" 시나리오 빌드

1. **트리거** — B1.church: 이벤트 감시(`group.member.added`).
2. **필터** — `data.groupId`가 "어린이 자원봉사자"(또는 동등한) 그룹과 일치하는 경우에만 계속합니다.
3. **작업** — B1.church: 사람 찾기(`data.personId`로) 이메일 + 이름/성을 가져옵니다.
4. **작업** — Checkr: 지원자 생성. 3단계에서 이름/성/이메일을 매핑합니다.
5. **작업** — Checkr: 배경 검사 초대 생성. 새 지원자 id를 4단계에서 *candidate_id* 필드로 매핑합니다. 검사 패키지(예: `tasker_standard` 또는 계정이 노출하는 것)를 선택합니다.
6. (선택 사항) **작업** — Slack: 안전 사역 조정자에 검사가 시작되었음을 알립니다.

시나리오를 켭니다. 대상 그룹의 새 자원봉사자는 자동으로 Checkr 초대를 이메일로 받습니다. 그들은 휴대폰이나 노트북에서 완료하고, Checkr은 검사를 실행합니다.

### 3. (선택 사항) 보고서를 받으세요

1. **트리거** — Checkr: 이벤트 감시(웹훅). Make는 활성화 시 Checkr 웹훅을 등록합니다.
2. **필터** — `event_type = report.completed`인 경우에만 계속합니다.
3. **작업** — Checkr: 보고서 가져오기(웹훅에서 보고서 id 사용).
4. **작업** — B1.church: 사람 찾기(지원자 이메일로).
5. **작업** — 조건부 Slack / 이메일: `clear` / `consider` / `suspended` 상태로 조정자에 알립니다.

참고: B1은 현재 "배경 검사 상태" 필드를 내장하지 않습니다. 실용적인 옵션은 (a) 결과를 개인 Slack 채널에 게시하여 검토, (b) Google 시트에 기록하여 감사, 또는 (c) 사람을 `clear` 시 "통과한 자원봉사자" B1 그룹에 추가합니다.

## 일반적인 레시피

### 2년마다 자원봉사자 다시 검사

위의 항목과 Make 스케줄 트리거를 쌍으로 만듭니다:

- **트리거** — Make: 스케줄(월별)
- **작업** — B1.church: "통과한 자원봉사자" 그룹 멤버 나열
- **작업** — Make로 필터: 통과 날짜가 22개월보다 오래
- **작업** — Checkr: 배경 검사 초대 생성(초기 흐름과 동일)

### 1단계 액세스 차단(검사 완료까지)

교회에서 B1 그룹 멤버십을 사용하여 액세스를 제한하는 경우(예: "통과" 그룹 멤버만 예배 스케줄에 나타남), 새 자원봉사자는 Checkr `report.completed` 이벤트가 그들을 뒤집을 때까지 대기 그룹에 유지합니다.

## 제한 및 주의사항

- **Checkr은 미국 전용입니다** — 대부분의 검사 패키지의 경우. 호주, 영국 및 캐나다 교회는 대안을 필요로 합니다.
- **가격 책정**은 검사당입니다 — Make에서 모든 초대 생성은 실제 검사를 소모합니다. 프로덕션 B1을 생성하기 전에 Checkr의 샌드박스 / 스테이징 계정에서 테스트합니다(Make의 Checkr 앱은 연결에서 전달하는 자격 증명을 존중하므로 자격 증명 교환이 샌드박스/라이브로 전환됨).
- **Checkr API 액세스는 계획 제한입니다.** 작은 Checkr 계정은 UI 전용 계층에 있을 수 있습니다. Checkr에 문의하여 API를 활성화하세요.

## 문제 해결

- **지원자 생성이 `403`으로 실패함** — Checkr API 토큰이 읽기 전용이거나 올바른 계정 권한이 없습니다. Checkr 대시보드에서 쓰기 범위로 다시 발급하세요.
- **초대가 도착하지 않음** — 3단계에서 지원자의 이메일을 확인합니다. B1은 해당 사람에 대해 비어 있을 수 있습니다. Checkr 단계 전에 이메일 필수 필터를 추가합니다.
- **웹훅 트리거가 실행되지 않음** — Checkr의 웹훅 등록은 Make 계정이 아웃바운드 웹훅을 지원하는 유료 계층에 없으면 조용히 실패할 수 있습니다. Checkr 대시보드 *웹훅* 페이지에서 Make의 URL이 나열되어 있는지 확인합니다.
