# 저장된 목록

<div class="article-intro">

**저장된 목록**을 사용하면 검색 쿼리를 이름 아래에 저장하고 언제든지 한 번의 클릭으로 다시 실행할 수 있습니다. 목록을 한 번 작성합니다. "Campus North의 활성 멤버," "지난 90일의 방문자" 또는 기타 고급 필터. 사이드바에 유지되므로 쿼리를 다시 작성할 필요가 없습니다.

</div>

<div class="prereqs">
<h4>시작하기 전에</h4>

- People 섹션에서 최소 1번의 검색을 실행하여 저장할 결과를 가집니다. [Searching People](./searching-people.md) 또는 [AI Search](./ai-search.md)을 참조하세요.

</div>

## 검색을 목록으로 저장

1. **People** 페이지에서 검색을 실행합니다. 빠른 검색, AI 검색 또는 고급 필터 검색.
2. 결과가 나타나면 **Save as List** 버튼(책갈피 아이콘)이 People 페이지 헤더에 표시됩니다.
3. **Save as List**를 클릭합니다.
4. 목록의 **Name**을 입력합니다(필수).
5. 선택적으로 **Category**를 입력하여 사이드바의 관련 목록을 함께 그룹화합니다(예: "Outreach" 또는 "Membership").
6. **Save**를 클릭합니다.

목록이 이제 교회 계정에 저장되고 People 페이지의 왼쪽에 있는 **Saved Lists** 패널에 표시됩니다.

:::info
목록은 교회 전체에서 공유됩니다. People 액세스 권한이 있는 모든 사람이 목록을 보고 로드할 수 있습니다. 적절한 권한이 있는 직원만 목록을 이름 바꾸거나 삭제할 수 있습니다.
:::

## 저장된 목록 로드

People 페이지의 왼쪽에 있는 **Saved Lists** 패널에서 목록 이름을 클릭합니다. 검색은 저장된 쿼리로 즉시 다시 실행되어 결과 표를 업데이트합니다.

목록은 카테고리별로 그룹화되고(할당한 경우) 분류되지 않은 목록이 마지막에 표시됩니다.

:::tip
목록은 스냅샷이 아닌 **live queries**입니다. 목록을 로드할 때마다 검색이 현재 데이터에 대해 다시 실행되므로 이제 기준과 일치하는 새로운 사람이 나타납니다.
:::

## 목록 이름 바꾸기 및 삭제

manage 권한이 있는 직원은 Saved Lists 패널에서 직접 목록을 이름 바꾸거나 삭제할 수 있습니다.

- 목록 이름 옆의 **pencil icon**을 클릭하여 이름을 변경합니다.
- **trash icon**을 클릭하여 삭제합니다(확인을 요청받습니다).

## 사용 사례

| 시나리오 | 목록을 빌드하는 방법 |
|---|---|
| 모든 방문자 | Advanced search: Membership Status = Visitor |
| 특정 캠퍼스의 멤버 | Advanced search: Campus = [your campus] |
| 이메일 주소가 없는 사람 | Advanced search: Email is empty |
| 만료된 신원조회가 있는 자원봉사자 | Advanced search: [Custom Field](../settings/custom-fields.md) "Background check expires" is before today |
| AI 질문의 결과 | AI Search에서 질문을 하고 저장하세요 |
