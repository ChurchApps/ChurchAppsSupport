# 데이터 가져오기

<div class="article-intro">

B1 Admin을 사용하면 기존 교인 데이터를 시스템에 쉽게 가져올 수 있습니다. 다른 교회 관리 플랫폼에서 마이그레이션하든 스프레드시트에서 기록을 불러오든, 가져오기 도구를 사용하면 모든 사람을 수동으로 입력할 필요가 없습니다. CSV 파일에서 가져오거나 Breeze ChMS에서 직접 마이그레이션할 수 있습니다.

</div>

<div class="prereqs">
<h4>시작하기 전에</h4>

- **설정**에 접근할 수 있는 활성 B1 Admin 계정이 필요합니다. 접근 권한이 확실하지 않은 경우 [역할 및 권한](roles-permissions.md)을 참조하세요.
- 스프레드시트 또는 이전 시스템에서 내보낸 교인 데이터를 준비하세요.
- Breeze에서 마이그레이션하는 경우, 먼저 Breeze에서 People, Tags, Contributions 파일을 내보냈는지 확인하세요.

</div>

## CSV에서 가져오기

스프레드시트나 다른 시스템에 교인 데이터가 있는 경우 CSV(쉼표로 구분된 값) 파일을 사용하여 가져올 수 있습니다.

1. 왼쪽 사이드바에서 **설정**으로 이동합니다.
2. 상단 내비게이션에서 **가져오기/내보내기**를 클릭합니다.
3. **데이터 소스** 드롭다운에서 **B1 Import Zip**을 선택합니다.
4. 예상 형식을 확인할 수 있도록 **샘플 파일 다운로드** 링크를 클릭합니다.
5. 샘플 `people.csv` 파일을 열고 샘플 데이터를 자신의 데이터로 교체합니다. 헤더 행은 그대로 유지하세요.
6. 교인 사진이 있는 경우, CSV의 `importKey` 번호와 일치하도록 이름을 지정한 400x300px 이미지를 폴더에 추가합니다.
7. 편집한 파일을 zip 파일로 압축합니다.
8. B1 Admin으로 돌아가서 **업로드**를 클릭하고 zip 파일을 선택합니다.
9. 데이터 미리보기를 확인하고 **대상으로 계속**을 클릭합니다.
10. **B1 Database**가 선택되어 있는지 확인하고, 가져오기 요약을 검토한 후 **전송 시작**을 클릭합니다.
11. 가져오기가 완료될 때까지 기다린 후 **B1으로 이동**을 클릭하여 대시보드로 돌아갑니다.

:::tip
항상 샘플 파일을 먼저 다운로드하여 검토하세요. 예상 열 형식에 맞추면 가져오기 오류를 방지할 수 있습니다.
:::

:::warning
데이터를 가져오면 데이터베이스에 새 기록이 추가됩니다. 같은 파일을 두 번 가져오면 중복 항목이 생길 수 있습니다. 전송을 시작하기 전에 파일을 다시 한 번 확인하세요.
:::

## Breeze ChMS에서 가져오기

Breeze에서 마이그레이션하는 경우, B1에는 변환을 자동으로 처리하는 전용 가져오기 옵션이 있습니다.

1. Breeze에서 **Settings**로 이동하고 왼쪽 사이드바에서 **Export**를 클릭합니다.
2. **People**, **Tags**, **Contributions** 세 가지 파일을 내보냅니다.
3. 세 개의 내보낸 파일을 모두 선택하고 마우스 오른쪽 버튼을 클릭하여 하나의 zip 파일로 압축합니다.
4. B1 Admin에서 **설정** 다음 **가져오기/내보내기**로 이동합니다.
5. **데이터 소스** 드롭다운에서 **Breeze Import Zip**을 선택합니다.
6. zip 파일을 업로드하고 화면의 안내에 따라 가져오기를 검토하고 완료합니다.

:::info
Breeze 가져오기는 교인, 사진, 그룹, 헌금, 출석, 양식 등을 전송합니다 -- 한 번에 완전한 마이그레이션을 제공합니다.
:::

## 가져오기 후

가져오기가 완료되면 몇 분간 데이터를 확인하세요:

1. [교인 관리](../people/adding-people.md) 페이지를 탐색하며 몇 개의 프로필을 점검합니다.
2. 이름, 이메일, 전화번호, 주소가 정확하게 전송되었는지 확인합니다.
3. 가구 연결이 올바른지 확인합니다.
4. 가져온 [그룹](../groups/creating-groups.md)이나 태그를 검토합니다.

문제가 발견되면 교인 관리 페이지에서 직접 개별 프로필을 편집할 수 있습니다. 또한 언제든지 [데이터를 내보내](exporting-data.md) 백업을 만들 수 있습니다.
