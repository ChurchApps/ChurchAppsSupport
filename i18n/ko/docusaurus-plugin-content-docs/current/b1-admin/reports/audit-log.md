# 감사 로그

<div class="article-intro">

감사 로그는 교회 관리 시스템 전체의 모든 중요한 작업과 변경을 추적합니다. 이를 사용하여 로그인 활동을 검토하고, 사람 기록에 변경한 사람을 추적하고, 권한 업데이트를 모니터링하고, 팀 전체에 책임을 유지합니다.

</div>

<div class="prereqs">
<h4>시작하기 전에</h4>

- 서버 관리 액세스 권한이 있는 B1 Admin 계정
- 감사 로그를 찾으려면 **설정**으로 이동합니다

</div>

## 감사 로그 보기

1. B1 Admin의 **설정**으로 이동합니다.
2. **감사 로그**를 선택합니다.
3. 로그는 다음 열이 있는 표에 최근 항목을 표시합니다:
   - **날짜** -- 작업이 발생했을 때입니다.
   - **범주** -- 작업 유형입니다(빠른 스캔을 위해 색상 코딩됨).
   - **작업** -- 수행된 것입니다(예: 만들기, 업데이트, 삭제, login_success).
   - **엔터티** -- 영향을 받은 기록의 유형 및 ID입니다.
   - **IP 주소** -- 작업을 수행한 사용자의 IP 주소입니다.
   - **세부 정보** -- 수행된 특정 변경 사항의 요약입니다.

## 로그 필터링

페이지 상단의 필터를 사용하여 결과를 좁히세요:

- **범주** -- 작업 유형별로 필터링:
  - **모든 범주** -- 모든 것을 표시합니다.
  - **로그인** -- 로그인 성공 및 실패입니다.
  - **사람** -- 사람 기록 만들기, 업데이트, 삭제입니다.
  - **권한** -- 권한 부여 및 취소입니다.
  - **기부금** -- 기부금 기록 변경입니다.
  - **그룹** -- 그룹 관리 작업입니다.
  - **양식** -- 양식 제출 활동입니다.
  - **설정** -- 구성 변경입니다.
- **시작 날짜** -- 이 날짜 이후의 항목을 표시합니다.
- **종료 날짜** -- 이 날짜까지의 항목을 표시합니다.

필터를 설정한 후 **검색**을 클릭하여 결과를 업데이트합니다.

## 범주 이해

각 범주는 빠른 식별을 위해 색상 코딩됩니다:

- **로그인** -- 파란 칩입니다. 성공 및 실패한 로그인 시도를 추적합니다.
- **사람** -- 보라색 칩입니다. 사람 기록 만들기, 업데이트, 삭제를 추적합니다.
- **권한** -- 빨간색 칩입니다. 액세스 권한을 부여하거나 취소할 때를 추적합니다.
- **기부금** -- 녹색 칩입니다. 기부금 기록 변경을 추적합니다.
- **그룹** -- 회색 칩입니다. 그룹 관리 작업을 추적합니다.
- **양식** -- 주황색 칩입니다. 양식 제출 활동을 추적합니다.
- **설정** -- 노란색 칩입니다. 구성 변경을 추적합니다.

## 로그 내보내기

로그 항목이 표시되면 **CSV 다운로드** 버튼이 나타납니다. 이를 클릭하여 현재 필터링된 결과를 오프라인 검토 또는 기록 유지를 위해 스프레드시트로 내보냅니다.

## 페이지 매김

표 하단의 페이지 매김 제어를 사용하여 결과를 탐색합니다. 페이지당 25, 50, 100개 항목을 표시할 수 있습니다.

:::info
감사 로그 항목은 자동으로 1년 동안 유지됩니다. 365일보다 오래된 항목은 시스템을 효율적으로 유지하기 위해 제거됩니다.
:::

:::tip
정기적으로 감사 로그를 검토합니다. 특히 새 팀 회원을 온보딩한 후 또는 상당한 구성 변경을 한 후입니다. 예상치 못한 활동을 조기에 식별하는 데 도움이 됩니다.
:::
