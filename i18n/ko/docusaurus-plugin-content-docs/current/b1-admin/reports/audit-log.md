# 감사 로그

<div class="article-intro">

감사 로그는 교회 관리 시스템 전반에 걸친 모든 중요한 작업 및 변경 사항을 추적합니다. 이를 사용하여 로그인 활동을 검토하고, 사람 기록 변경을 추적하고, 권한 업데이트를 모니터링하고, 팀 전체에서 책임을 유지합니다.

</div>

<div class="prereqs">
<h4>시작하기 전에</h4>

- 서버 관리자 액세스 권한이 있는 B1 Admin 계정
- B1 Admin의 **설정**으로 이동하여 감사 로그 찾기

</div>

## 감사 로그 보기

1. B1 Admin의 **설정**으로 이동합니다.
2. **감사 로그**를 선택합니다.
3. 로그는 다음 열이 있는 테이블에 최근 항목을 표시합니다:
   - **날짜** -- 작업이 발생한 시기입니다.
   - **카테고리** -- 작업의 유형(빠른 검색을 위해 색상 코드 처리됨).
   - **작업** -- 수행된 작업(예: create, update, delete, login_success).
   - **엔티티** -- 영향을 받은 기록의 유형 및 ID입니다.
   - **IP 주소** -- 작업을 수행한 사용자의 IP 주소입니다.
   - **세부 정보** -- 수행된 특정 변경 사항 요약입니다.

## 로그 필터링

페이지 상단의 필터를 사용하여 결과를 좁힙니다:

- **카테고리** -- 작업 유형별 필터:
  - **모든 카테고리** -- 모두 표시합니다.
  - **로그인** -- 로그인 성공 및 실패입니다.
  - **사람** -- 사람 기록 생성, 업데이트 또는 삭제입니다.
  - **권한** -- 권한 부여 및 취소입니다.
  - **헌금** -- 헌금 기록 변경입니다.
  - **그룹** -- 그룹 관리 작업입니다.
  - **양식** -- 양식 제출 활동입니다.
  - **설정** -- 구성 변경입니다.
- **시작 날짜** -- 이 날짜 이후의 항목을 표시합니다.
- **종료 날짜** -- 이 날짜까지의 항목을 표시합니다.

필터를 설정한 후 **검색**을 클릭하여 결과를 업데이트합니다.

## 카테고리 이해

각 카테고리는 빠른 식별을 위해 색상 코드 처리됩니다:

- **로그인** -- 파란색 칩입니다. 로그인 시도 성공 및 실패를 추적합니다.
- **사람** -- 보라색 칩입니다. 사람 기록 생성, 업데이트 및 삭제를 추적합니다.
- **권한** -- 빨간색 칩입니다. 액세스 권한이 부여되거나 취소되는 시기를 추적합니다.
- **헌금** -- 녹색 칩입니다. 헌금 기록 변경을 추적합니다.
- **그룹** -- 회색 칩입니다. 그룹 관리 작업을 추적합니다.
- **양식** -- 주황색 칩입니다. 양식 제출 활동을 추적합니다.
- **설정** -- 노란색 칩입니다. 구성 변경을 추적합니다.

## 로그 내보내기

로그 항목이 표시되면 **CSV 다운로드** 버튼이 나타납니다. 이를 클릭하여 현재 필터링된 결과를 오프라인 검토 또는 기록 보관용 스프레드시트로 내보냅니다.

## 페이지 나누기

테이블 하단의 페이지 나누기 컨트롤을 사용하여 결과를 탐색합니다. 페이지당 25, 50 또는 100개 항목을 표시할 수 있습니다.

:::info
감사 로그 항목은 자동으로 1년 동안 보존됩니다. 365일보다 오래된 항목은 시스템을 효율적으로 유지하기 위해 제거됩니다.
:::

:::tip
특히 새 팀 구성원을 온보딩하거나 중요한 구성 변경을 한 후에 정기적으로 감사 로그를 검토하세요. 예상 외의 활동을 조기에 식별하는 데 도움이 됩니다.
:::
