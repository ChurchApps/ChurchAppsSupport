# Stripe 가져오기

<div class="article-intro">

Stripe를 통해 온라인 헌금을 받는 경우, Stripe 가져오기 도구를 사용하면 해당 거래를 B1 Admin으로 가져와 모든 헌금 데이터를 한 곳에서 관리할 수 있습니다. 이는 자동으로 동기화되지 않은 거래를 포착하는 데 특히 유용합니다.

</div>

<div class="prereqs">
<h4>시작하기 전에</h4>

- [온라인 헌금 설정](online-giving-setup.md)을 완료하여 Stripe 계정을 B1 Admin에 연결하세요
- 가져오려는 날짜 범위에 해당하는 헌금이 Stripe 대시보드에 있는지 확인하세요

</div>

## 작동 방식

가져오기 프로세스는 두 단계 워크플로우를 사용합니다: 먼저 가져올 내용을 미리 보고, 그런 다음 가져오기를 확인합니다. 이 사전 실행 방식을 통해 데이터가 생성되기 전에 모든 것을 검토할 수 있습니다.

## 거래 가져오기

1. **B1 Admin**에서 **헌금 > 일괄 처리**로 이동합니다.
2. 일괄 처리 페이지 하단의 **Stripe 가져오기** 링크를 클릭합니다.
3. 가져오려는 거래의 **날짜 범위**를 선택합니다.
4. **미리보기**를 클릭하여 사전 확인을 실행합니다.

## 미리보기 검토

미리보기에는 Stripe의 각 거래와 상태 표시가 함께 표시됩니다:

- **신규** -- 이 거래는 아직 가져오지 않았으며 진행하면 포함됩니다.
- **이미 가져옴** -- 이 거래는 B1 Admin에 이미 존재하므로 건너뜁니다.
- **건너뜀** -- 이 거래는 다른 이유로 제외되었습니다 (예: 환불 또는 실패한 결제).

상단의 요약 섹션에는 각 카테고리의 총 거래 수와 관련 금액이 표시됩니다.

:::info
미리보기 단계에서는 어떤 기록도 생성되지 않습니다. 커밋하기 전에 무엇이 발생할지 확인하기 위한 읽기 전용 검사입니다.
:::

## 가져오기 완료

1. 미리보기 결과와 요약 합계를 검토합니다.
2. **누락 항목 가져오기**를 클릭하면 **신규**로 표시된 모든 거래가 가져와집니다.
3. 가져오기가 완료되면 각 거래 옆의 상태 칩이 결과를 표시하도록 업데이트됩니다.

## Stripe 가져오기 활용 팁

- 기록을 최신 상태로 유지하기 위해 정기적으로 (주간 또는 월간) 가져오기를 실행하세요.
- 거래가 **이미 가져옴**으로 표시되면 B1 Admin에 이미 일치하는 기록이 있다는 의미입니다 -- 별도의 조치가 필요하지 않습니다.
- 특정 거래를 찾고 있다면 날짜 범위 필터를 사용하여 특정 기간에 집중하세요.

:::tip
가져오기 후 [일괄 처리](batches.md) 페이지를 방문하여 가져온 헌금이 올바르게 표시되고 합계가 Stripe 대시보드에서 보이는 것과 일치하는지 확인하세요.
:::
