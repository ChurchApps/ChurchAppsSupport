# 헌금 일괄 처리

<div class="article-intro">

일괄 처리는 추적과 정산을 쉽게 하기 위해 헌금을 함께 그룹으로 묶습니다. 일반적인 일괄 처리는 주일 헌금이나 특별 행사와 같은 단일 수집을 나타냅니다. 일괄 처리를 사용하면 체계적으로 관리할 수 있고, 기록이 실제 입금액과 일치하는지 간단히 확인할 수 있습니다.

</div>

<div class="prereqs">
<h4>시작하기 전에</h4>

- 헌금 기록 시 사용할 수 있도록 [기금을 설정](funds.md)했는지 확인하세요
- B1 Admin의 **헌금** 섹션에 접근 권한이 필요합니다

</div>

## 일괄 처리 페이지

**헌금 > 일괄 처리**로 이동하면 모든 일괄 처리 목록을 볼 수 있습니다. 각 행에는 다음이 표시됩니다:

- **이름** -- 일괄 처리에 지정한 레이블
- **날짜** -- 수집 날짜
- **헌금** -- 일괄 처리 내 개별 헌금 수
- **합계** -- 총 금액

상단 헤더에는 총 일괄 처리 수, 모든 일괄 처리의 총 헌금 수, 전체 금액을 포함한 요약 통계가 표시됩니다.

## 새 일괄 처리 만들기

1. 페이지 상단의 **일괄 처리 추가**를 클릭합니다.
2. 설명적인 이름을 입력합니다 (예: "주일 헌금 - 2월 9일").
3. 수집 날짜를 선택합니다.
4. **저장**을 클릭합니다.

새 일괄 처리가 목록에 나타나며 헌금을 추가할 준비가 됩니다.

## 일괄 처리 작업

- **헌금 보기** -- 일괄 처리 이름을 클릭하면 열리며 포함된 모든 개별 헌금을 볼 수 있습니다. 여기서 헌금을 추가, 편집 또는 제거할 수 있습니다.
- **일괄 처리 세부 정보 편집** -- 일괄 처리 행의 **편집** 버튼을 클릭하여 이름이나 날짜를 변경합니다.
- **정렬** -- 열 헤더를 사용하여 이름이나 날짜별로 일괄 처리를 정렬합니다.
- **내보내기** -- **CSV로 내보내기**를 클릭하여 일괄 처리 목록을 스프레드시트로 다운로드합니다.

:::tip
일괄 처리의 이름을 일관되게 지정하면 나중에 찾기 쉽습니다. 날짜와 수집 유형을 포함하면 (예: "주일 오전 - 2025-02-09") 목록이 늘어나도 체계적으로 관리할 수 있습니다.
:::
