# 헌금 기록

<div class="article-intro">

B1 Admin에서 헌금을 기록하는 것은 일괄 처리 시스템을 통해 이루어집니다. 수집(예: 주일 헌금)을 나타내는 일괄 처리를 만든 다음 해당 일괄 처리에 개별 헌금을 추가합니다. 이를 통해 헌금 기록을 체계적으로 관리하고 쉽게 정산할 수 있습니다.

</div>

<div class="prereqs">
<h4>시작하기 전에</h4>

- 올바른 카테고리에 헌금을 할당할 수 있도록 [기금](funds.md)을 설정하세요
- 입력할 헌금을 담을 [일괄 처리](batches.md)를 만드세요
- 헌금 입력 시 기부자를 검색할 수 있도록 [교인 명부](../people/adding-people.md)에 기부자가 등록되어 있는지 확인하세요

</div>

## 일괄 처리 만들기 및 헌금 추가

1. **B1 Admin**에서 사이드바의 **헌금**을 클릭한 다음 **일괄 처리**를 클릭합니다.
2. **일괄 처리 추가**를 클릭합니다.
3. 일괄 처리 이름을 입력하고 (예: "주일 헌금 - 1월 5일") 날짜를 선택합니다. **저장**을 클릭합니다.
4. 새 일괄 처리가 헌금 0건, $0.00으로 목록에 나타납니다.
5. **일괄 처리 이름**을 클릭하여 엽니다.

## 개별 헌금 입력

1. 일괄 처리 상세 페이지에서 **검색 필드**에 기부자 이름을 입력하여 찾습니다.
2. 사람을 선택하면 **날짜**, **결제 방법**, **기금**, **금액**, **수표 번호** 필드가 있는 헌금 입력 양식이 나타납니다.
3. 세부 정보를 입력하고 **헌금 추가**를 클릭합니다.
4. 아래 테이블에 헌금이 추가되고, 양식이 다음 입력을 위해 초기화됩니다.

:::tip
일괄 처리 페이지를 벗어나지 않고 여러 헌금을 연속으로 빠르게 입력할 수 있습니다. 각 입력 후 양식이 초기화되므로 수표나 봉투 더미를 효율적으로 처리할 수 있습니다.
:::

## 여러 기금에 헌금 분할

한 기부자가 한 번의 거래로 둘 이상의 기금에 헌금하는 경우가 있습니다. 이를 처리하려면:

1. 헌금 행의 **편집** 버튼을 클릭합니다.
2. 편집 양식에서 각 기금에 금액을 추가합니다. 총액은 개별 기금 금액에서 자동으로 계산됩니다.
3. **저장**을 클릭하여 헌금을 업데이트합니다.

:::info
기금별 헌금 분할은 기부자가 일반 기금과 선교 기금 등 여러 목적으로 지정된 단일 수표를 작성할 때 일반적입니다.
:::

## 헌금 편집 또는 제거

헌금을 편집하려면 일괄 처리 내 해당 행의 **편집** 버튼을 클릭합니다. 날짜, 금액, 기금, 결제 방법 또는 기타 세부 정보를 변경할 수 있습니다. 완료되면 **저장**을 클릭합니다.

:::tip
일괄 처리 페이지 헤더는 항목을 추가하거나 편집할 때 자동으로 총 헌금 수와 합산 금액을 업데이트합니다. 이를 사용하여 입금 전표와 정산하세요.
:::
