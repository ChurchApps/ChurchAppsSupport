# 페이지 편집기 사용하기

<div class="article-intro">

B1 페이지 편집기는 코드를 작성하지 않고 교회 웹사이트 페이지를 디자인할 수 있게 해주는 시각적 드래그 앤 드롭 빌더입니다. 섹션과 콘텐츠 블록을 추가하고, 스타일을 사용자 정의하고, 작업을 미리보고, 변경 사항을 취소할 수 있습니다. 모두 브라우저 내에서 말입니다.

</div>

<div class="prereqs">
<h4>시작하기 전에</h4>

- 웹사이트를 구성하려면 [초기 설정](initial-setup)을 완료합니다
- [페이지 관리](managing-pages)에서 최소 1개의 페이지를 만듭니다
- 편집기에 액세스하려면 **content.edit** 권한이 필요합니다

</div>

## 편집기 열기

1. B1 Admin에서 왼쪽 메뉴의 **웹사이트**를 클릭합니다.
2. 페이지 표에서 편집하려는 페이지를 찾고 **편집**을 클릭합니다.

편집기가 전체 화면 모드로 열립니다. 왼쪽 패널은 페이지 구조와 사용 가능한 콘텐츠 요소를 보여주고, 중앙 영역은 페이지의 라이브 미리보기를 보여줍니다.

:::info
편집기는 항상 라이트 모드로 표시되며, B1 Admin 테마 설정과 무관합니다. 이는 미리보기가 웹사이트 방문자에게 표시되는 모습과 정확히 일치함을 보장합니다.
:::

## 페이지 구조: 섹션과 요소

모든 페이지는 두 가지 수준으로 구축됩니다:

- **섹션** -- 페이지를 수평 밴드로 나누는 최상위 컨테이너입니다(예: 히어로 섹션, 콘텐츠 블록 또는 푸터 스트립). 콘텐츠를 추가하기 전에 모든 페이지는 최소 1개의 섹션이 있어야 합니다.
- **요소** -- 텍스트, 이미지, 버튼, 카드, 양식, 캘린더와 같은 섹션 내부에 배치된 개별 콘텐츠 조각입니다.

### 섹션 추가하기

1. **섹션 추가**(또는 왼쪽 패널 상단의 **+** 버튼)를 클릭합니다.
2. 시작하는 방법을 선택합니다:
   - **템플릿에서** — 카테고리별로 정렬된 섹션 템플릿 갤러리(히어로, 회사소개, 서비스, 기부 등)를 찾아보고 하나를 클릭하여 완전히 스타일이 지정된 미리 채워진 섹션으로 삽입합니다. 추가 후 모든 것을 사용자 정의할 수 있습니다.
   - **빈 섹션** — 열 레이아웃(단일, 2열, 3열 등)을 선택하고 처음부터 구축합니다.
3. 새 섹션이 미리보기에 나타납니다. 섹션을 클릭하여 선택하고 배경색, 패딩 및 기타 스타일 옵션을 구성합니다.

### 섹션에 요소 추가하기

1. 미리보기에서 섹션 내부를 클릭하여 선택합니다.
2. **콘텐츠 추가**를 클릭하고 목록에서 요소 유형을 선택합니다:
   - **텍스트** -- 제목, 단락 및 서식 있는 텍스트
   - **이미지** -- 사진을 업로드하거나 링크합니다
   - **버튼** -- 클릭할 수 있는 행동 유도 링크
   - **카드** -- 제목과 설명이 있는 이미지
   - **양식** -- [양식](../forms/creating-forms)을 페이지에 직접 포함합니다
   - **캘린더** -- 이벤트 캘린더를 표시합니다
   - **자주 묻는 질문** -- 아코디언 스타일의 질문 및 답변 블록
   - **비디오** -- URL로 비디오를 포함합니다
   - **그룹 브라우저** -- 선택적 검색, 카테고리 필터 및 레이블 필터가 있는 모든 교회 그룹의 필터링 가능한 디렉토리
3. 나타나는 설정 패널을 사용하여 요소를 구성합니다.

### 콘텐츠 재정렬하기

각 항목의 왼쪽에 있는 핸들 아이콘(6개 점)을 사용하여 섹션 또는 요소를 드래그하여 재정렬합니다. 섹션 내에서 요소를 드래그하거나 섹션 간에 이동할 수 있습니다.

## 페이지 스타일 지정하기

### 섹션 스타일

섹션을 클릭하여 스타일 패널을 엽니다. 다음을 설정할 수 있습니다:

- **배경** -- 단색, 그래디언트 또는 이미지
- **패딩** -- 섹션 내 위 및 아래 간격
- **너비** -- 전체 너비 또는 중앙/포함

### 요소 스타일

요소를 클릭하여 스타일 패널을 엽니다. 일반 옵션에는 글꼴 크기, 색상, 정렬, 여백 및 패딩이 포함됩니다. 이미지의 경우 대체 텍스트와 링크 대상을 설정할 수 있습니다.

### 사용자 정의 CSS

고급 스타일 지정의 경우 각 섹션과 요소에는 자신의 CSS 규칙을 작성할 수 있는 **사용자 정의 CSS** 필드가 있습니다. 이들은 해당 요소에 범위가 지정되므로 실수로 나머지 페이지에 영향을 주지 않습니다.

:::tip
사용자 정의 글꼴이나 전역 색상과 같이 전체 사이트에 스타일을 적용해야 하는 경우, 개별 페이지의 사용자 정의 CSS 대신 [모양새](appearance) 설정을 사용합니다.
:::

## 페이지 미리보기하기

도구 모음의 미리보기 제어를 사용하여 다양한 화면 크기에서 페이지가 어떻게 보이는지 확인합니다:

- **데스크톱** -- 전체 너비 브라우저 보기
- **모바일** -- 좁은 휴대폰 크기 보기

**미리보기**를 클릭하여 방문자가 보는 것과 정확히 같은 방식으로 새 브라우저 탭에서 페이지의 라이브 버전을 엽니다.

## 변경 사항 취소하기

편집기는 편집 기록을 자동으로 추적합니다. 도구 모음 버튼이나 키보드 바로 가기를 사용하여 탐색합니다:

- **취소** (Ctrl+Z / Cmd+Z) -- 마지막 작업을 되돌립니다
- **다시 실행** (Ctrl+Y / Cmd+Y) -- 실행 취소된 작업을 다시 적용합니다

페이지를 이전 스냅샷으로 복원할 수도 있습니다. 도구 모음의 **기록**을 클릭하여 설명과 함께 저장된 스냅샷 목록을 보고, 항목을 클릭하여 해당 지점으로 복원합니다.

:::warning
스냅샷을 복원하면 현재 페이지 콘텐츠를 스냅샷 버전으로 바꿉니다. 이는 표준 취소 버튼으로 실행 취소할 수 없습니다. 이전 버전으로 돌아갈 수 있는 옵션을 유지하려면 복원하기 전에 현재 상태의 스냅샷을 저장합니다.
:::

## 저장 및 발행

변경 사항은 작업할 때 자동으로 저장됩니다. 도구 모음의 상태 표시기는 변경 사항이 저장되었는지 여부를 표시합니다.

### 초안 및 발행된 상태

페이지는 방문자가 변경 사항을 언제 보는지를 제어하는 **발행된** 상태를 가질 수 있습니다. 도구 모음은 현재 상태를 보여주는 상태 칩을 표시합니다:

- **저장할 때 라이브** -- 페이지가 발행 워크플로를 사용하지 않습니다. 저장된 모든 변경은 즉시 라이브됩니다. 새 페이지의 기본값입니다.
- **발행되지 않은 변경 사항** -- 페이지가 이전에 발행되었지만 마지막 발행 이후 변경했습니다. 방문자는 여전히 이전에 발행된 버전을 봅니다.
- **발행됨** -- 페이지가 라이브 상태이고 저장된 콘텐츠가 방문자가 보는 것과 일치합니다.

변경 사항을 발행하려면 도구 모음의 **발행** 버튼을 클릭합니다. 페이지가 즉시 라이브됩니다.

방문자가 보는 내용에 영향을 주지 않고 마지막으로 발행된 버전으로 되돌리려면 오버플로우 메뉴(⋮)를 열고 **변경 사항 취소**를 클릭합니다.

페이지를 완전히 오프라인 상태로 두려면 오버플로우 메뉴를 열고 **발행 취소**를 클릭합니다. 방문자는 다시 발행할 때까지 해당 페이지를 더 이상 보지 않습니다.

:::tip
다가오는 이벤트와 같이 페이지를 준비하고 싶을 때 초안/발행 워크플로를 사용하고 적절한 시점에만 라이브 상태로 만듭니다. 페이지를 구축하고 미리보고, 준비되면 발행을 클릭합니다.
:::
