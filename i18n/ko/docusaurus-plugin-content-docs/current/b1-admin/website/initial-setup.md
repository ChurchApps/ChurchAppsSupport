# 초기 설정

<div class="article-intro">

모든 B1 계정에는 바로 사용할 수 있는 웹사이트가 포함되어 있습니다. 이 가이드는 교회 도메인 설정, 사이트 외관 구성, 첫 페이지 만들기, 내비게이션 정리 과정을 안내합니다.

</div>

<div class="prereqs">
<h4>시작하기 전에</h4>

- 관리자 권한이 있는 B1.church 계정이 필요합니다
- 맞춤 도메인을 사용하는 경우, DNS 제공업체 로그인 정보를 준비하십시오 (예: GoDaddy, Cloudflare 또는 AWS)
- 최상의 결과를 위해 투명 배경의 PNG 형식으로 교회 로고를 준비하십시오

</div>

## 도메인 설정

교회는 자동으로 B1.church의 하위 도메인을 부여받습니다 (예: `yourchurch.b1.church`). 또한 자체 맞춤 도메인을 B1 사이트에 연결할 수도 있습니다.

1. admin.b1.church를 방문하거나 프로필 드롭다운에서 **앱 전환**을 선택하여 **B1.church 관리자**로 이동합니다.
2. 왼쪽 사이드바에서 **대시보드**를 클릭한 다음, 드롭다운 메뉴에서 **설정**을 선택합니다.
3. **관리**를 클릭하여 하위 도메인을 확인합니다. 공백 없이 짧고 인식하기 쉬운 이름으로 설정하십시오.
4. 맞춤 도메인을 사용하려면, DNS 제공업체(GoDaddy, Cloudflare 또는 AWS 등)에 로그인하여 두 개의 레코드를 추가합니다:
   - 루트 도메인에 대한 **A 레코드**를 `3.23.251.61`로 지정
   - `www`에 대한 **CNAME 레코드**를 `proxy.b1.church`로 지정
5. B1.church 관리자로 돌아가서, 맞춤 도메인을 목록에 추가하고 **추가** 후 **저장**을 클릭합니다. 몇 분 내에 맞춤 도메인에서 사이트에 접근할 수 있습니다.

:::tip
설정 옵션이 보이지 않으면, 교회 계정을 설정한 담당자에게 "교회 설정 편집" 권한을 부여해 달라고 요청하십시오. 자세한 내용은 [역할 및 권한](../settings/roles-permissions.md)을 참조하십시오.
:::

## 첫 페이지 만들기

1. B1 Admin에서 왼쪽 메뉴의 **웹사이트**를 클릭하여 웹사이트 페이지 보기를 엽니다.
2. 오른쪽 상단의 **페이지 추가**를 클릭합니다.
3. 페이지 유형으로 **빈 페이지**를 선택하고 "홈"으로 이름을 지정합니다.
4. **페이지 설정**을 클릭하고 홈 페이지의 URL 경로를 `/`(텍스트 없이 슬래시만)로 설정합니다. 다른 페이지는 `/page-name` 형식을 사용합니다.
5. **콘텐츠 편집**을 클릭하여 구성을 시작합니다. 모든 페이지는 **섹션**으로 시작해야 합니다 -- 이것은 다른 모든 요소의 컨테이너입니다.
6. 섹션을 추가한 후, **콘텐츠 추가**를 다시 클릭하여 텍스트, 이미지, 동영상, 카드, 양식 등을 섹션에 드래그하여 삽입합니다.

:::info
페이지, 내비게이션, 페이지 유형 작업에 대한 자세한 안내는 [페이지 관리](managing-pages)를 참조하십시오.
:::

## 사이트 외관 설정

1. 웹사이트 페이지 보기에서 상단의 **외관** 탭을 클릭합니다.
2. **색상 팔레트**를 사용하여 기본, 보조, 강조 색상의 브랜드 색상을 설정합니다.
3. **타이포그래피 설정**에서 폰트 브라우저를 통해 제목 및 본문 폰트를 선택합니다.
4. 스타일 설정의 **로고**에서 교회 로고를 업로드합니다. 밝은 배경용과 어두운 배경용 두 가지 버전을 제공하십시오.
5. 교회의 연락처 정보와 링크로 **사이트 푸터**를 구성합니다.

:::info
외관에서 변경한 내용은 전체 웹사이트에 적용됩니다. 각 설정에 대한 자세한 안내는 [외관](appearance) 페이지를 참조하십시오.
:::

## 내비게이션 설정

내비게이션 링크는 웹사이트 페이지 보기의 왼쪽 사이드바에 표시됩니다. 정리하려면:

1. **추가**를 클릭하여 새 내비게이션 링크를 만들고 페이지 중 하나에 연결합니다.
2. 링크를 드래그 앤 드롭하여 순서를 변경하거나 상위 항목 아래에 중첩합니다.
3. 사이트를 미리 보기하여 내비게이션이 올바르게 보이는지 확인합니다.
