# 페이지 관리

<div class="article-intro">

웹사이트 페이지 보기는 교회 웹사이트의 모든 페이지를 만들고, 편집하고, 구성하기 위한 중앙 허브입니다. 이 단일 화면에서 페이지 콘텐츠와 사이트의 네비게이션을 모두 관리할 수 있습니다.

</div>

<div class="prereqs">
<h4>시작하기 전에</h4>

- [초기 설정](initial-setup)을 완료하여 도메인과 기본 사이트 설정 구성
- 콘텐츠와 이미지를 준비합니다. 먼저 [파일](files) 관리자를 사용하여 미디어 자산을 업로드합니다.

</div>

## 페이지 유형 이해

**페이지** 표는 사이트의 모든 페이지와 그 상태를 나열합니다:

- **생성됨** -- 교회 데이터(예: 그룹 페이지 또는 설교 페이지)를 기반으로 시스템에서 자동으로 만든 페이지입니다. 이 페이지는 데이터가 변경될 때 자신을 업데이트합니다.
- **사용자 정의** -- 자신의 콘텐츠와 레이아웃으로 직접 만든 페이지입니다.

자동 생성 페이지를 사용자 정의 페이지로 변환하여 콘텐츠와 디자인을 완전히 제어할 수 있습니다.

## 페이지 추가 및 편집

1. 페이지 표의 오른쪽 상단에 있는 **페이지 추가** 버튼을 클릭합니다.
2. 페이지 유형(빈 페이지 또는 템플릿)을 선택하고 이름을 지정합니다.
3. 모든 페이지 옆에 있는 **편집**을 클릭하여 페이지 빌더를 열고 섹션, 텍스트, 이미지 및 기타 요소를 추가할 수 있습니다.
4. **페이지 설정**을 클릭하여 페이지 제목, URL 경로 및 기타 메타데이터를 업데이트합니다.
5. **미리보기** 버튼을 사용하여 새 창에서 페이지를 열고 방문자에게 정확히 어떻게 보이는지 확인합니다.

:::tip
홈페이지의 경우 URL 경로를 단순히 `/`로 설정합니다. 다른 모든 페이지의 경우 `/about` 또는 `/contact` 같은 설명적 경로를 사용합니다.
:::

## 네비게이션 관리

웹사이트 페이지 보기의 왼쪽 사이드바는 네비게이션 링크를 표시합니다. 이 링크는 방문자가 웹사이트에서 보는 메뉴를 제어합니다.

1. **추가**를 클릭하여 새 네비게이션 링크를 만듭니다. 사이트의 모든 페이지 또는 외부 URL을 가리킬 수 있습니다.
2. 링크를 다시 정렬하려면 원하는 순서로 끌어서 놓습니다. 부모 항목 아래에 링크를 중첩하여 드롭다운 메뉴를 만들 수도 있습니다.
3. 링크 옆의 **편집** 아이콘을 클릭하여 레이블, URL 또는 위치를 변경합니다.
4. 네비게이션에서 링크를 제거하려면 **삭제** 아이콘을 클릭합니다.

:::info
네비게이션 링크를 제거해도 페이지 자체는 삭제되지 않습니다. 페이지는 여전히 존재하고 직접 URL로 액세스할 수 있습니다 — 메뉴에만 나타나지 않습니다.
:::

## 사이트 구성 팁

- 방문자가 빨리 찾을 수 있도록 최상위 네비게이션을 5개 또는 6개 항목으로 유지합니다.
- 관련 하위 페이지에 중첩된 링크를 사용합니다(예를 들어 "팀", "신앙", "역사"가 있는 "정보" 드롭다운).
- **모바일 미리보기**를 클릭하여 모바일에서 네비게이션을 검토하여 작은 화면에서 잘 작동하는지 확인합니다.
- 방문자가 찾을 내용을 이해하는 데 도움이 되는 명확하고 설명적인 페이지 이름을 제공합니다.

:::tip
[양식](../forms/creating-forms.md)을 페이지에 추가하여 방문자로부터 등록, 기도 요청 또는 기타 정보를 수집할 수 있습니다.
:::

## 이미지 라이트박스

방문자가 웹사이트의 이미지를 클릭하면 전체 화면 라이트박스 오버레이에서 열립니다. 이를 통해 사람들은 페이지를 벗어나지 않고 사진을 더 큰 크기로 볼 수 있습니다. 구성이 필요 없습니다 — 라이트박스는 페이지 콘텐츠의 이미지에 대해 자동으로 활성화됩니다.
