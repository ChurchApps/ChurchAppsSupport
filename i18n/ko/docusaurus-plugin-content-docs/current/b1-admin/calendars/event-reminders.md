# 이벤트 알림

<div class="article-intro">

이벤트 알림은 이벤트가 발생하기 전에 적절한 사람들에게 자동으로 알립니다 — 예를 들어, "놓치지 마세요! 의료 워크숍은 내일 오전 9시에 시작합니다." 이벤트에 한 번 알림을 구성하면, B1은 푸시 알림과 이메일을 통해 일정대로 보냅니다. 회원은 자신의 [알림 환경설정](../../b1-church/getting-started/notification-preferences)에서 받을 알림을 제어할 수 있습니다.

</div>

<div class="prereqs">
<h4>시작하기 전에</h4>

- 상기시킬 이벤트를 만듭니다([Creating Calendars](creating-calendars) 참조).
- 등록된 참석자에게 도달하려면 이벤트에서 [registration을 활성화](creating-calendars)합니다.
- 전체 그룹에 도달하려면 이벤트가 [그룹](../groups/creating-groups)에 속하고 회원이 있는지 확인합니다.

</div>

## 알림 설정

**Reminders** 섹션의 이벤트에서 알림을 구성합니다.

- **새 이벤트를 만들 때**, 저장하기 전에 이벤트 편집기에서 **Reminders** 섹션을 확장합니다.
- **기존 이벤트**의 경우, 이벤트의 **Registration Details** 페이지(Registrations 섹션에서)를 열어 알림을 추가하거나 변경합니다.

1. **Enable reminders**를 켭니다.
2. **When**을 선택하여 보낼 시기를 선택합니다. 최대 3가지 시간대를 선택합니다: **7 days before**, **3 days before**, **1 day before**, 그리고 **Day of**.
3. 알림이 나가야 하는 **Time of day**를 설정합니다(기본값은 **9:00 AM**, 교회의 현지 시간대).
4. **Who**를 선택하여 누가 상기받아야 하는지 선택합니다([Who Gets Reminded](#who-gets-reminded) 아래 참조).
5. 선택적으로 **Message**를 추가합니다. 기본 문구를 사용하려면 공백으로 두거나 직접 작성합니다 — `{{eventTitle}}`을 포함할 수 있으며 이벤트 이름으로 대체됩니다.
6. **Channels**를 선택합니다: **Push** 알림, **Email**, 또는 둘 다.
7. 이벤트를 저장합니다.

변경 사항을 만들면 **live preview**는 상기받을 대략적인 사람 수, 도달할 수 없는 참석자 수 및 다음 일정 발송 시간을 표시합니다 — 따라서 저장하기 전에 알림이 올바른지 확인할 수 있습니다.

## 누가 상기받습니까

**Who** 설정은 알림이 누구에게 전달되는지를 제어합니다:

- **Registrants only** — 이벤트에 등록되고 사람 기록에 연결된 모든 사람입니다. 이것은 이벤트에 등록이 활성화되어 있을 때의 기본값이므로 작은 등록 이벤트에 대한 알림이 실수로 전체 그룹에 전달되지 않습니다.
- **Heads / registrants only** — 등록당 한 번의 알림(등록한 사람)입니다. 등록의 모든 가족 구성원이 아닙니다.
- **Group members** — 이벤트의 그룹의 모든 사람입니다. 이것은 이벤트에서 등록을 사용하지 않을 때의 기본값입니다.
- **Auto** — 등록이 활성화되면 등록자를 사용하고, 그렇지 않으면 그룹을 사용합니다.

:::info
이름만으로 추가된 손님(연결된 사람 기록이 없음)은 계정, 장치 또는 이메일을 보낼 수 없으므로 알림을 받을 수 없습니다. 미리보기는 이 그룹에 해당하는 참석자 수를 알려줍니다. 커뮤니케이션을 거부한 회원도 건너뜁니다.
:::

## 알림이 전송되는 시기

- 알림은 교회의 현지 시간대에서 선택한 **time of day**에 선택한 각 오프셋에서 발생합니다.
- **이벤트의 날짜 또는 시간을 변경하면** 대기 중인 알림이 자동으로 다시 일정됩니다 — 알림을 편집할 필요가 없습니다.
- **이벤트를 삭제하면**(또는 반복 이벤트의 단일 발생을 취소하면) 대기 중인 알림이 자동으로 취소됩니다.
- 반복 이벤트는 자동으로 처리됩니다: 각 예정된 발생은 자신의 알림을 받습니다.

:::tip
알림은 **푸시를 먼저 사용하고 이메일을 폴백으로 사용합니다**. 회원이 푸시 알림을 활성화하면 푸시를 받습니다. 그렇지 않으면 이메일 대신 받습니다. 회원은 [알림 환경설정](../../b1-church/getting-started/notification-preferences)에서 알림 유형별로 원하는 채널을 선택합니다.
:::

## 회원이 제어할 수 있는 것

알림은 항상 각 회원의 [알림 환경설정](../../b1-church/getting-started/notification-preferences)을 준수합니다. 회원은 다음을 수행할 수 있습니다:

- 다른 알림을 켜 두면서 푸시 또는 이메일에 대해 **Event Reminders**를 끕니다.
- **quiet hours**를 설정하여 긴급하지 않은 알림이 합리적인 시간까지 대기하도록 합니다.

회원이 이벤트 알림을 거부하도록 선택하는 것을 무시할 수 없습니다 — 이것은 B1이 스팸 방지 규칙을 준수하고 회원이 자신의 편지함을 제어하도록 유지합니다.

## 자원 봉사 알림 제공

계획에 예약된 자원 봉사자는 계획 세부 사항이 포함된 별도의 **serving reminder**와 아직 응답하지 않은 경우 이메일에 바로 **Accept / Decline** 단추를 받습니다. 이러한 알림은 달력 이벤트가 아니라 계획 유형에서 구성됩니다 — 자원 봉사자 일정 및 알림이 어떻게 작동하는지는 [Sunday Volunteers](../guides/sunday-volunteers)를 참조하세요.
