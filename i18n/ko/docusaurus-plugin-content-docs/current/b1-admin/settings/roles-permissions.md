# 역할 및 권한

<div class="article-intro">

역할을 사용하면 ChurchApps 계정 내에서 사용자별로 접근할 수 있는 항목을 제어할 수 있습니다. 직원, 자원봉사자 및 기타 팀원을 위한 사용자 정의 역할을 만들어 각각 고유한 접근 수준을 부여하여 데이터를 안전하게 보호할 수 있습니다.

</div>

<div class="prereqs">
<h4>시작하기 전에</h4>

- 도메인 관리자 권한 또는 역할을 관리할 수 있는 권한이 있는 역할이 필요합니다
- 팀원 목록과 각자 접근해야 하는 영역을 준비하십시오
- 역할을 계획하기 위해 아래 사용 가능한 권한 범주를 검토하십시오

</div>

## 역할 접근

1. 왼쪽 사이드바에서 **설정** 페이지로 이동합니다.
2. 헤더에서 **역할** 버튼을 클릭합니다.
3. 역할 페이지에 교회에 대해 현재 정의된 모든 역할이 표시됩니다.

## 역할 페이지 이해

역할 페이지는 두 개의 패널로 나뉘어 있습니다:

- **왼쪽** -- 선택한 역할에 할당된 구성원 목록이 표시됩니다.
- **오른쪽** -- 해당 역할에 대해 구성할 수 있는 권한 설정이 표시됩니다.

역할 이름을 클릭하면 해당 구성원과 권한을 보고 관리할 수 있습니다.

## 역할에 사용자 추가

1. 구성원을 추가하려는 역할을 선택합니다.
2. 왼쪽의 **검색 필드**를 사용하여 추가할 사람을 찾습니다.
3. 검색 결과에서 해당 사람을 선택합니다.
4. 해당 사람이 즉시 역할에 추가됩니다.

## 역할에서 사용자 제거

1. 제거하려는 사용자가 포함된 역할을 선택합니다.
2. 왼쪽의 구성원 목록에서 해당 사람을 찾습니다.
3. 이름 옆의 **제거 버튼**을 클릭합니다.

## 권한 구성

각 역할에 B1 Admin의 특정 영역에 대한 접근 권한을 부여할 수 있습니다. 권한은 섹션별로 정리되어 있습니다:

- **사람** -- 교인 디렉토리 및 개인 기록에 대한 접근입니다.
- **헌금** -- 헌금 기록 및 기금 관리에 대한 접근입니다.
- **출석** -- 출석 추적 및 보고서에 대한 접근입니다.
- **콘텐츠** -- 웹사이트 및 콘텐츠 관리에 대한 접근입니다.
- 기타 추가 영역이 제공됩니다.

역할 페이지 오른쪽의 체크박스를 사용하여 각 영역에 대한 접근을 활성화하거나 비활성화합니다.

:::warning
**도메인 관리자**는 ChurchApps 계정의 모든 영역에 대한 전체 접근 권한을 가지고 있습니다. 이 역할의 권한은 수정하거나 제한할 수 없습니다. 가장 신뢰할 수 있는 관리자에게만 이 역할을 사용하십시오.
:::

:::tip
"회계 담당자"에게는 **헌금** 접근만 부여하거나, "체크인 봉사자"에게는 **출석** 접근만 부여하는 등 특정 역할을 만드십시오. 이는 최소 권한 원칙을 따르며 데이터를 안전하게 보호합니다. ChurchApps가 정보를 보호하는 방법에 대한 자세한 내용은 [데이터 보안](./data-security.md)을 참조하십시오.
:::
