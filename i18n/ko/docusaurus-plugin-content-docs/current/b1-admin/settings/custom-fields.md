# 사용자 정의 필드

<div class="article-intro">

**사용자 정의 필드**는 모든 사람 기록에서 자신의 정보를 추적할 수 있게 해줍니다 -- B1이 기본 제공 필드를 갖지 않은 것들 (배경 확인 만료 날짜, T셔츠 사이즈, 또는 세례 수업 상태 같은). 설정에서 필드를 한 번 정의한 다음 각 사람의 프로필에 값을 채우고 이에 대해 검색하거나 리스트를 구축합니다. 이것은 단일 데이터 항목을 저장하기 위해 사람 양식을 생성하는 더 오래된 해결책을 대체합니다.

</div>

<div class="prereqs">
<h4>시작하기 전에</h4>

- 필드를 정의하고 값을 채우려면 **사람** 편집 권한이 필요하고 **설정** 영역에 액세스할 수 있어야 합니다. 사람 보기 권한이 있는 모든 사람이 값을 볼 수 있습니다. [역할 및 권한](./roles-permissions.md) 참조.
- 시작하기 전에 추적할 것과 어떤 타입이 가장 적합한지 (텍스트, 숫자, 날짜, 예/아니요 또는 선택 목록) 결정합니다.

</div>

## 사용자 정의 필드 열기

B1 Admin에서 왼쪽 사이드바의 **설정**으로 이동하고 **사용자 정의 필드** 카드를 선택합니다. **/settings/custom-fields**로 직접 이동할 수도 있습니다. 정의한 모든 필드를 보여주는 목록이 표시되며 해당 **이름** 및 **필드 타입**을 표시합니다. 아직 생성하지 않았다면 패널은 *"아직 사용자 정의 필드가 추가되지 않았습니다."*를 읽습니다.

## 필드 추가

1. **필드 추가**를 클릭합니다.
2. 오른쪽에서 열리는 에디터에 **이름** (예: *배경 확인 만료*)을 입력합니다 -- 이것은 직원이 사람 프로필 및 검색에서 볼 레이블입니다.
3. **필드 타입** 선택:
   - **텍스트박스** -- 무료 형식 짧은 텍스트.
   - **정수** -- 소수점 없는 숫자.
   - **소수** -- 소수점을 포함할 수 있는 숫자.
   - **날짜** -- 달력 날짜.
   - **예/아니요** -- 단순 예 또는 아니요 답변.
   - **다중 선택** -- 선택 목록. 이 타입을 선택하면 **선택 에디터**가 나타나므로 사람들이 선택할 수 있는 각 옵션을 추가할 수 있습니다.
4. **저장**을 클릭합니다.

필드는 이제 모든 사람의 프로필에서 사용 가능합니다.

:::info
필드 타입은 [양식 질문](../forms/creating-forms.md)에 사용되는 동일한 세트이므로 값은 B1 전체에서 일관되게 동작합니다.
:::

## 필드 편집

목록의 모든 필드 행을 클릭하여 에디터에서 다시 열기. 이름, 타입 또는 선택을 변경하고 **저장**을 클릭합니다.

:::warning
이미 값을 가진 필드의 **필드 타입** 변경 (예: 텍스트박스에서 날짜로)은 더 이상 새로운 타입과 일치하지 않는 형식으로 이전에 입력된 값을 남길 수 있습니다. 직원이 필드를 채우기 시작하면 타입 변경할 때 주의하세요.
:::

## 필드 삭제

편집을 위해 필드를 열고 **삭제**를 클릭합니다. 확인됩니다: *"이 사용자 정의 필드를 삭제하시겠습니까? 저장된 값도 제거됩니다."* 필드를 삭제하면 그것과 모든 사람에게 저장된 모든 값이 영구적으로 제거되며 이를 실행 취소할 수 없습니다.

## 사람에게서 값 채우기

최소 하나의 사용자 정의 필드가 존재하면 그 값은 각 사람의 기록에서 기본 제공 상세정보와 나란히 있습니다 -- **개인 상세**에서 보고 같은 양식에서 사람의 나머지 정보를 편집하는 데 사용합니다. 첫 번째 필드를 정의할 때까지 아무것도 나타나지 않습니다.

1. **사람**에서 사람의 기록을 엽니다.
2. **개인 상세** 섹션에서 **편집** (연필) 버튼을 클릭합니다.
3. 편집 양식 하단의 **사용자 정의 필드** 영역으로 스크롤하고 각 필드에 대한 값을 채웁니다. 각 필드는 그 타입과 일치하는 입력을 표시합니다 -- 날짜 필드의 날짜 피커, 예/아니요 필드의 예/아니요 드롭다운, 다중 선택의 선택 목록 등.
4. **저장**을 클릭합니다. 사용자 정의 필드 값은 사람의 나머지 상세정보와 함께 저장됩니다.

프로필로 돌아가면 값을 가진 모든 필드는 이제 **개인 상세** 섹션에 나타납니다 (예/아니요 답변은 *예* 또는 *아니요*로 읽고, 다중 선택은 옵션의 레이블을 표시합니다). 빈 필드는 단순히 숨겨집니다. 값을 제거하려면 사람을 편집하고, 필드를 지우고, 저장합니다 -- 빈 값은 레코드에서 저장되지 않고 삭제됩니다.

:::tip
전형적인 사용 사례는 자원봉사자 안전입니다: *배경 확인 만료*라고 불리는 **날짜** 필드를 생성하고, 각 자원봉사자의 날짜를 기록한 다음, [저장된 리스트](../people/lists.md)를 구축합니다 -- 날짜가 지났을 때 모든 사람을 표시하는.
:::

## 사용자 정의 필드에서 검색 및 리스트 구축

사용자 정의 필드는 완전히 검색 가능합니다:

1. **사람** 페이지에서 [고급 검색](../people/searching-people.md)을 엽니다.
2. **사용자 정의 필드** 카테고리를 확장합니다.
3. 필터링할 필드를 확인하고, 연산자를 선택하고, 값을 입력합니다. 제공된 연산자는 필드의 타입과 일치합니다:
   - **텍스트박스** -- 포함, 같음, 시작, 끝.
   - **정수 / 소수** -- 같음, 보다 큼, 이상, 보다 작음, 이하.
   - **날짜** -- 같음, 다음 (보다 큼), 이전 (보다 작음).
   - **예/아니요** -- 예 또는 아니요와 같음.
   - **다중 선택** -- 선택 중 하나와 같음 또는 포함.

모든 사용자 정의 필드 검색을 [리스트](../people/lists.md)로 저장합니다. 리스트는 라이브 쿼리이므로 *배경 확인 만료는 오늘 이전*에서 구축된 리스트는 열 때마다 모든 사람을 재확인합니다 -- 수동 유지보수가 없습니다.

## 병합 시 어떻게 되는가

두 사람의 기록을 [병합할 때](../people/adding-people.md) 사용자 정의 필드 값은 자동으로 이월됩니다. 유지하는 사람은 자신의 값에 대해 고수합니다. 제거된 사람만 값을 가진 필드의 경우 그 값이 복사되므로 아무것도 잃지 않습니다.
