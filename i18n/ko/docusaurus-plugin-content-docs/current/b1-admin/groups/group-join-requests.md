# 그룹 참여 요청

<div class="article-intro">

그룹이 승인 기반 참여 정책으로 구성되면 사람들은 참여 요청을 제출할 수 있습니다. 그룹 지도자와 관리자가 이러한 요청을 검토하고 승인하거나 거부합니다. 이것은 교회에 그룹 구성원 자격을 제어하면서도 사람들이 참여에 관심을 표현하기 쉽게 만듭니다.

</div>

<div class="prereqs">
<h4>시작하기 전에</h4>

- 그룹 관리 권한이 있거나 특정 그룹의 지도자여야 합니다. 자세한 내용은 [역할 및 권한](../people/roles-permissions.md)을 참조하세요.
- 그룹의 참여 정책이 **요청**(승인 필요)으로 설정되어야 합니다. 참여 정책 구성 방법은 [그룹 만들기](./creating-groups.md)를 참조하세요.

</div>

## 참여 정책 이해하기

그룹은 세 가지 다른 참여 정책을 가질 수 있습니다:

- **개방형** -- 누구나 승인 없이 즉시 참여할 수 있습니다.
- **요청** -- 사람들이 승인이 필요한 참여 요청을 제출합니다.
- **폐쇄형** -- 아무도 참여를 요청할 수 없습니다(구성원은 수동으로 추가되어야 함).

그룹이 **요청** 정책을 사용할 때, 모든 참여 시도는 이 페이지에 설명된 승인 워크플로우를 거칩니다.

## 보류 중인 요청 보기

### 그룹 지도자의 경우

1. B1 Admin의 **그룹**으로 이동합니다.
2. 그룹 이름을 클릭합니다.
3. **참여 요청** 탭을 클릭합니다.
4. 이 그룹에 대한 모든 보류 중인 요청이 표시됩니다.

### 관리자의 경우

그룹 관리 권한이 있는 관리자는 모든 그룹의 보류 중인 요청을 볼 수 있습니다:

1. B1 Admin의 **그룹**으로 이동합니다.
2. 알림 배지 또는 보류 중인 요청 표시기를 찾습니다.
3. 클릭하여 교회 전역의 모든 보류 중인 요청을 봅니다.

## 참여 요청 검토

각 참여 요청은 다음을 표시합니다:

- **사람 이름 및 사진** -- 참여를 요청하는 사람
- **선택적 메시지** -- 왜 그룹에 참여하고 싶은지 설명하는 개인 메시지(제공된 경우)
- **요청 날짜** -- 요청이 제출된 시기

요청을 검토하려면:

1. 제공된 경우 사람의 메시지를 읽습니다.
2. 필요한 경우 사람의 이름을 클릭하여 프로필을 봅니다.
3. 승인할지 거부할지 결정합니다.

## 요청 승인

1. 참여 요청에서 **승인**을 클릭합니다.
2. 그 사람이 즉시 그룹 구성원으로 추가됩니다.
3. 요청자는 요청이 승인되었음을 알리는 알림을 받습니다.
4. 요청이 시스템에서 승인됨으로 표시됩니다.

:::tip
요청을 승인하면 그 사람이 일반 그룹 구성원이 됩니다. 나중에 [그룹 구성원](./group-members.md) 페이지에서 그들을 그룹 지도자로 승격할 수 있습니다.
:::

## 요청 거부

1. 참여 요청에서 **거부**를 클릭합니다.
2. 선택적으로 거부 사유를 입력합니다(최대 500자).
3. **확인**을 클릭합니다.
4. 요청자는 거부 사유를 포함한 알림을 받습니다(제공된 경우).
5. 요청이 거부됨으로 표시됩니다.

:::info
거부 사유를 제공하면 그 사람이 요청이 승인되지 않은 이유를 이해하고 나중에 다시 시도하거나 다른 그룹을 탐색하도록 장려할 수 있습니다.
:::

## 알림

참여 요청 시스템은 자동으로 알림을 보냅니다:

- **요청이 제출될 때** -- 모든 그룹 지도자가 알림을 받습니다.
- **요청이 승인될 때** -- 요청자가 확인을 받습니다.
- **요청이 거부될 때** -- 요청자가 거부 사유를 포함한 알림을 받습니다.

알림은 B1.church의 알림 센터 및 모바일 앱에 나타납니다.

## 구성원 측에서 요청 관리

사람들은 B1.church에서 자신의 참여 요청을 관리할 수 있습니다:

- 그룹 상세 페이지에서 보류 중인 요청의 상태를 봅니다.
- 마음이 바뀌면 보류 중인 요청을 취소합니다.
- 요청이 승인되었는지 거부되었는지 확인합니다.

## 모범 사례

- **신속하게 응답** -- 사람들이 기다리지 않도록 24-48시간 내에 요청을 검토하세요.
- **거부 사유를 명확히** -- 사람들이 다음 단계 또는 대안을 이해하도록 돕습니다.
- **프로필 확인** -- 그 사람의 프로필을 검토하여 그룹에 적합한지 확인합니다.
- **기대치 전달** -- 그룹 설명이 그룹이 누구를 위한 것인지 명확하게 명시하도록 합니다.
