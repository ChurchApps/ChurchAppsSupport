# 그룹 가입 요청

<div class="article-intro">

그룹이 승인 기반 가입 정책으로 구성되어 있으면 사람들이 가입 요청을 제출할 수 있습니다. 그룹 리더와 관리자가 이러한 요청을 검토하고 승인하거나 거부합니다. 이를 통해 교회는 그룹 회원 자격을 제어하면서 사람들이 가입에 관심을 표현하기 쉽게 할 수 있습니다.

</div>

<div class="prereqs">
<h4>시작하기 전에</h4>

- 그룹을 관리할 수 있는 권한이 있거나 특정 그룹의 리더여야 합니다. 자세한 내용은 [역할 및 권한](../people/roles-permissions.md)을 참조하세요.
- 그룹의 가입 정책이 **Request**(승인 필요)로 설정되어 있어야 합니다. 가입 정책을 구성하는 방법은 [그룹 만들기](./creating-groups.md)를 참조하세요.

</div>

## 가입 정책 이해하기

그룹은 세 가지 다른 가입 정책을 가질 수 있습니다:

- **Open** -- 승인 없이 누구나 즉시 가입할 수 있습니다
- **Request** -- 사람들이 승인이 필요한 가입 요청을 제출합니다
- **Closed** -- 아무도 가입을 요청할 수 없습니다(회원은 수동으로 추가해야 합니다)

그룹이 **Request** 정책을 사용하는 경우 모든 가입 시도는 이 페이지에 설명된 승인 워크플로를 거칩니다.

## 대기 중인 요청 보기

### 그룹 리더의 경우

1. B1 Admin에서 **Groups**로 이동합니다
2. 그룹 이름을 클릭합니다
3. **Join Requests** 탭을 클릭합니다
4. 이 그룹에 대한 모든 대기 중인 요청을 볼 수 있습니다

### 관리자의 경우

그룹 관리 권한이 있는 관리자는 모든 그룹에서 대기 중인 요청을 볼 수 있습니다:

1. B1 Admin에서 **Groups**로 이동합니다
2. 알림 배지 또는 대기 중인 요청 표시기를 찾습니다
3. 클릭하여 교회 전체의 모든 대기 중인 요청을 봅니다

## 가입 요청 검토하기

각 가입 요청은 다음을 표시합니다:

- **개인의 이름과 사진** -- 가입을 요청하는 사람
- **선택적 메시지** -- 가입하려는 이유를 설명하는 개인 메시지(제공된 경우)
- **요청 날짜** -- 요청이 제출된 시간

요청을 검토하려면:

1. 제공된 경우 개인의 메시지를 읽습니다
2. 필요한 경우 개인의 이름을 클릭하여 프로필을 봅니다
3. 승인 또는 거부할지 결정합니다

## 요청 승인하기

1. 가입 요청에서 **Approve**를 클릭합니다
2. 개인이 즉시 회원으로 그룹에 추가됩니다
3. 요청자는 요청이 승인되었다는 알림을 받습니다
4. 요청이 시스템에서 승인됨으로 표시됩니다

:::tip
요청을 승인하면 개인은 일반 그룹 회원이 됩니다. 필요한 경우 나중에 [그룹 회원](./group-members.md) 페이지에서 그룹 리더로 승격할 수 있습니다.
:::

## 요청 거부하기

1. 가입 요청에서 **Decline**을 클릭합니다
2. 선택적으로 거부 이유를 제공합니다(최대 500자)
3. **Confirm**을 클릭합니다
4. 요청자는 거부 이유(제공된 경우)와 함께 알림을 받습니다
5. 요청이 거부됨으로 표시됩니다

:::info
거부 이유를 제공하면 개인이 요청이 승인되지 않은 이유를 이해하는 데 도움이 되며 나중에 다시 시도하거나 다른 그룹을 탐색하도록 장려할 수 있습니다.
:::

## 알림

가입 요청 시스템은 자동으로 알림을 보냅니다:

- **요청이 제출될 때** -- 모든 그룹 리더가 알림을 받습니다
- **요청이 승인될 때** -- 요청자가 확인을 받습니다
- **요청이 거부될 때** -- 요청자가 거부 이유와 함께 알림을 받습니다

알림은 B1.church 및 모바일 앱의 사용자 알림 센터에 나타납니다.

## 회원 측에서 요청 관리하기

사람들은 B1.church에서 자신의 가입 요청을 관리할 수 있습니다:

- 그룹 세부 정보 페이지에서 대기 중인 요청의 상태를 봅니다
- 마음이 바뀌면 대기 중인 요청을 취소합니다
- 요청이 승인 또는 거부되었는지 확인합니다

## 모범 사례

- **신속하게 응답** -- 사람들이 기다리지 않도록 24-48시간 이내에 요청을 검토하세요
- **거부 이유를 명확히** -- 사람들이 다음 단계나 대안을 이해하도록 도와주세요
- **프로필 확인** -- 개인의 프로필을 검토하여 그룹에 적합한지 확인하세요
- **기대치 전달** -- 그룹 설명에 그룹이 누구를 위한 것인지 명확하게 명시하세요
