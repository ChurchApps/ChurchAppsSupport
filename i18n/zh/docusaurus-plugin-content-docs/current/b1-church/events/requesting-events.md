# 请求事件和房间

<div class="article-intro">

如果您的教会跟踪可预订的房间和资源，您可以直接从成员门户请求一个事件 — 带上它需要的房间和设备。与现有预订的冲突会在您选择时被标记，您的请求会发送给教会进行批准，您可以从"我的请求"跟踪或取消它。

</div>

<div class="prereqs">
<h4>开始前</h4>

- 您必须[登录](../getting-started/logging-in.md)成员门户
- 您的教会必须在B1 Admin中设置房间或资源 — 请求快捷方式仅在设置时出现

</div>

## 提交请求

1. 在成员门户中打开**我**页面，然后在快捷方式下点击**请求事件**。
2. 填写事件基础信息：**事件标题**、可选的**描述**、**开始**和**结束**时间（或**全天**）以及是否**重复**。
3. 选择您需要的**房间**和**资源**。对于资源，您可以设置**数量**。
4. 注意冲突警告 — 如果您的时间与房间或资源的现有预订重叠，您会看到*"此时间与现有预订冲突。"*您可以调整您的时间或选择其他。
5. 点击**提交请求**。

提交后，会显示**请求已提交**摘要，显示您请求的每个房间和资源的状态：

- **已批准** -- 立即预订（房间或资源没有批准组）。
- **待批准** — 教会中的某人需要先批准。

## 跟踪您的请求

从"我"页面打开**我的请求**。每个请求显示事件、您请求它的时间和一个状态标签：

- **待定** — 等待决定
- **已批准** — 您的预订已确认
- **被拒绝** — 请求被拒绝

在请求仍待定时，您可以用**取消请求**撤回它（系统会要求您确认 — 取消不能撤销）。

:::info
请求由教会工作人员在B1 Admin [批准页面](../../b1-admin/calendars/approvals)上审查。哪些房间和资源需要批准由您的教会决定 — 请参见[房间和资源](../../b1-admin/calendars/rooms-resources)。
:::
