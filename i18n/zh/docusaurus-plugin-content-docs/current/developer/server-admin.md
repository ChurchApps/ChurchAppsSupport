# 服务器管理

<div class="article-intro">

ChurchApps 中的服务器管理功能仅对具有 **Server.Admin** 权限的用户可用。这些工具用于平台操作、支持和跨系统中所有教会的故障排除。

</div>

:::warning 访问受限
本页面描述的功能需要 **Server.Admin** 权限,对常规教会管理员不可用。它们仅供平台运营商和支持人员使用。
:::

## 访问服务器管理员

具有 Server.Admin 权限的用户可以从 B1 Admin 访问服务器管理面板:

1. 登录 [admin.b1.church](https://admin.b1.church)
2. 点击主导航中的 **Admin** 选项卡
3. 服务器管理面板包括用于管理教会、用户和系统操作的选项卡

## 用户模拟

模拟功能允许服务器管理员以另一个用户身份登录,以进行支持和故障排除。这在调查用户报告的问题或帮助教会配置其系统时很有用。

### 如何模拟用户

1. 导航到服务器管理面板中的 **Impersonate** 选项卡
2. 在搜索字段中输入用户的姓名或电子邮件地址
3. 点击 **Search** 或按 Enter
4. 从搜索结果中,点击要模拟的用户
5. 在出现的对话框中确认模拟
6. 您将以该用户身份登录并重定向到其帐户

### 重要说明

- 模拟会创建一个具有目标用户权限和教会访问权限的新会话
- 当您模拟另一个用户时,您的原始管理员会话结束
- 模拟时执行的所有操作都会记录在审计跟踪中
- 要返回您的管理员帐户,请注销并使用您的凭据重新登录
- 仅在支持目的需要时使用模拟,并在访问其帐户以提供支持时始终通知用户

### API 端点

模拟功能由成员 API 中的 `/users/:userId/impersonate` 端点支持。有关技术详细信息,请参阅[成员端点](/docs/developer/api/endpoints/membership#users)。

### 安全注意事项

- 模拟需要 Server.Admin 权限 - 此权限应谨慎授予,并且只授予受信任的平台运营商
- 所有模拟事件都记录有管理员用户 ID 和目标用户 ID
- 模拟发生时不会通知教会,因此请制定关于何时以及如何使用此功能的明确政策
- 考虑在您的支持票据系统中记录模拟事件以保持问责制
