# 自定义字段

<div class="article-intro">

**自定义字段**让您可以在每条人员记录上追踪自己的信息 —— 也就是 B1 没有内置字段支持的内容，比如背景调查到期日期、T 恤尺码，或洗礼课程状态。您只需在设置中定义一次字段，然后在每个人的资料上填入一个值，即可对其进行搜索或建立列表。这取代了以往为了存储单一自定义数据而不得不创建人员表单的变通做法。

</div>

<div class="prereqs">
<h4>开始之前</h4>

- 您需要**人员**编辑权限才能定义字段和填写值，并需要访问**设置**区域的权限。拥有人员查看权限的任何人都可以看到这些值。请参阅[角色与权限](./roles-permissions.md)。
- 在开始之前，先想清楚您要追踪什么内容，以及哪种类型最合适（文本、数字、日期、是/否答案，或选择列表）。

</div>

## 打开自定义字段

在 B1 Admin 中，前往左侧边栏的**设置**，然后选择**自定义字段**卡片。您也可以直接访问 **/settings/custom-fields**。您将看到一份已定义字段的列表，显示每个字段的**名称**和**字段类型**。如果您还没有创建任何字段，面板会显示*"No custom fields have been added yet."*。

## 添加字段

1. 点击**添加字段**。
2. 在右侧打开的编辑器中，输入一个**名称** —— 这是员工在人员资料和搜索中会看到的标签（例如，*Background check expires*）。
3. 选择一种**字段类型**：
   - **文本框** —— 自由格式的短文本。
   - **整数** —— 不含小数的数字（例如，计数）。
   - **小数** —— 可包含小数的数字。
   - **日期** —— 一个日历日期。
   - **是/否** —— 简单的是或否答案。
   - **多项选择** —— 一个选择列表。选择此类型时，会出现一个**选项编辑器**，供您添加人们可选择的每个选项。
4. 点击**保存**。

现在该字段已在每个人的资料上可用。

:::info
这些字段类型与[表单问题](../forms/creating-forms.md)使用的是同一套类型，因此各值在整个 B1 中表现一致。
:::

## 编辑字段

点击列表中的任意字段行即可在编辑器中重新打开它。修改名称、类型或选项，然后点击**保存**。

:::warning
更改已有值的字段的**字段类型**（例如，从文本框改为日期）可能会使先前输入的值格式与新类型不再匹配。一旦员工开始填写该字段，更改类型时请务必谨慎。
:::

## 删除字段

打开一个字段进行编辑，然后点击**删除**。系统会要求您确认："Are you sure you wish to delete this custom field? Its stored values will also be removed."删除字段会永久移除该字段**以及所有人身上存储的对应值**——此操作无法撤销。

## 在人员记录上填写值

只要存在至少一个自定义字段，其值就会与内置详情一起出现在每个人的记录中 —— 您可以在**个人详情**中查看它们，并在用于该人其余信息的同一表单上编辑它们。在您定义第一个字段之前，不会出现任何额外内容。

1. 在**人员**中打开一个人的记录。
2. 在**个人详情**部分，点击**编辑**（铅笔）按钮。
3. 滚动到编辑表单底部的**自定义字段**区域，为每个字段填入一个值。每个字段都会显示与其类型匹配的输入方式 —— 日期字段有日期选择器，是/否字段有是/否下拉菜单，多项选择有选择列表，依此类推。
4. 点击**保存**。您的自定义字段值会与该人的其余详情一起保存。

回到资料页面后，任何有值的字段现在都会显示在**个人详情**部分中（是/否答案会显示为*Yes*或*No*，多项选择会显示选项的标签）。留空的字段则不会显示。要删除一个值，请编辑该人员、清空该字段并保存 —— 空值会从记录中被删除，而不是作为空白值存储。

:::tip
最典型的用例是志愿者安全管理：创建一个名为*Background check expires*的**日期**字段，记录每位志愿者的日期，然后建立一个[保存的列表](../people/lists.md)，标记出日期已过期的人。
:::

## 在自定义字段上搜索并建立列表

自定义字段完全可搜索：

1. 在**人员**页面上，打开[高级搜索](../people/searching-people.md)。
2. 展开**自定义字段**类别。
3. 勾选您要筛选的字段，选择一个运算符，然后输入一个值。系统提供的运算符与字段类型相匹配：
   - **文本框** —— 包含、等于、开头是、结尾是。
   - **整数 / 小数** —— 等于、大于、大于或等于、小于、小于或等于。
   - **日期** —— 等于、晚于（大于）、早于（小于）。
   - **是/否** —— 等于是或否。
   - **多项选择** —— 等于或包含某个选项。

将任何自定义字段搜索保存为[列表](../people/lists.md)。列表是实时查询，因此基于*Background check expires is before today*建立的列表在您每次打开时都会重新检查每个人 —— 无需手动维护。

## 合并时会发生什么

当您[合并两条人员记录](../people/adding-people.md)时，自定义字段的值会自动一并转移。您保留的那个人会保留自己原有的值；对于只有被移除的人才有值的字段，该值会被复制过来，确保不会丢失任何信息。
