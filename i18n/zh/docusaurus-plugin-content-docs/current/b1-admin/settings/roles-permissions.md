# 角色与权限

<div class="article-intro">

角色让您可以控制不同用户在您的 ChurchApps 账户中可以访问什么。您可以为员工、志愿者和其他团队成员创建自定义角色，每个角色都有自己的访问级别，以确保您的数据安全。

</div>

<div class="prereqs">
<h4>开始之前</h4>

- 您需要 Domain Admin 访问权限或具有管理角色权限的角色
- 准备好团队成员列表以及他们需要访问的区域
- 查看下方的可用权限类别以规划您的角色

</div>

## 访问角色

1. 导航到左侧边栏中的**设置**页面。
2. 点击页头中的**角色**按钮。
3. 角色页面显示您教会当前定义的所有角色。

## 了解角色页面

角色页面分为两个面板：

- **左侧** -- 显示分配给所选角色的成员列表。
- **右侧** -- 显示您可以为该角色配置的权限设置。

点击任何角色名称以查看和管理其成员和权限。

## 向角色添加用户

1. 选择您要添加成员的角色。
2. 使用左侧的**搜索字段**查找您要添加的人。
3. 从搜索结果中选择该人。
4. 他们将立即被添加到角色中。

## 从角色中移除用户

1. 选择包含您要移除的用户的角色。
2. 在左侧的成员列表中找到该人。
3. 点击其名称旁边的**移除按钮**。

## 配置权限

每个角色可以被授予对 B1 Admin 特定区域的访问权限。权限按部分组织：

- **人员** -- 访问成员目录和个人记录。
- **捐款** -- 访问捐款记录和基金管理。
- **出勤** -- 访问出勤跟踪和报告。
- **内容** -- 访问网站和内容管理。
- 以及其他可用的区域。

使用角色页面右侧的复选框来启用或禁用每个区域的访问权限。

:::warning
**Domain Admin** 拥有对您 ChurchApps 账户所有区域的完全访问权限。他们的权限无法被修改或限制。此角色仅用于您最信任的管理员。
:::

:::tip
创建特定角色，例如仅具有**捐款**访问权限的"财务主管"，或仅具有**出勤**访问权限的"签到志愿者"。这遵循最小权限原则并确保您的数据安全。有关 ChurchApps 如何保护您的信息的更多内容，请参阅[数据安全](./data-security.md)。
:::
