# 服务顺序

<div class="article-intro">

服务顺序定义了您的礼拜服务中的元素序列。使用它来规划从开始歌曲到结束祷告的所有内容，以便您的整个团队提前知道服务的流程。

</div>

<div class="prereqs">
<h4>开始前</h4>

- [创建一个服务计划](./plans.md)为您想要构建服务顺序的日期
- 可选地，[添加歌曲](./songs.md)到您的库中，以便您可以直接将它们链接到服务顺序中

</div>

## 访问服务顺序

1. 导航到**服务**并选择您的事工。
2. 单击**计划类型**，然后打开特定的**计划**。
3. 单击计划详情页面上的**服务顺序**选项卡。

## 构建您的服务顺序

服务顺序是按照他们将在服务期间发生的顺序排列的项目列表。常见的元素包括：

- 敬拜歌曲
- 祷告
- 经文朗读
- 通告
- 讲道或信息
- 奉献
- 结束

## 添加项目到服务顺序

1. 在**服务顺序**选项卡上，单击**添加项目**。
2. 选择您想要添加的元素类型（例如，来自您的库的歌曲、自定义项目或标题）。
3. 输入项目的详情，例如标题和任何备注。
4. 单击**保存**。

:::tip
您可以直接从您的[歌曲](./songs.md)库添加歌曲。这将歌曲详情（包括歌词和编排）链接到您的服务计划，以便您的敬拜团队拥有他们需要的一切。
:::

## 重新排序项目

通过每行左侧的把手拖动项目以重新排列顺序 -- 放在您想要项目落地的蓝线上，或放在部分标题上将其移入该部分。屏幕上显示的顺序是您的团队在服务期间将遵循的顺序。

## 时间和媒体

每个项目和部分显示它运行多长时间以及它何时开始：

- 如果计划选择了单个服务时间，左侧栏显示每个项目开始的实际**钟点时间**（例如上午 9:14）。否则它显示从服务开始的经过时间。
- 部分标题将其内部的所有运行时间相加。
- 歌曲和自定义项目在您在项目编辑屏幕上输入**分钟**/**秒**后计入此日程。
- 您从媒体库（图像或视频剪辑）添加的项目显示**缩略图**并自动计时 -- 视频的持续时间是从文件本身测量的，而图像贡献约 5:00 的计划估计（以斜体显示），因为它在屏幕上保持直到操作员推进它。您仍然可以编辑任何一个以获得特定的持续时间。

## 编辑和删除项目

- 单击服务顺序中的任何项目以**编辑**其详情。
- 使用**删除**选项从顺序中删除项目。

:::info
服务顺序特定于每个计划。您在此处所做的更改仅影响所选的计划，不会改变其他计划或您的歌曲库。
:::

## 在特定服务时间中包含项目

如果您的计划有多个服务时间（例如上午 8 点和上午 10 点服务），您可以选择每个项目应出现在哪些服务中。这很有用，当通告仅与一项服务相关，或当歌曲在一项服务中唱但在另一项中不唱时。

1. 打开在**时间**选项卡上定义了两个或更多服务时间的计划。
2. 在**服务顺序**选项卡上，单击项目进行编辑。
3. 在**包含在服务中**下，您将看到每个服务时间的复选框，标有时间。
4. 取消选中应跳过该项目的任何服务。
5. 单击**保存**。

默认情况下，每个项目都包含在每个服务中。当您打印筛选到该服务的计划时，排除的服务时间被隐藏，因此每个服务都收到一个仅包含相关项目的干净运行表。

:::tip
**包含在服务中**部分仅在计划有多个服务时间时显示。如果您只看到一个服务，首先编辑计划的**时间**选项卡以添加其他服务。
:::

## 分享服务顺序

一旦您的服务顺序完成，您可以从计划详情页面打印完整计划（包括服务顺序）。这给您的团队一个完整的服务概述。

## 后续步骤

- 在[歌曲](./songs.md)页面上管理您的敬拜曲目
- 在[计划](./plans.md)分配选项卡上将志愿者分配给服务
- 为与服务相关的任何后续项目创建[任务](./tasks.md)
