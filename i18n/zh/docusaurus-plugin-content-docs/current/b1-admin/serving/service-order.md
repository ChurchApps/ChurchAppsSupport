# 服务顺序

<div class="article-intro">

服务顺序定义了您敬拜服务中元素的顺序。使用它来布置从开始歌曲到结束祷告的所有内容，以便您的整个团队提前了解服务的流程。

</div>

<div class="prereqs">
<h4>开始前</h4>

- [为您想要构建服务顺序的日期创建一个服务计划](./plans.md)
- 可选择：添加[歌曲](./songs.md)到您的库中，以便您可以直接将它们链接到服务顺序中

</div>

## 访问服务顺序

1. 导航到 **Serving** 并选择您的事工。
2. 点击一个**计划类型**，然后打开一个特定的**计划**。
3. 点击计划详情页面上的 **Service Order** 选项卡。

## 构建您的服务顺序

服务顺序是按照它们在服务期间发生的顺序排列的项目列表。常见的元素包括：

- 敬拜歌曲
- 祷告
- 圣经朗读
- 公告
- 讲道或讯息
- 奉献
- 结束

## 向服务顺序添加项目

1. 在 **Service Order** 选项卡上，点击 **Add Item**。
2. 选择您想添加的元素类型（例如，来自您库的歌曲、自定义项目或标题）。
3. 输入项目的详细信息，例如标题和任何注释。
4. 点击 **Save**。

:::tip
您可以直接从您的[歌曲](./songs.md)库中添加歌曲。这会将歌曲详细信息（包括歌词和编排）链接到您的服务计划中，以便您的敬拜团队拥有他们需要的一切。
:::

## 重新排列项目

拖放项目来重新排列顺序。屏幕上显示的顺序是您的团队在服务期间将遵循的顺序。

## 编辑和移除项目

- 点击服务顺序中的任何项目来**编辑**其详细信息。
- 使用**删除**选项来从顺序中移除项目。

:::info
服务顺序特定于每个计划。您在此处进行的更改仅影响所选计划，不会更改其他计划或您的歌曲库。
:::

## 在特定服务时间中包含项目

如果您的计划有多个服务时间（例如，早上 8 点和 10 点的服务），您可以选择每个项目应出现在哪些服务中。这在公告仅与一个服务相关，或歌曲在一个服务中唱诵但在另一个服务中不唱诵时很有用。

1. 打开一个在 **Times** 选项卡上定义了两个或多个服务时间的计划。
2. 在 **Service Order** 选项卡上，点击一个项目来编辑它。
3. 在 **Include in Services** 下，您会看到每个服务时间的复选框，标有时间。
4. 取消选中应该跳过该项目的任何服务。
5. 点击 **Save**。

默认情况下，每个项目包含在每个服务中。被排除的服务时间在您打印针对该服务的计划时被隐藏，因此每个服务只收到包含相关项目的清晰排练单。

:::tip
当计划有多个服务时间时，**Include in Services** 部分才出现。如果您只看到一个服务，请先编辑计划的 **Times** 选项卡来添加其他服务。
:::

## 分享服务顺序

一旦您的服务顺序完成，您可以从计划详情页面打印完整的计划（包括服务顺序）。这给您的团队提供了服务的完整概览。

## 后续步骤

- 在[歌曲](./songs.md)页面上管理您的敬拜曲目库
- 在[计划](./plans.md) Assignments 选项卡上为服务分配志愿者
- 为与服务相关的任何后续项目创建[任务](./tasks.md)
