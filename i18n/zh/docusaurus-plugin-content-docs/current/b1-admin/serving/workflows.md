# 工作流

<div class="article-intro">

工作流在视觉板上将人们移动通过一系列步骤。每个人都变成一张卡片，从一个步骤移动到下一个 -- 从首次访客跟进到成员过程再到首次捐赠者感谢，以及任何其他你需要追踪许多人通过相同一组阶段的事情。一个步骤可以要求志愿者去做某事（打电话、进行谈话）**以及**自动运行操作 -- 发送电子邮件、等待几天、添加到小组 -- 所以工作流同时处理人工跟进和其周围的重复性工作。工作流将[任务](./tasks.md)扩展到拖放 Kanban 板上，所以没有任何事情或没有人会从缝隙中漏落。

</div>

<div class="prereqs">
<h4>开始之前</h4>

- 确保你想要追踪的人存在于 B1 Admin 中
- 熟悉[任务](./tasks.md)的工作方式，因为板上的每张卡片都是一个任务
- 要使用**发送电子邮件**操作，首先创建你想要发送的电子邮件模板（在**消息传送 → 管理模板**下管理）
- 你将需要适当的任务权限。查看、编辑卡片和管理工作流是分开的权限级别（详见[角色和权限](../settings/roles-permissions.md)）

</div>

## 查看工作流

导航到**服侍**，打开**任务**区域，并从菜单中选择**工作流**。你会看到你的工作流被列出并按类别分组，活跃工作流被突出显示。单击任何工作流来打开其板。

## 创建工作流

1. 在工作流页面上，单击**添加工作流**。
2. 选择如何开始：
   - **空白工作流** -- 从头开始并构建你自己的步骤。
   - **从模板** -- 以可编辑的现成步骤集开始。内置模板包括：
     - **新访客跟进** -- 发送欢迎电子邮件 → 个人电话 → 邀请下一步 → 已连接
     - **成员课程** -- 表达兴趣 → 注册课程 → 参加课程 → 完成成员身份
     - **首次捐赠者感谢** -- 发送感谢信 → 分享捐赠影响 → 已管理
3. 给工作流一个**名字**。
4. 可选地分配一个**类别**来将相关工作流分组在一起。你可以直接从下拉菜单创建一个新类别。
5. 保持工作流**活跃**以便人们可以被添加到它，或将其设置为**不活跃**以从添加到工作流列表中隐藏它。
6. 单击**保存**。

:::tip
在工作流列表上使用**复制**按钮来复制现有工作流 -- 包括其步骤、自动化操作和路由 -- 作为新工作流的起点。
:::

## 使用步骤构建板

每个工作流板由**步骤**组成，从左到右显示为列。打开工作流并使用**添加步骤**来创建过程的每个阶段。

当你添加或编辑一个步骤时，你可以配置：

- **步骤名字** -- 列标题（例如"欢迎电话"或"等待注册"）。
- **在 (天数) 内到期** -- 当卡片进入此步骤时自动设置到期日期。过期的卡片被标记为**逾期**。
- **默认被分配者** -- 在此步骤上的新卡片自动被分配给的人或小组。
- **自动化操作** -- 当卡片到达时系统自动执行的事情（见下文）。
- **路由** -- 卡片离开步骤时的去向（详见[使用结果和条件路由卡片](#routing-cards-with-outcomes-and-conditions)）。

将步骤列拖动到与你的过程匹配的顺序。顺序也定义了当没有其他路由应用时卡片所采用的默认路径。

:::info
首先保存新步骤。自动化操作和路由附加到步骤，所以一旦步骤存在，编辑器就会解锁这些部分。
:::

## 自动化操作

每个步骤都可以包含**自动化操作**列表，这些操作在卡片**进入**步骤时自动运行 -- 在任何人接触它之前。这是一个步骤如何既提示志愿者*又*处理跟进周围的例行工作的方式。

在步骤编辑器中，打开**自动化操作**，单击**添加操作**，选择一个类型，填写其设置，并单击该操作上的保存图标。根据需要添加尽可能多的操作；它们**从上到下按顺序**运行。

| 操作 | 它做什么 |
|---|---|
| **发送电子邮件** | 向该人发送你选择的电子邮件模板。你可以覆盖主题行。 |
| **等待** | 在继续之前暂停卡片一定天数（见下文）。 |
| **添加到小组** | 将该人添加到你选择的[小组](../groups/index.md)。 |
| **添加到工作流** | 在另一个工作流上开始该人 -- 对于在过程之间交接很有用。 |
| **添加备注** | 在卡片的历史中记录一条备注。 |
| **设置字段** | 更新人员记录上的字段：成员身份、婚姻状态、性别、城市、州或邮编。 |
| **Webhook** | 将卡片的详情发送到你提供的外部网址 (URL)，用于连接到其他系统。 |

在步骤的所有操作完成后，卡片**停留在该步骤**上以便一个人可以处理它 -- 除非该步骤有一个自动路由将其向前移动（详见[完全自动化步骤](#fully-automated-steps)）。

:::info
自动化操作仅在卡片通过正常流进入时运行 -- 当它首次添加、当结果或自动路由将其带入或在等待完成后。当员工手动将卡片拖动到步骤或将其发送回去时，它们**不**重新运行，所以一个人不会收到相同的电子邮件两次。
:::

### 发送电子邮件

选择**发送电子邮件**，选择你的电子邮件模板之一，并可选地输入自定义主题。当卡片进入步骤时，该人自动收到该电子邮件。（如果该人档案上没有电子邮件地址，该步骤只是跳过此操作。）

### 等待几天（滴灌序列）

**等待**操作将卡片保留你设置的天数。当它等待时，卡片显示为**已打盹**。当等待结束时：

1. **相同步骤上的任何剩余操作**运行 -- 这样你可以构建一个滴灌，如**发送电子邮件 → 等待 3 天 → 发送提醒电子邮件**。
2. 然后，如果步骤有一个自动路由，卡片向前移动；否则它停留在步骤上供一个人拿起。

:::tip
一个步骤最开始的**等待**是在它浮出到志愿者之前"保留"卡片的简单方式 -- 例如，*等待 7 天，然后教练联系*。
:::

## 将人员添加为卡片

有多种方式可以将人员放在板上：

- **从板上** -- 单击步骤列底部的**添加卡片**并选择一个人。你也可以选择一个小组，该小组的每个成员都被添加为一张卡片。
- **从一个人的记录** -- 在一个人的页面上使用**添加到工作流**来将他们放到一个工作流。
- **从成员搜索** -- 选择多个人并使用批量**添加到工作流**操作来一次全部添加。
- **自动触发** -- 当发生某事时添加人员，如表格提交或首次捐赠（详见下面的[触发器](#triggers)）。

## 处理板

打开一个工作流来看到其板。每张卡片显示该人的名字、它被分配给谁以及一个到期日期或状态筹码（**逾期**或**已打盹**）。步骤列也显示其运行的任何自动化操作的小徽章和其路由的注释，给你一个关于卡片如何流动的一览无余的地图。

- **移动卡片** -- 当该人向前推进时，将卡片从一列拖到下一列。
- **打开卡片** -- 双击一张卡片（或单击它）来打开其详情抽屉，你可以在其中改变步骤、重新分配它、添加备注并审查已经发生的事情。

从卡片抽屉你可以：

- **分配**卡片给不同的人或小组。
- **打盹**卡片 1 天、3 天或 1 周来临时隐藏其到期日期。
- **发送回**上一个步骤或**跳过**到下一个步骤。
- **固定分配** -- 即使当卡片在步骤之间移动时也保持相同的所有者。默认情况下，将卡片移动到新步骤会将其重新分配给该步骤的默认被分配者；固定保持当前人员在整个过程中负责。
- **完成**卡片来完成它，或如果步骤配置了结果，选择一个**结果**按钮（详见[路由](#routing-cards-with-outcomes-and-conditions)）。
- **添加备注**并审查卡片的**历史** -- 包括已运行的自动化操作的日志（发送的电子邮件、等待等）。

### 批量操作

选择多张卡片上的复选框来一起对它们采取行动。一个工具栏出现让你**完成**、**打盹**、**重新分配**或一次**移动**所有选定的卡片到另一个步骤。

## 使用结果和条件路由卡片

路由控制卡片离开步骤时的去向。打开步骤的编辑器来配置两种路由。

### 结果按钮

结果是当你在该步骤上完成卡片时卡片抽屉上显示的按钮。与单个**完成**按钮不同，你可以提供选择，如"加入了小组"或"不感兴趣"。每个结果都可以：

- 将卡片发送到**此工作流中的另一个步骤**，
- **交接卡片**到完全不同的工作流，或
- **关闭**卡片。

这让一个决定将该人分支到不同的路径。

### 自动路由（条件）

自动路由在卡片**进入步骤时立即**移动卡片（在其自动化操作完成后），不需要任何人点击，如果该人符合一组条件。添加一个路由，选择目标步骤，并定义一个或多个**条件**（例如，一个人的分校区、年龄或成员身份）。一个没有条件的路由匹配所有人。

:::info
在板上，每个步骤列显示描述其路由的小注释 -- 例如，结果标签或"如果匹配"后跟一个箭头到目标步骤或工作流。
:::

## 完全自动化步骤

你可以使一个步骤完全自动运行，没有人处理它。给步骤其**自动化操作**并添加一个**自动路由**（无条件）指向下一个步骤。当卡片进入时，操作运行，然后路由立即推进它 -- 卡片直接通过。

:::tip
与**等待**结合：*发送欢迎电子邮件 → 等待 3 天 → 自动推进到"个人电话"步骤*。电子邮件和时间为你处理，志愿者仅在该时间进行人工操作时才看到卡片。
:::

## 触发器

触发器在发生某事时自动将人员添加到工作流，所以你永远不用手动添加卡片。在工作流板上，单击**触发器**选项卡，然后**添加触发器**。有两种：

### 事件触发器

当 B1 中的记录改变时立即触发。选择事件，然后可选地添加**条件**以仅添加匹配的人：

- **人员 · 创建 / 更新** -- 例如添加任何状态变为*访客*的人。
- **捐赠 · 创建** -- 例如向感谢工作流添加首次或大额捐赠（在金额、基金或方法上匹配）。
- **小组 · 成员加入** / **小组 · 已创建**。
- **表格 · 已提交** -- 添加提交选定表格的任何人（对于"我是新的"或"连接"卡片很好用）。

### 日程触发器

对一组条件定期运行 -- 每天、每周、每月或每年。使用这些进行基于时间的外展，如*每个成员身份周年纪念日是今天的人*或一个*每月*检查。

对于任何触发器，你也可以设置：

- 新卡片开始的**进入步骤**（默认为首个步骤）。
- **每个人一次** -- 这样触发器不会将同一个人添加到工作流两次。
- **活跃** -- 无需删除它就打开或关闭触发器。

:::tip
将一个**表格 · 已提交**触发器与**新访客跟进**模板配对，将你的"连接卡片"或"我是新的"表格转变为自动跟进管道。
:::

## 我的卡片

志愿者和员工不需要深入挖掘每个板来找到他们的工作。**我的卡片**页面（从工作流页面链接）列出分配给当前用户的每张卡片，跨所有工作流。单击卡片打开它所属的板。

## 报告

打开一个工作流并单击**报告**来查看该工作流的分析：

- **逾期** -- 过期的卡片数。
- **每个步骤的卡片** -- 目前在每个步骤上有多少张卡片，显示为柱状图。
- **已完成 (30 天)** -- 过去 30 天的吞吐量，显示为线形图。

使用这些来发现瓶颈 -- 例如，一个卡片堆积且从不向前的步骤。
