# 服事计划

<div class="article-intro">

服事计划组织谁在什么时候服事。每个计划都与特定的日期和事工相关联，使您可以逐周协调志愿者团队并确保每个服事都配备充足的人员。

</div>

<div class="prereqs">
<h4>开始前</h4>

- 在服事区域设置您的事工和团队
- 确保志愿者已添加到您的[人员名录](../people/adding-people.md)并分派给团队

</div>

## 访问计划

1. 从主菜单导航到 **Serving**（服事）。
2. 在页面顶部选择一个 **ministry tab**（事工标签页）。
3. 点击一个 **plan type**（计划类型）以查看该类型的计划列表。
4. 点击特定计划以打开它。

:::info
管理完全访问权限不是管理计划所必需的。任何事工成员都可以导航到服事部分，为自己的事工创建、编辑和安排计划，无需计划编辑权限。具有计划编辑角色的编辑可以跨所有事工管理计划。
:::

## 创建计划

1. 从计划类型视图中，点击 **New Plan**（新建计划）。
2. 给计划起名或使用日期作为名称。选择服事的 **date**（日期）。
3. 如果您想从之前的计划复制，请选择仅职位或职位和分派。如果您不想复制，只需选择不复制。您也可以复制上一个计划的服事顺序。
4. 保存计划。您现在可以开始分派团队成员并构建[服事顺序](./service-order.md)。

## 计划详细信息页面

当您打开计划时，您会看到两个标签页：

- **Assignments**（分派）— 管理哪些团队成员被分派到此计划。您可以从现有团队添加人员，并查看谁已确认或仍待确认。
- **[Service Order](./service-order.md)**（服事顺序）— 使用敬拜歌曲、祷告、公告和讲道等元素构建服事顺序。

## 分派团队成员

1. 打开一个计划并转到 **Assignments**（分派）标签页。
2. 点击 **add Position** 展开它。在添加职位表单中填写信息。对于类别名称，添加您喜欢的任何类别。
3. 点击 **People Needed**（需要人员）并选择志愿者来填充该职位。
4. 通过点击 **Add** 从您的团队名册添加成员。
5. 分派的成员将与他们的分派状态一起显示在其团队下。
6. 点击通知志愿者以在 B1 应用中或通过电子邮件通知他们。

:::tip
在创建计划之前在事工设置中设置您的团队。这样，您将有一个现成的志愿者池来从中分派。
:::

## 志愿者提醒

B1 可以在志愿者计划服事的前自动提醒他们，以便您每周都不必追踪您的团队。提醒发送给 **everyone scheduled**（所有计划的人）— 那些已确认的和那些还未回应的 — 通过电子邮件和应用内/推送通知。每个提醒包括志愿者的职位、服事日期、计划备注和您的自定义消息。

提醒的时间和内容是按 **plan type**（计划类型）设置的，因此每种类型的服事都可以保持自己的日程。

1. 从 **Serving**（服事）区域，选择包含计划类型的事工。
2. 点击计划类型旁边的 **edit（pencil）icon**（编辑铅笔图标）。
3. 在 **Reminders**（提醒）部分，设置：
   - **Reminder days before service**（服事前提醒天数）— 以逗号分隔的提前发送多少天的列表，例如 `7,1,0`。使用 `0` 在服事当天发送提醒。将此字段留空以关闭此计划类型的提醒。
   - **Custom reminder message**（自定义提醒消息）*（可选）* — 添加到提醒的额外文本，例如"提前 30 分钟到达以进行排练。"
4. 保存计划类型。

新计划类型默认在每个服事 **2 天前** 提醒志愿者，直到您更改此设置。

:::tip
尚未确认的志愿者在提醒电子邮件中获得 **Accept** 和 **Decline** 按钮，因此他们可以在不登录的情况下做出回应。
:::

:::info
每个提醒只发送一次。仍在草稿中（尚未发送给团队）的计划不会触发提醒。
:::

## 将组与计划类型关联

在计划类型页面上计划列表下方，**Groups**（组）部分让您决定哪些组可以从其成员门户查看此计划类型的计划。这是一个向正确的团队展示即将进行的服事的快速方式，而无需给予他们管理员访问权限。

1. 在计划类型页面上，向下滚动到 **Groups**（组）部分。
2. 点击 **Add Group**（添加组）并从下拉菜单中选择一个组。
3. 在 **Shows**（显示）列中，选择该组的成员应该看到该计划类型的 **Past**（过去）、**Future**（未来）还是 **Both**（两者）计划。
4. 重复以关联其他组，或点击垃圾桶图标以移除一个组。

:::info
只有标记为 **Standard**（标准）的组才会出现在选择器中。关联组的成员会自动在 B1 成员门户的 [Plans](/docs/b1-church/plans/)（计划）标签页中看到此计划类型的计划 — 限制在您选择的过去/未来/两者的窗口内。
:::

## 打印计划

您可以打印一个计划以分发给您的团队。打开计划、打开服事顺序标签页并使用 **Print** 选项生成可打印版本，包括分派和服事顺序。这对于在排练中分发或贴在公共区域很有用。

:::info
计划按事工组织。在创建或查看计划前，确保您在正确的事工标签页上。
:::

## 后续步骤

- 将计划的结构保存为[计划模板](./plan-templates.md)，以便您可以一键将其邮票到未来的计划
- 使用歌曲、诵读和其他元素构建[服事顺序](./service-order.md)
- 直接将[歌曲](./songs.md)从您的库添加到服事顺序
- 使用[任务](./tasks.md)为团队成员分派后续行动项
