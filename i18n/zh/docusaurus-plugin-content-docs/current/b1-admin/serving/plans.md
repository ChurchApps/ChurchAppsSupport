# 服侍计划

<div class="article-intro">

服侍计划组织谁在服侍以及何时服侍。每个计划都与特定日期和事工相关联，使您可以轻松地按周协调志愿者队伍，并确保每个服务都有充足的人员。

</div>

<div class="prereqs">
<h4>开始前</h4>

- 在服侍区域设置您的事工和团队
- 确保志愿者已被添加到您的[人员目录](../people/adding-people.md)并分配到团队

</div>

## 访问计划

1. 从主菜单导航到 **Serving**。
2. 在页面顶部选择一个**事工选项卡**。
3. 点击一个**计划类型**以查看该类型的计划列表。
4. 点击特定计划来打开它。

:::info
管理计划不需要完全的管理员访问权限。任何属于某个事工的成员都可以导航到"Serving"并为其自己的事工创建、编辑和安排计划，而无需"Plans Edit"权限。具有"Plans Edit"角色的编辑可以管理每个事工的计划。
:::

## 创建计划

1. 从计划类型视图中，点击 **New Plan**。
2. 为计划命名或使用日期作为名称。选择服务的**日期**。
3. 如果您想从之前的计划复制，请选择仅位置或位置和分配。如果您不想复制，就不选择任何东西。您还可以从我以前的计划中复制服务顺序。
4. 保存计划。现在您可以开始分配团队成员并构建[服务顺序](./service-order.md)。

## 计划详情页面

当您打开计划时，您会看到两个选项卡：

- **Assignments**——管理分配到此计划的团队成员。您可以添加现有团队中的人员，并查看谁已确认或仍处于待定状态。
- **[Service Order](./service-order.md)**——使用敬拜歌曲、祷告、公告和讲道等元素构建服务顺序。

## 分配团队成员

1. 打开计划并转到 **Assignments** 选项卡。
2. 点击 **add Position** 来展开它。在添加位置表单中填写信息。对于类别名称，添加您喜欢的任何类别。
3. 点击 **People Needed** 并选择志愿者来填充该位置。
4. 通过点击 **Add** 从您的团队名单中添加成员。
5. 分配的成员会在其团队下方显示，并显示其分配状态。
6. 点击通知志愿者来在 B1 应用中或通过电子邮件通知他们。

:::tip
在创建计划之前，在事工设置中设置您的团队。这样，您将拥有一个随时可用的志愿者资源库来分配。
:::

## 将组关联到计划类型

在计划类型页面的计划列表下方，**Groups** 部分允许您决定哪些组可以从其成员门户看到此计划类型的计划。这是一种快速方式，可以向正确的团队显示即将到来的服务，而无需授予他们管理员访问权限。

1. 在计划类型页面上，向下滚动到 **Groups** 部分。
2. 点击 **Add Group** 并从下拉菜单中选择一个组。
3. 在 **Shows** 列中，选择该组的成员是否应该看到此计划类型的 **Past**、**Future** 或 **Both** 计划。
4. 重复以关联其他组，或点击垃圾箱图标来移除组。

:::info
只有标记为 **Standard** 的组出现在选择器中。关联组的成员会自动在 B1 成员门户的 [Plans](/docs/b1-church/plans/) 选项卡上看到此计划类型的计划——仅限于您选择的过去/未来/两者时间窗口。
:::

## 打印计划

您可以打印计划来分发给您的团队。打开计划，打开服务顺序选项卡并使用 **Print** 选项来生成可打印版本，其中包括分配和服务顺序。这对于在排练时分发或在公共区域张贴很有用。

:::info
计划按事工组织。在创建或查看计划之前，请确保您在正确的事工选项卡上。
:::

## 后续步骤

- 使用 [Plans Overview](./plans-overview.md) 在一个网格中查看多个周的所有即将到来的分配，并找出未填充的位置
- 使用歌曲、阅读和其他元素构建您的[服务顺序](./service-order.md)
- 直接从您的库中添加[歌曲](./songs.md)到服务顺序中
- 使用[任务](./tasks.md)为团队成员分配后续行动项
