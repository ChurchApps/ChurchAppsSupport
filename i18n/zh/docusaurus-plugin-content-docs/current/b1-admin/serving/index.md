# 服事

<div class="article-intro">

服事部分是协调志愿者和规划教会服务的中心。无论您是组织主日敬拜、平日聚集还是特殊活动，这里是您汇集一切的地方，让您的团队确切地知道谁在什么时候服事。

</div>

## 服事页面如何工作

当您打开 **Serving**（服事）页面时，您会看到您的事工显示为顶部的标签页。每个事工都有自己的计划类型和团队，因此您可以独立管理多个事工。例如，您可能有单独的标签页用于敬拜、儿童事工和款待服事。

属于某个事工的非管理员工作人员也可以访问服事页面。他们只能看到自己所属的事工，并且可以为自己的事工创建和编辑计划、分派、计划类型和服事顺序，无需完整的管理员权限。

:::tip
如果您是管理员，切换 **Show All** 可同时查看所有事工。当您需要检查多个团队的分派时，这很有帮助。
:::

## 子部分

服事部分包括以下关键区域：

- **[Plans](./plans.md)**（计划）— 为特定日期创建服事计划并将志愿者分派给团队。计划帮助您组织谁在什么时候服事，使每个人都能提前了解自己的角色。设置自动[志愿者提醒](./plans.md#volunteer-reminders)，使您的团队在每次服事前收到通知，并使用 **[Plans Overview](./plans-overview.md)**（计划概览）在一个网格中查看多周的所有即将进行的分派。
- **[Songs](./songs.md)**（歌曲）— 管理您的敬拜歌曲库，包括歌词、编排和外部链接。您的敬拜团队可以使用这些来为即将进行的服事做准备。
- **[Tasks](./tasks.md)**（任务）— 为您的团队成员分派和跟踪任务。设置[自动化](./automations.md)以自动创建定期任务，这样就不会有任何遗漏。
- **[Workflows](./workflows.md)**（工作流）— 通过视觉看板将人员移动到一系列步骤中，例如新访客跟进或成员流程。步骤可以运行自动化操作（发送电子邮件、等待几天、添加到组），卡可以通过结果按钮、条件路由或事件和计划触发器进行推进 — 具有内置报告。

## 入门

1. 从 B1 Admin 的主菜单导航到 **Serving**（服事）。
2. 点击添加事工按钮以创建新事工或切换"show all"查看现有事工标签页。选择一个 **ministry tab**（事工标签页）以查看其计划类型和团队。
3. 选择"create a plan type"或选择现有计划类型以开始[创建服事计划](./plans.md)，或探索 [Songs](./songs.md) 和 [Tasks](./tasks.md) 子部分。

:::info
在您可以创建计划之前，您需要在服事区域创建您的事工，并在每个事工中创建团队。
:::
