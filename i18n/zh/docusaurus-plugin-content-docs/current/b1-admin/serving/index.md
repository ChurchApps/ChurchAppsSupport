# 服侍

<div class="article-intro">

服侍部分是你协调志愿者和计划教会服务的中心。无论你是组织周日敬拜、周间聚集还是特殊活动，这就是你汇聚一切的地方，这样你的团队就会确切知道谁在侍奉以及何时侍奉。

</div>

## 服侍页面如何工作

当你打开**服侍**页面时，你会看到你的事工显示为顶部的选项卡。每个事工都有自己的计划类型和团队，所以你可以独立管理多个事工。例如，你可能有为敬拜、儿童事工和款待分别设置的选项卡。

非管理员员工如果是某个事工的成员，也可以访问服侍页面。他们只会看到他们所属的事工，并可以在没有完全管理员权限的情况下为他们的事工创建和编辑计划、分配、计划类型和服务顺序。

:::tip
如果你是管理员，切换**显示全部**来一次查看所有事工。这在你需要检查多个团队的分配时很有帮助。
:::

## 子部分

服侍部分包括这些关键领域：

- **[计划](./plans.md)** -- 为特定日期创建服侍计划并将志愿者分配给团队。计划帮助你组织谁在何时侍奉，所以每个人都提前知道他们的角色。使用**[计划概览](./plans-overview.md)**在一个网格中查看多周内的所有即将到来的分配。
- **[歌曲](./songs.md)** -- 管理你的敬拜歌曲库，包含歌词、编排和外部链接。你的敬拜团队可以用这个为即将到来的服务做准备。
- **[任务](./tasks.md)** -- 为你的团队成员分配和追踪任务。设置[自动化](./automations.md)以自动定期创建任务，这样就不会有任何事情从缝隙中漏落。
- **[工作流](./workflows.md)** -- 在视觉 Kanban 板上将人们移动通过一系列步骤，例如新访客跟进或成员过程。步骤可以运行自动化操作（发送电子邮件、等待几天、添加到小组），卡片通过结果按钮、条件路由或事件和日程触发器推进 -- 包括内置报告。

## 开始使用

1. 从 B1 Admin 中的主菜单导航到**服侍**。
2. 单击添加事工按钮来创建一个新事工或切换"显示全部"来查看现有事工选项卡。选择一个**事工选项卡**来查看其计划类型和团队。
3. 选择"创建计划类型"或选择现有计划类型来开始[创建服侍计划](./plans.md)，或探索[歌曲](./songs.md)和[任务](./tasks.md)子部分。

:::info
在你能创建计划之前，你需要在服侍区域中创建你的事工，以及在每个事工中创建团队。
:::
