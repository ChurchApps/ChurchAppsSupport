# 计划验证与志愿者通知

<div class="article-intro">

B1 Admin 会在周日之前自动检查您的计划是否存在问题 -- 未填充的职位、时间冲突以及已标记该日期不可用的志愿者。当一切就绪后，您可以一键通知整个团队。

</div>

<div class="prereqs">
<h4>开始之前</h4>

- 创建一个[服务计划](./plans.md)并将志愿者分配到职位
- 向计划中添加[礼拜时间](./plans.md)，以便冲突检测可以检查时间重叠
- 确保志愿者已安装 B1 Mobile 应用以接收推送通知

</div>

## 验证面板

每个计划都有一个**验证**面板，在您构建计划时自动运行。它检查三项内容：

### 未填充的职位
如果某个职位需要的人数多于当前已分配的人数，验证面板会准确列出仍然需要什么 -- 例如，*"音响技术员：还需要1人。"* 您可以一目了然地看到在本周到来之前您的计划是否已完全配备人员。

### 时间冲突
如果一名志愿者被分配到同一计划中时间重叠的两个职位，验证面板会标记冲突 -- 例如，*"Jane Smith：主日礼拜期间敬拜主领和儿童签到之间的时间冲突。"* 这可以在周日早晨出问题之前发现重复预订。

### 屏蔽日期
志愿者可以在 B1 Mobile 中设置他们不可用的日期。如果某人被分配到的计划与他们的屏蔽日期重叠，验证面板会自动显示冲突，以便您找到替代人员。

### 跨计划冲突
验证还会同时检查您的所有计划。如果同一名志愿者被分配到两个时间重叠的不同计划中 -- 例如，上午9点的礼拜和上午10点的礼拜都持续到上午10:30 -- B1 Admin 会将该人员标记为跨计划重复预订。

:::tip
您无需执行任何操作来运行验证 -- 每次添加或更改分配时它都会自动更新。在构建计划时只需关注面板即可。
:::

## 通知志愿者

计划准备就绪后，您可以直接从验证面板一次性通知所有已分配的志愿者。

1. 打开计划并滚动到**验证**面板
2. 如果有未通知的志愿者，您将看到一个链接，显示需要通知多少人（例如，*"通知8名志愿者"*）
3. 点击链接向所有尚未收到通知的人发送推送通知
4. 志愿者会在手机上收到通知，告知他们已被安排，并提示他们确认分配

:::info
只有尚未收到通知的志愿者才会被包含在内。如果您稍后将某人添加到计划中，链接将重新出现，以便您通知新添加的人而不会重复通知团队其他成员。
:::

:::warning
志愿者必须安装 B1 Mobile 应用并启用通知才能接收推送通知。请参阅[通知指南](/docs/b1-mobile/community/notifications)了解志愿者如何在其设备上启用此功能。
:::
