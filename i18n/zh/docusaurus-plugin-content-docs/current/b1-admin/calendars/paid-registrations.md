# 付费注册

<div class="article-intro">

事件注册可以超越简单的人数统计。您可以定义价格参与者类型（如成人和儿童）、提供带有自己的价格和数量的可选附加项、创建折扣代码，并在注册时通过您教会的现有奉献提供商收取付款。当事件满时，可选的候补名单让感兴趣的成员排队，并随着空位打开自动升级他们。

</div>

<div class="prereqs">
<h4>开始之前</h4>

- 首先在事件上启用注册——参见 [创建日历](creating-calendars#enabling-event-registration)
- 要收取付款，您的教会需要 [配置在线奉献](../donations/online-giving-setup.md)（Stripe、PayPal 或 Kingdom Funding）。免费事件不需要奉献设置。

</div>

## 打开注册设置

1. 在 B1 Admin 中，转到 **注册** 页面并打开您的事件（或从其日历打开事件）。
2. **注册设置** 卡显示基础——**启用注册**、**容量**、**注册打开/关闭**、**标签** 和 **注册问题**。
3. 基础下方有三个手风琴：**参与者类型**、**选项** 和 **折扣代码**。

## 参与者类型

参与者类型让您为不同种类的参与者收取不同的价格——并分别为每个参与者设置限制。

1. 展开 **参与者类型** 手风琴并点击 **添加类型**。
2. 输入 **名称**（例如 "成人"、"儿童"、"学生"）。
3. 设置 **价格**。对于免费类型使用 0。
4. 可选地为仅此类型设置 **容量**（例如仅 20 个儿童名额）。留空表示无按类型限制。
5. 点击 **保存**。

在注册期间，每个参与者选择一个类型；售完的类型显示为 **已售完** 且无法选择。名单显示每个参与者的类型和运行的按类型计数。

## 选项

选项是可选的价格附加项——T 恤、膳食计划、活动升级。

1. 展开 **选项** 手风琴并点击 **添加选项**。
2. 输入 **名称**、可选 **描述** 和 **价格**（0 显示为"免费"）。
3. 可选地设置 **容量**（所有注册中总共可用）和 **最大数量**（一个注册可以订购的最多）。
4. 点击 **保存**。

注册者在注册期间选择数量，总计计入容量，以便您永远不会超额销售。

## 折扣代码

1. 展开 **折扣代码** 手风琴并点击 **添加折扣代码**。
2. 输入注册者将键入的 **代码**。
3. 选择 **类型**——**百分比** 或 **金额**——及其 **值**。
4. 可选地用 **开始日期** / **结束日期**、**最少成员**（注册上的最少参与者数）和 **最多使用次数** 限制代码。
5. 点击 **保存**。

每个代码显示一个 **使用** 计数，以便您可以看到它被兑现的频率。注册者在应用代码时获得即时反馈——包括当代码已过期、尚未开始或需要更多参与者时的清晰消息。

## 候补名单

在注册设置卡中打开 **启用候补名单**。当事件达到容量时：

- 新的注册者被提供候补名单名额，而不是被拒绝。他们完成相同的注册（在候补名单上时跳过付款）。
- 当有人取消时，最早的候补名单注册被 **自动升级** 并收到一封电子邮件，说明一个名额已打开。如果他们欠余额，电子邮件会链接他们以完成付款。
- 您可以随时使用候补名单行上的 **升级** 操作手动升级某人——在提高事件容量后很有用。

:::info
升级的注册保持 *待处理* 直到任何余额被支付；支付（或无需支付）确认它们。
:::

## 注册名单

从注册页面打开事件以查看每个注册。表格显示 **姓名**、**成员**、**类型**（每个参与者的类型）、**已支付/总计**（当仍欠钱时带有余额警告）、**状态** 和 **日期**，加上表上方的按类型计数芯片。

- 点击行的详情图标以打开 **注册详情** 对话框——成员、选项、已支付/余额和 **付款** 表，列出每笔费用（金额、方法、日期）。
- **导出 CSV** 下载完整名单，其列为成员、参与者类型、选项、已支付/总计/余额、状态和每个注册问题一列。
- **添加参与者** 仍允许您手动记录离线注册。

:::info
退款不在 B1 内处理。如果您需要退款已取消的付费注册，请从您的奉献提供商的仪表板（例如 Stripe）发出退款。
:::

## 付款如何工作

付款通过您的教会已经用于奉献的相同奉献网关进行——卡详情直接进入提供商，永远不会接触 B1 的服务器。价格始终从您配置的类型、选项和折扣代码计算在服务器上，以便注册者无法篡改总计。已登录的成员可以用保存的卡支付；客人在结账时输入卡。
