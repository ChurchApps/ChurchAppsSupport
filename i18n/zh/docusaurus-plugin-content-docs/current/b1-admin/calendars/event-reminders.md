# 事件提醒

<div class="article-intro">

事件提醒自动在事件发生前通知相关人员 — 例如，"不要错过！医疗保健研讨会明天上午 9:00 开始。"你在事件上配置一次提醒，B1 通过推送通知和电子邮件按计划发送。成员可以从他们自己的[通知偏好设置](../../b1-church/getting-started/notification-preferences)中控制他们接收哪些提醒。

</div>

<div class="prereqs">
<h4>开始前</h4>

- 创建你想提醒人们的事件（参见[创建日历](creating-calendars)）
- 要联系已注册的参与者，[启用事件注册](creating-calendars)
- 要联系整个组群，确保事件属于有成员的[组群](../groups/creating-groups)

</div>

## 设置提醒

你在事件的**提醒**部分配置提醒。

- 当你**创建新事件**时，在保存前在事件编辑器中展开**提醒**部分。
- 对于**现有事件**，打开事件的**注册详情**页面（从**注册**部分）来添加或更改其提醒。

1. 打开**启用提醒**。
2. 选择**何时**发送。选择最多三个时间：**7 天前**、**3 天前**、**1 天前**和**当天**。
3. 设置提醒应该发出的**每天的时间**（默认为你所在教堂的本地时区的**上午 9:00**）。
4. 选择**谁**应该被提醒（参见下面的[谁会被提醒](#who-gets-reminded)）。
5. 可选地添加**消息**。留空以使用默认措辞，或写你自己的 — 你可以包括 `{{eventTitle}}`，它将被替换为事件的名称。
6. 选择**频道**：**推送**通知、**电子邮件**或两者。
7. 保存事件。

当你进行更改时，一个**实时预览**显示大约有多少人会被提醒、有多少参与者无法联系以及下一个计划的发送时间 — 所以你可以在保存前确认提醒看起来正确。

## 谁会被提醒

**谁**设置控制提醒发送给谁：

- **仅注册者** — 每个为事件注册且链接到人员记录的人。这是当事件启用注册时的默认设置，所以小型注册事件的提醒绝不会意外发送给整个组群。
- **仅组长/注册者** — 每个注册一个提醒（注册的人），而不是注册上的每个家庭成员。
- **组群成员** — 事件所属组群中的每个人。这是当事件不使用注册时的默认设置。
- **自动** — 在启用注册时使用注册者，否则使用组群。

:::info
仅由名字添加的客人（没有链接的人员记录）无法接收提醒，因为没有账户、设备或电子邮件可以发送。预览告诉你有多少参与者属于这个组群，所以没有意外。选择不进行交流的成员也会被跳过。
:::

## 提醒何时发送

- 提醒在你选择的**每天的时间**发出，在你教堂的本地时区，在你选择的每个偏移上。
- 如果你**更改事件的日期或时间**，待处理的提醒会自动重新计划 — 你不需要编辑提醒。
- 如果你**删除事件**（或取消定期事件的单次发生），其待处理的提醒会自动取消。
- 定期事件会自动处理：每个即将到来的发生都会获得自己的提醒。

:::tip
提醒**先发推送，以电子邮件作为备用**。如果成员启用了推送通知，他们会收到推送；如果没有，他们会收到电子邮件。成员在他们的[通知偏好设置](../../b1-church/getting-started/notification-preferences)中按通知类型选择他们想要的频道。
:::

## 成员可以控制什么

提醒始终尊重每个成员的[通知偏好设置](../../b1-church/getting-started/notification-preferences)。成员可以：

- 关闭推送或电子邮件的**事件提醒**，同时保持其他通知打开。
- 设置**静音时段**以便非紧急通知等到合理的时间。

你无法覆盖成员选择退出事件提醒 — 这使 B1 符合反垃圾邮件规则并让成员控制他们的收件箱。

## 服务提醒

在计划中安排的志愿者会收到一个单独的**服务提醒**，包含计划详情，以及当他们还没有回应时，**接受/拒绝**按钮就在电子邮件中。这些提醒在计划类型而不是日历事件上配置 — 参见[周日志愿者](../guides/sunday-volunteers)以了解志愿者安排和提醒如何工作。
