# Stripe 导入

<div class="article-intro">

如果您通过 Stripe 接受在线捐赠，Stripe 导入工具可让您将这些交易导入 B1 Admin，使所有奉献数据集中在一个地方。这对于捕获未自动同步的交易特别有用。

</div>

<div class="prereqs">
<h4>开始之前</h4>

- 完成[在线奉献设置](online-giving-setup.md)以将 Stripe 帐户连接到 B1 Admin
- 验证 Stripe 仪表板中有您要导入的日期范围内的捐赠

</div>

## 工作原理

导入过程使用两步流程：首先预览将要导入的内容，然后确认导入。这种预演方式让您可以在创建任何数据之前查看所有内容。

## 导入交易

1. 在 **B1 Admin** 中，导航到**捐赠 > 批次**。
2. 点击批次页面底部的 **Stripe 导入**链接。
3. 选择要导入的交易的**日期范围**。
4. 点击**预览**运行预演检查。

## 查看预览

预览显示来自 Stripe 的每笔交易及其状态指示器：

- **新** -- 此交易尚未导入，如果您继续将被包含。
- **已导入** -- 此交易已存在于 B1 Admin 中，将被跳过。
- **已跳过** -- 此交易因其他原因被排除（例如，退款或失败的收费）。

顶部的摘要部分显示每个类别中的交易总数和涉及的金额。

:::info
预览步骤不会创建任何记录。这是一个只读检查，让您可以在确认之前验证将要发生的事情。
:::

## 完成导入

1. 查看预览结果和摘要总计。
2. 点击**导入缺失项**以导入所有标记为**新**的交易。
3. 导入完成后，每笔交易旁边的状态标签更新以显示结果。

## 使用 Stripe 导入的提示

- 定期运行导入（每周或每月）以保持记录最新。
- 如果交易显示为**已导入**，表示 B1 Admin 中已有匹配记录——无需采取任何操作。
- 使用日期范围筛选器聚焦于特定时期（如果您在查找特定交易）。

:::tip
导入后，访问[批次](batches.md)页面验证导入的捐赠是否正确显示，以及总额是否与 Stripe 仪表板中看到的一致。
:::

## 后续步骤

- 查看[捐赠报告](donation-reports.md)以审查导入的交易和其他奉献数据
- 确保导入的捐赠被分配到正确的[基金](funds.md)，以获得准确的[奉献报表](giving-statements.md)
