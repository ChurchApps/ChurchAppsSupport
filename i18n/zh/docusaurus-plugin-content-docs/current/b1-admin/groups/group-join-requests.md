# 小组加入申请

<div class="article-intro">

当小组配置了基于批准的加入政策时，人们可以提交加入申请。小组领导者和管理员审查这些申请并批准或拒绝。这让您的教会能够控制小组成员，同时使人们很容易表达加入的兴趣。

</div>

<div class="prereqs">
<h4>开始前</h4>

- 您需要小组管理权限，或您需要是特定小组的领导者。有关详细信息，请参阅 [角色与权限](../people/roles-permissions.md)。
- 小组必须将其加入政策设置为 **申请**（需要批准）。有关如何配置加入政策的信息，请参阅 [创建小组](./creating-groups.md)。

</div>

## 理解加入政策

小组可以有三种不同的加入政策：

- **开放** -- 任何人都可以立即加入，无需批准
- **申请** -- 人们提交需要批准的加入申请
- **关闭** -- 没有人可以请求加入（成员必须手动添加）

当小组使用 **申请** 政策时，所有加入尝试都通过本页面描述的批准工作流程进行。

## 查看待处理申请

### 对于小组领导者

1. 在 B1 Admin 中导航至 **小组**
2. 点击小组名称
3. 点击 **加入申请** 标签
4. 您将看到该小组的所有待处理申请

### 对于管理员

具有小组管理权限的管理员可以查看所有小组的待处理申请：

1. 在 B1 Admin 中导航至 **小组**
2. 查找通知徽章或待处理申请指示器
3. 点击查看全教会范围内的所有待处理申请

## 审查加入申请

每个加入申请显示：

- **申请人的名字和照片** -- 请求加入的人
- **可选消息** -- 说明他们为什么想加入的个人消息（如果提供）
- **申请日期** -- 申请提交的时间

要审查申请：

1. 如果申请人提供了消息，请阅读该消息
2. 点击申请人的名字查看他们的个人资料（如果需要）
3. 决定是批准还是拒绝

## 批准申请

1. 点击加入申请上的 **批准**
2. 申请人立即被添加到小组作为成员
3. 申请人收到申请被批准的通知
4. 申请在系统中被标记为已批准

:::tip
当您批准申请时，申请人将成为普通小组成员。您稍后可以从 [小组成员](./group-members.md) 页面将其升级为小组领导者。
:::

## 拒绝申请

1. 点击加入申请上的 **拒绝**
2. 可选地提供拒绝的原因（最多 500 个字符）
3. 点击 **确认**
4. 申请人收到通知，其中包含您的拒绝原因（如果提供）
5. 申请在系统中被标记为已拒绝

:::info
提供拒绝原因有助于申请人了解为什么他们的申请未被批准，并可能鼓励他们稍后再试或探索其他小组。
:::

## 通知

加入申请系统自动发送通知：

- **当提交申请时** -- 所有小组领导者收到通知
- **当申请被批准时** -- 申请人收到确认
- **当申请被拒绝时** -- 申请人收到包含任何拒绝原因的通知

通知出现在用户在 B1.church 和移动应用中的通知中心。

## 从成员角度管理申请

人们可以从 B1.church 管理他们自己的加入申请：

- 在小组详情页面查看他们待处理申请的状态
- 如果改变主意，取消待处理的申请
- 查看他们的申请是否被批准或拒绝

## 最佳实践

- **及时回应** -- 尝试在 24-48 小时内审查申请，以便人们不会一直等待
- **拒绝原因要清晰** -- 帮助人们了解后续步骤或替代选项
- **检查个人资料** -- 查看申请人的个人资料，看看他们是否适合该小组
- **沟通期望** -- 确保您的小组描述清楚地说明了小组的目标人群
