# 小组加入请求

<div class="article-intro">

当小组配置为基于批准的加入政策时,人们可以提交加入请求。小组领导者和管理员审查这些请求并批准或拒绝它们。这使您的教会能够控制小组成员资格,同时让人们轻松表达加入的兴趣。

</div>

<div class="prereqs">
<h4>开始之前</h4>

- 您需要管理小组的权限,或者您需要成为特定小组的领导者。详情请参阅[角色与权限](../people/roles-permissions.md)。
- 小组必须将其加入政策设置为 **请求**(需要批准)。有关如何配置加入政策,请参阅[创建小组](./creating-groups.md)。

</div>

## 了解加入政策

小组可以有三种不同的加入政策:

- **开放** -- 任何人都可以立即加入,无需批准
- **请求** -- 人们提交需要批准的加入请求
- **封闭** -- 没有人可以请求加入(必须手动添加成员)

当小组使用 **请求** 政策时,所有加入尝试都会经过本页面描述的批准工作流程。

## 查看待处理请求

### 对于小组领导者

1. 在 B1 Admin 中导航到 **小组**
2. 点击小组名称
3. 点击 **加入请求** 选项卡
4. 您将看到此小组的所有待处理请求

### 对于管理员

具有小组管理权限的管理员可以查看所有小组的待处理请求:

1. 在 B1 Admin 中导航到 **小组**
2. 查找通知徽章或待处理请求指示器
3. 点击查看全教会范围内的所有待处理请求

## 审查加入请求

每个加入请求显示:

- **人员姓名和照片** -- 请求加入的人
- **可选消息** -- 解释他们为什么想加入的个人消息(如果提供)
- **请求日期** -- 请求提交的时间

要审查请求:

1. 阅读此人的消息(如果他们提供了消息)
2. 如果需要,点击此人的姓名查看其个人资料
3. 决定是批准还是拒绝

## 批准请求

1. 点击加入请求上的 **批准**
2. 该人立即作为成员添加到小组
3. 请求者收到请求已批准的通知
4. 请求在系统中标记为已批准

:::tip
当您批准请求时,该人成为普通小组成员。如果需要,您可以稍后从[小组成员](./group-members.md)页面将他们提升为小组领导者。
:::

## 拒绝请求

1. 点击加入请求上的 **拒绝**
2. 可选择提供拒绝原因(最多500个字符)
3. 点击 **确认**
4. 请求者收到包含您的拒绝原因的通知(如果提供)
5. 请求标记为已拒绝

:::info
提供拒绝原因可以帮助该人了解为什么他们的请求未获批准,并可能鼓励他们稍后再试或探索其他小组。
:::

## 通知

加入请求系统会自动发送通知:

- **提交请求时** -- 所有小组领导者收到通知
- **批准请求时** -- 请求者收到确认
- **拒绝请求时** -- 请求者收到包含任何拒绝原因的通知

通知出现在 B1.church 和移动应用的用户通知中心。

## 从成员端管理请求

人们可以从 B1.church 管理自己的加入请求:

- 在小组详细信息页面查看其待处理请求的状态
- 如果改变主意,取消待处理的请求
- 查看他们的请求是否被批准或拒绝

## 最佳实践

- **及时响应** -- 尽量在24-48小时内审查请求,这样人们就不会一直等待
- **在拒绝原因中明确说明** -- 帮助人们了解后续步骤或替代选项
- **检查个人资料** -- 审查此人的个人资料,看他们是否适合该小组
- **沟通期望** -- 确保您的小组描述清楚地说明该小组的对象
