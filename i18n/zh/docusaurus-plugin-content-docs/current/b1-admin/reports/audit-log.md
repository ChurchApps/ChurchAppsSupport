# 审计日志

<div class="article-intro">

审计日志跟踪您的教会管理系统中的所有重要操作和更改。使用它来审查登录活动、跟踪谁更改了人员记录、监控权限更新，以及维护您团队的问责制。

</div>

<div class="prereqs">
<h4>开始前</h4>

- 具有服务器管理员访问权限的B1 Admin账户
- 导航到**设置**以查找审计日志

</div>

## 查看审计日志

1. 在B1 Admin中转到**设置**。
2. 选择**审计日志**。
3. 日志在一个表中显示最近的条目，包含以下列：
   - **日期** — 操作发生的时间。
   - **类别** — 操作的类型（颜色编码以便快速扫描）。
   - **操作** — 完成的操作（例如，创建、更新、删除、login_success）。
   - **实体** — 受影响的记录的类型和ID。
   - **IP地址** — 执行操作的用户的IP地址。
   - **详细信息** — 所做具体更改的摘要。

## 筛选日志

使用页面顶部的筛选器来缩小结果：

- **类别** — 按操作类型筛选：
  - **所有类别** — 显示所有内容。
  - **登录** — 登录成功和失败。
  - **人员** — 创建、更新或删除人员记录。
  - **权限** — 权限的授予和撤销。
  - **捐赠** — 捐赠记录更改。
  - **小组** — 小组管理操作。
  - **表单** — 表单提交活动。
  - **设置** — 配置更改。
- **开始日期** — 从此日期开始显示条目。
- **结束日期** — 显示到此日期的条目。

设置筛选器后单击**搜索**以更新结果。

## 理解类别

每个类别都有颜色编码以便快速识别：

- **登录** — 蓝色芯片。追踪成功和失败的登录尝试。
- **人员** — 紫色芯片。追踪人员记录的创建、更新和删除。
- **权限** — 红色芯片。追踪何时授予或撤销访问权限。
- **捐赠** — 绿色芯片。追踪捐赠记录的更改。
- **小组** — 灰色芯片。追踪小组管理操作。
- **表单** — 橙色芯片。追踪表单提交活动。
- **设置** — 黄色芯片。追踪配置更改。

## 导出日志

当显示日志条目时，会出现**CSV下载**按钮。单击它以将当前过滤结果导出到电子表格以进行离线审查或记录保存。

## 分页

使用表底部的分页控件来浏览结果。您可以每页显示25、50或100个条目。

:::info
审计日志条目自动保留一年。365天以上的条目被移除以保持系统的性能。
:::

:::tip
定期审查审计日志，尤其是在入职新团队成员或进行重大配置更改后。这有助于尽早识别意外活动。
:::
