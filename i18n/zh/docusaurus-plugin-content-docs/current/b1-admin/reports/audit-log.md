# 审计日志

<div class="article-intro">

审计日志跟踪您的教会管理系统中的所有重要操作和更改。使用它来审查登录活动、跟踪谁对人员记录进行了更改、监控权限更新，并维护您团队中的问责制。

</div>

<div class="prereqs">
<h4>开始前</h4>

- B1 Admin 账户，具有服务器管理员访问权限
- 导航至 **设置** 查找审计日志

</div>

## 查看审计日志

1. 在 B1 Admin 中转到 **设置**。
2. 选择 **审计日志**。
3. 日志在一个表格中显示最近的条目，包含以下列：
   - **日期** -- 操作发生的时间。
   - **类别** -- 操作类型（用颜色编码以便快速扫描）。
   - **操作** -- 做了什么（例如，创建、更新、删除、登录成功）。
   - **实体** -- 受影响的记录的类型和 ID。
   - **IP 地址** -- 执行操作的用户的 IP 地址。
   - **详细信息** -- 所做特定更改的摘要。

## 过滤日志

使用页面顶部的过滤器来缩小结果：

- **类别** -- 按操作类型过滤：
  - **所有类别** -- 显示所有内容。
  - **登录** -- 登录成功和失败。
  - **人员** -- 创建、更新或删除人员记录。
  - **权限** -- 权限授予和撤销。
  - **捐献** -- 捐献记录更改。
  - **小组** -- 小组管理操作。
  - **表单** -- 表单提交活动。
  - **设置** -- 配置更改。
- **开始日期** -- 显示此日期及以后的条目。
- **结束日期** -- 显示直到此日期的条目。

设置过滤器后点击 **搜索** 以更新结果。

## 理解类别

每个类别都用颜色编码以便快速识别：

- **登录** -- 蓝色芯片。跟踪成功和失败的登录尝试。
- **人员** -- 紫色芯片。跟踪人员记录创建、更新和删除。
- **权限** -- 红色芯片。跟踪何时授予或撤销访问权限。
- **捐献** -- 绿色芯片。跟踪捐献记录更改。
- **小组** -- 灰色芯片。跟踪小组管理操作。
- **表单** -- 橙色芯片。跟踪表单提交活动。
- **设置** -- 黄色芯片。跟踪配置更改。

## 导出日志

当显示日志条目时，会出现 **CSV 下载** 按钮。点击它以将当前过滤结果导出到电子表格以供离线审查或记录保存。

## 分页

使用表格底部的分页控件浏览结果。您可以每页显示 25、50 或 100 个条目。

:::info
审计日志条目会自动保留一年。早于 365 天的条目会被移除以保持系统性能。
:::

:::tip
定期审查审计日志，特别是在引入新团队成员或进行重大配置更改后。它有助于尽早识别意外活动。
:::
