# 已保存的列表

<div class="article-intro">

**已保存的列表**让您存储一个带有名称的搜索查询，并可以随时只需一次点击重新运行它。构建一次列表 -- "校区北部的活跃成员"、"过去 90 天内的访客"或任何高级筛选 -- 它会保留在您的侧边栏中，因此您永远不必再次重建查询。

</div>

<div class="prereqs">
<h4>开始前</h4>

- 在人员部分进行至少一次搜索，以便您有结果可保存。参见[搜索人员](./searching-people.md)或 [AI 搜索](./ai-search.md)。

</div>

## 将搜索保存为列表

1. 在**人员**页面上运行搜索 -- 快速搜索、AI 搜索或高级筛选搜索。
2. 当结果出现时，人员页面标题中会显示**保存为列表**按钮（书签图标）。
3. 单击**保存为列表**。
4. 为列表输入**名称**（必需）。
5. 可选地输入**类别**以在侧边栏中将相关列表分组在一起（例如"外展"或"成员身份"）。
6. 单击**保存**。

该列表现在已保存到您的教堂账户，并将出现在人员页面左侧的**已保存的列表**面板中。

:::info
列表在您的教堂中共享 -- 任何有人员访问权限的人都可以看到和加载它们。只有具有适当权限的员工才能重命名或删除列表。
:::

## 加载已保存的列表

在人员页面左侧的**已保存的列表**面板中，单击任何列表名称。搜索立即使用存储的查询重新运行，更新结果表。

列表按类别分组（如果您分配了一个），未分类的列表最后出现。

:::tip
列表是**实时查询**，不是快照。每次加载列表时，搜索都会针对您的当前数据新鲜运行，因此现在与条件匹配的新人员将出现。
:::

## 重命名和删除列表

具有管理权限的员工可以直接从已保存的列表面板重命名或删除任何列表。

- 单击列表名称旁边的**铅笔图标**以重命名它。
- 单击**垃圾桶图标**删除它（您将被要求确认）。

## 使用场景

| 场景 | 如何构建列表 |
|---|---|
| 所有访客 | 高级搜索：成员身份状态 = 访客 |
| 特定校区的成员 | 高级搜索：校区 = [您的校区] |
| 没有电子邮件地址的人员 | 高级搜索：电子邮件为空 |
| 具有过期背景检查的志愿者 | 高级搜索：[自定义字段](../settings/custom-fields.md)"背景检查到期"在今天之前 |
| AI 问题的结果 | 在 AI 搜索中提问，然后保存 |
