# 保存的列表

<div class="article-intro">

**保存的列表**让你将搜索查询存储在一个名字下，并随时只需单击一次就重新运行。构建一次列表 -- "校区北的活跃成员"、"过去 90 天的访客"或任何高级过滤 -- 它就会留在你的侧边栏中，所以你再也不用重新构建查询。

</div>

<div class="prereqs">
<h4>开始之前</h4>

- 在成员部分运行至少一次搜索，这样你有结果可以保存。详见[搜索成员](./searching-people.md)或[AI 搜索](./ai-search.md)。

</div>

## 将搜索保存为列表

1. 在**成员**页面上运行搜索 -- 快速搜索、AI 搜索或高级过滤搜索。
2. 当结果出现时，**保存为列表**按钮（书签图标）会在成员页面标题中显示。
3. 单击**保存为列表**。
4. 为列表输入一个**名字**（必需）。
5. 可选地输入一个**类别**来在侧边栏中将相关列表分组在一起（例如"外展"或"成员身份"）。
6. 单击**保存**。

列表现在已保存到你的教会账户，并将出现在成员页面左侧的**保存的列表**面板中。

:::info
列表在你的教会中被共享 -- 任何有成员访问权限的人都可以查看和加载它们。只有具有适当权限的员工可以重命名或删除列表。
:::

## 加载保存的列表

在成员页面左侧的**保存的列表**面板中，单击任何列表名字。搜索会立即用存储的查询重新运行，更新结果表。

列表按类别分组（如果你指定了一个），未分类的列表最后出现。

:::tip
列表是**动态查询**，不是快照。每次你加载一个列表，搜索会针对你当前的数据新鲜运行，所以现在符合条件的新人会出现。
:::

## 重命名和删除列表

具有管理权限的员工可以直接从保存的列表面板重命名或删除任何列表。

- 单击列表名字旁的**铅笔图标**来重命名它。
- 单击**垃圾桶图标**来删除它（你会被要求确认）。

## 使用案例

| 场景 | 如何构建列表 |
|---|---|
| 所有访客 | 高级搜索：成员身份 = 访客 |
| 特定分校区的成员 | 高级搜索：分校区 = [你的分校区] |
| 没有电子邮件地址的人 | 高级搜索：电子邮件为空 |
| AI 问题的结果 | 在 AI 搜索中提问，然后保存 |
