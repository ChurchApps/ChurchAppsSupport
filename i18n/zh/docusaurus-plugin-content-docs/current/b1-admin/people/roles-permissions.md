# 分配角色

<div class="article-intro">

B1 Admin 使用基于角色的权限系统来控制团队中每个用户可以看到和执行的操作。通过分配角色，您可以为工作人员和志愿者提供他们所需区域的精确访问权限——仅此而已。适当的角色管理可以确保教会数据安全，同时使您的团队能够高效工作。

</div>

<div class="prereqs">
<h4>开始之前</h4>

- 您需要 **Domain Admin** 访问权限或具有管理 B1 Admin 中 **Settings** 权限的角色。
- 您要分配角色的人员必须已存在于您的目录中。如果需要先添加他们，请参阅[添加人员](adding-people.md)。

</div>

## 了解角色

角色是您分配给一个或多个用户的一组权限。例如，您可以创建一个"财务团队"角色，授予访问[奉献记录](../donations/recording-donations.md)的权限，或创建一个"签到志愿者"角色，仅允许访问[考勤功能](../attendance/check-in.md)。

每个角色控制对 B1 Admin 特定区域的访问，包括：

- **人员** -- 查看和编辑成员个人资料
- **奉献** -- 管理捐款和财务报告
- **考勤** -- 记录和查看考勤数据
- **表单** -- 创建和管理[自定义表单](../forms/creating-forms.md)
- **小组** -- 管理[小组成员](../groups/group-members.md)和日历
- **设置** -- 配置全教会设置

:::warning
**域管理员**对 B1 Admin 的每个区域都拥有完全访问权限。他们的权限无法编辑或限制。此角色仅用于您的主要管理员。
:::

## 查看和管理角色

1. 点击左侧边栏中的 **Settings**。
2. 点击顶部导航中的 **Roles**。
3. 您将看到为您教会配置的所有角色列表。
4. 点击任何角色以查看其成员和权限。

## 向角色添加用户

1. 导航到 **Settings**，然后 **Roles**。
2. 点击您要添加用户的角色。
3. 在 **Members** 部分，按姓名搜索人员。
4. 点击 **Add** 将其分配到该角色。

该用户在下次登录时将拥有与该角色关联的所有权限。

## 编辑角色权限

1. 导航到 **Settings**，然后 **Roles**。
2. 点击您要修改的角色。
3. 在 **Permissions** 部分，勾选或取消勾选您希望该角色访问的区域。
4. 点击 **Save** 以应用更改。

:::tip
遵循最小权限原则——只给每个角色真正需要的权限。这可以保持数据安全并减少意外更改的可能性。
:::

## 常见角色示例

- **办公室工作人员** -- 访问人员、奉献、考勤和表单
- **小组组长** -- 仅访问[小组](../groups/creating-groups.md)
- **签到志愿者** -- 仅访问[考勤](../attendance/check-in.md)
- **财务团队** -- 访问[奉献](../donations/recording-donations.md)和报告
